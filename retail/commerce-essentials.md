# <a name="commerce-essentials-overview"></a>商业要素概览

[!include[banner](includes/banner.md)]


本文提供有关商业要素的信息，其是 Microsoft Dynamics 365 for Operations 的零售销售点 (POS) 和电子商务配置选项。 商业要素提供可以帮助零售商使用基于 Microsoft Dynamics 的零售解决方案迅速运营并让他们关注特定于零售的功能的现成零售工作负载。 

<a name="overview"></a>概览
--------

商业软件环境通常包含许多系统，每个系统均执行非常具体的功能。 通常，零售商希望利用 Microsoft Dynamics 365 for Operations 的零售管理功能，而不希望替换其现有的财务或人力资源 (HR) 系统。 对于这些零售商，需要特定于零售方面的工作负载。 商业要素将这些特定于零售的功能作为 Dynamics 365 for Operations 的配置选项提供。 在完成配置后，解决方案的所有非零售方面将隐藏，以便将精力放在重要方面：管理零售组织。 商业要素首先是作为 Microsoft Dynamics AX 2012 R3 CU8 中的配置选项引入的。 自那以后，已实施许多改进以使该解决方案适用于全方位渠道零售商，而不只是适用于传统组织。

## <a name="configuration"></a>配置
要配置商业要素，请转至**系统管理** &gt; **设置** &gt; **许可证配置**，然后向下滚动至**零售**。 展开**“零售”**节点，然后选中**“商业要素”**的复选框。 单击**“保存”**，查看更改，然后刷新页面。 要切换回完整解决方案，请转至**商业要素** &gt; **总部设置** &gt; **许可证配置**，然后向下滚动至**零售**。 展开**“零售”**节点，然后清除**“商业要素”**的复选框。 单击**“保存”**，查看更改，然后刷新页面。

## <a name="what-is-included"></a>包含哪些内容？
商业要素包含内置于核心 Dynamics 365 for Operations 零售解决方案中的所有零售后端管理系统功能（呼叫中心功能除外）。 所有渠道、工作人员、产品、商店、收银机和设备管理功能都包含在一个导航节点中。 此外，将包含所有零售工作区以提供现用的角色特定的登录页。 商业要素面向的是具有传统或移动方案的零售商，他们可从 Retail Modern POS (MPOS) 和电子商务渠道获益。 通过使用商业要素，零售商可在一个位置管理这些渠道，从而获得真正的全渠道功能。

## <a name="how-is-commerce-essentials-different"></a>商业要素有哪些不同？
除了提供统一的导航和仅零售页面之外，商业要素将隐藏所有非零售页面元素，例如字段和控件。 结果是一个简化的用户界面，可帮助零售商将精力放在手头的任务上。 具有完整产品许可证的零售商可使用商业要素作为适用于多个面向零售的用户的起始点。 如果需要，可根据现用的许可协议的条款向商业要素添加其他元素（例如，高级 HR 功能）。 作为商业要素实现的简化的一部分，会计导出功能可让零售商将商业要素中的应计总量映射到其现有会计系统中的帐号。 在零售系统生成财务数据时，可轻松地将此数据从商业要素导出到零售商的会计系统。 如果需要更高级的集成功能，商业要素还可包含 Dynamics 365 for Operations 附带的完整导入/导出框架。 利用默认情况下包含的数以百计的数据实体，商业要素能够灵活地适应使用了多个专用系统的多层环境。

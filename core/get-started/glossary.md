# <a name="dynamics-365-for-operations-glossary"></a>Dynamics 365 for Operations 词汇表

[!include[banner](../includes/banner.md)]


Microsoft Dynamics 365 for Operations 词汇表定义 Dynamics 365 for Operations 产品中的重要术语和概念。 

### <a name="a"></a>**A**

###### <a name="accounting-journal"></a>**会计日记帐**

用于在会计系统中记录会计事件财务影响的日志。

###### <a name="accounting-rule"></a>**会计规则**

控制原则、方法与分类、记录和报告会计事件财务影响的程序的会计系统中的规则。

###### <a name="ach-automated-clearing-house"></a>**ACH（自动化清算中心）**

电子资金转帐系统在接收当事方和源当事方之间便于资金转账的美国可用。

###### <a name="actual-quantity"></a>**actual quantity / 实际数量**

输入或从活动的输出的测定数量。

###### <a name="address-verification-service"></a>**地址验证服务**

由信用卡处理程序所提供的服务验证了持卡人提供的结算地址与在发货银行的文档有关信息匹配。

###### <a name="adjusted-unit-price"></a>**调整后的单价**

根据基于属性的定价和已接收的实际效能计算的单位价格。

###### <a name="advanced-pricing"></a>**高级定价**

用于计算含量物料的采购价的逻辑。

###### <a name="aging"></a>**aging / 帐龄**

按年限分类时段的流程。

###### <a name="aging-period"></a>**帐龄期间**

该时段的天数用于报告逾期客户付款余额。

### <a name="b"></a>**B**

###### <a name="balance-sheet"></a>**balance sheet / 资产负债表**

组织的财务状况的报表报告在指定日期资产、负债和权益的状态。

###### <a name="balance-sheet-account"></a>**资产负债表科目**

在资产负债表报表描述在值中进行的更改的帐户。

###### <a name="bank-reconciliation"></a>**bank reconciliation / 银行对帐**

通过将银行对帐单物料与会计科目惯例进行匹配表示银行帐户的对帐会计科目实践。

###### <a name="batch-attribute"></a>**batch attribute / 批次属性 **

处理批或转移批处理的产品属性。

###### <a name="best-before-date"></a>**此日期前最佳**

获取产品的最佳质量或特性的建议日期。

###### <a name="best-before-period"></a>**最佳使用期限**

获取产品的最佳质量或特征的时段。

###### <a name="bill-of-exchange"></a>**bill of exchange / 汇票**

论述第三方按需支付第二个当事方的无条件请求的源文档。

###### <a name="bill-of-materials"></a>**bill of materials / 物料清单**

要求生产一个产品的产品列表及其数量。

###### <a name="blocking"></a>**锁定**

将文档或产品的操作处于暂停状态。

###### <a name="budget-control"></a>**预算控制**

仅当保留预算资金以实现未来的付款承诺时授权支出的行为。

###### <a name="budget-control-dimension"></a>**预算控制维度**

有效的财务维度值组合用于分配的调拨预算资金到计划活动付款。

###### <a name="budget-control-rule"></a>**预算控制规则**

业务决策的代码根据分配给由有效的预算控制维度值组合定义的详细或聚合活动可用的预算资金检查承诺和实际支出。

###### <a name="budget-cycle-time-span"></a>**预算周期跨度**

指定为多个会计年度期间的时段。 预算期间可以不同于会计期间。

###### <a name="budget-group"></a>**预算组**

在预算维度结构层次中通过合计预算分配给子公司的财务维度值用于计算聚集预算分配给高级的财务维度值的一组财务维度值。

###### <a name="budget-model"></a>**budget model / 预算模型**

用于安排预算资金分配和支出的计划结构。

###### <a name="bulk-item"></a>**bulk item / 散装物料**

配方物料输入到产品交货活动。

###### <a name="bundle"></a>**bundle / 捆绑**

多个产品的组合作为一个单元销售。

###### <a name="business-unit"></a>**业务单位**

创建来满足战略业务目标的一个半自主运营单位。

### <a name="c"></a>**C**

###### <a name="capacity-load"></a>**capacity load / 产能负荷**

工作中心在要求的产能等级可以执行计划工作的最大量。

###### <a name="capacity-planning"></a>**capacity planning / 产能规划**

在特定时段确定满足将来输出要求所需的资源产能需求的过程。

###### <a name="carry-forward-budget"></a>**结转预算**

从一个会计年度转移到下一个，并且在新的会计年度为未结采购订单预留的预算。

###### <a name="cash-generating-unit"></a>**现金产出单元**

在组织内独立于其他资产产出现金的可分类资产的最小组。 这些资产组用于度量将会引发的减损损失。

###### <a name="catch-weight"></a>**实际称重**

销售物料或库存物料的实际重量或标称重量。

###### <a name="chart-of-accounts"></a>**chart of accounts / 会计科目表**

主账目列表。

###### <a name="cogs-cost-of-goods-sold"></a>**COGS（所售货物成本）**

用于合计制造产品和附有库存财务影响的会计类别。

###### <a name="commodity-pricing"></a>**commodity pricing / 商品定价 **

从生产主题到常用的价格变更生产的更新销售物料价格的价格策略。

###### <a name="consolidated-order"></a>**consolidated order / 合并的订单**

提供多个引用订单的源文档。

###### <a name="contractor"></a>**contractor / 转包商**

参与和法人的承包商雇主关系的人员假定的角色。

###### <a name="co-product"></a>**co-product / 联产品 **

与其他物料共同生成的物料。

###### <a name="cost-center"></a>**成本中心**

经理对预算的和实际支出负问责性的工序单位。

###### <a name="cost-of-goods-sold"></a>**所售货物成本**

用于合计制造产品和附有库存财务影响的会计类别。

###### <a name="cost-variance"></a>**成本差异**

预期成本和实际成本之间的差额。

###### <a name="costing"></a>**成本计算**

计算、分配和分摊由组织要求、生产或交货的经济资源成本的过程。

###### <a name="ctp-capable-to-promise"></a>**CTP（可承诺量）**

可用工序运营资源和可用输入产品中产品输出的部分要求满足特定用户的要求。

###### <a name="currency-code"></a>**币种代码**

表示币种单位的字母数字标识符。

###### <a name="cycle-time"></a>**周期时间**

所用的时间完成活动。

### <a name="d"></a>**日**

###### <a name="dashboard"></a>**仪表板**

Dynamics 365 for Operations 中的典型起始页。 在仪表板中，用户将看到其有权访问的每个工作区的部分。 每个部分显示相关工作区的汇总部分的磁贴。 仪表板包括磁贴的名称和部分。

###### <a name="data-entity"></a>**数据实体**

表示数据实体特征的信息结构。

###### <a name="delegate"></a>**委托**

委托是表示对包含特定参数列表和退货类型的方法的引用的类型。

###### <a name="delivery-note"></a>**交货通知**

论述产品交货双方之间交货的业务单据。

###### <a name="demand-forecast"></a>**需求预测**

将来的产品需求的预测。

###### <a name="demand-forecasting"></a>**需求预测**

根据历史交易记录数据估计未来需求并创建需求预测的业务流程。

###### <a name="deployable-package"></a>**可部署包**

用于在用户接受测试 (UAT) 和生产环境部署的车辆。

###### <a name="designer"></a>**设计器**

用于创建、更新和检查您的模型元素的 Visual Studio 工具。

###### <a name="financial-dimensionsdimension-based-configuration"></a>**  \财务维度\基于维度的配置**

通过选择产品维度值用于创建产品变型的配置技术。

###### <a name="distinct-product"></a>**独特产品**

唯一可识别的产品。

###### <a name="document-blocking"></a>**单据锁定**

在订购的物料在质量保证过程时，将工序方式处理为暂停状态。

###### <a name="document-management-system"></a>**文档管理系统**

存储和处理组织文档的应用程序服务。

###### <a name="duty"></a>**税**

在安全模型中，要求用户执行他们的职责的一组申请访问权限。

### <a name="e"></a>**E**

###### <a name="eft-electronic-funds-transfer"></a>**EFT（电子资金转帐）**

转移的资金的网络系统从某一银行帐户到另一个。

###### <a name="event"></a>**事件**

事件允许类或对象在出现感兴趣的事物时通知其他类或对象。

###### <a name="epe-every-product-every"></a>**EPE（生产批次）**

用于建立一个常规重复执行的生产周期中的精益概念。

###### <a name="extension"></a>**分机**

通过添加功能到现有的代码来自定义应用程序。

### <a name="f"></a>**五**

###### <a name="financial-dimension"></a>**财务维度**

组织中的当事方、职位、产品和活动创建的财务数据分类符并用于管理报告。

###### <a name="financial-dimension-value"></a>**财务维度值**

在财务维度的域中的数据元素。

###### <a name="financial-statement"></a>**financial statement / 财务报表**

论述组织的财务信息和财务状况的报表。

###### <a name="fixed-cost"></a>**fixed cost / 固定成本**

成本不随产品交货生产量或输出上的更改而更改。

###### <a name="fixed-currency"></a>**固定汇率币种**

具有与其他币种有关的固定汇率的币种。

###### <a name="fixed-quantity-kanban"></a>**固定数量看板**

当分配给规则的看板数固定时使用的看板类型。

###### <a name="flexible-authentication"></a>**灵活认证**

各种标识管理系统中的用户可以进行验证的身份验证过程。

###### <a name="formula"></a>**配方**

在生产流程输入和生产流程输出中的一个数值关系。

###### <a name="formula-item"></a>**配方物料**

配方控制的批处理的输出。

### <a name="g"></a>**G**

###### <a name="general-budget-reservation"></a>**常规预算预留**

由公共部门实体保留或标记预算资金使用的文件，以便这些资金不能用于其他目的。

###### <a name="grace-period"></a>**grace period / 宽限期**

无需补偿满足义务期间超过特定日期的时段。

###### <a name="gst-goods-and-services-tax"></a>**GST（商品劳务税）**

在某些国家/地区的征收增值税。

### <a name="i"></a>**I**

###### <a name="intercompany"></a>**intercompany / 内部公司**

之间发生或合并所有法人帐户的部分相同组织的相关两个或多个法人。

###### <a name="invoice-matching"></a>**发票匹配**

匹配的供应商发票价格和产品数量的实践到采购订单和产品收货。

###### <a name="item-allocation-key"></a>**物料分配参数**

用于预测和需求计划产品系列分组。

###### <a name="item-relation"></a>**物料关系**

看板规则中对物料分配组或物料及其产品维度的参考。

### <a name="k"></a>**K**

###### <a name="kanban"></a>**看板**

传达产品数量的要求的符号。

###### <a name="kanban-flow"></a>**看板流**

定义为看板规则创建的看板执行的活动的顺序。

###### <a name="kanban-job"></a>**看板作业**

由看板触发的生产流中流程或转移活动。

###### <a name="kanban-job-consumption"></a>**看板作业消耗量**

撤销来自库存的产品组件以便完成看板生产作业。

###### <a name="kanban-product-quantity"></a>**看板产品数量**

基于看板规则在看板中装运的物料需求数量。

###### <a name="kanban-rule"></a>**看板规则**

通过控制如何流程并转移在生产流中通用的活动认识到物料计划和补充策略的一个精益制造系统中的规则。

### <a name="l"></a>**L**

###### <a name="lean-manufacturing"></a>**倾斜制造**

制造工序促进精益生产流和业务活动的理念。

###### <a name="lean-schedule-group"></a>**精益计划组**

聚合生产物料的方法，例如，基于设置组、装运组或运输组。

###### <a name="ledger-account"></a>**ledger account / 会计科目**

从主科目值的组合以及其他财务维度值创建的分类符在会计科目表中列出并用于分类经济活动财务影响。

###### <a name="license-code"></a>**许可证代码**

根据许可证协议的条件授予权限以使用软件或软件组件，并且启用和停用软件模块、软件功能和软件职能的字母数字参数。

###### <a name="liquidity"></a>**liquidity / 流动性**

当事方使用当前资产结算流动负债的能力。

### <a name="m"></a>**一**

###### <a name="manufacturing-overhead-based-on-material"></a>**基于材料的制造开销**

制造开销表示为针对附加费百分比的间接成本计算公式。 分配给材料组件的成本组充当计算公式的基础。 可为单独的产品或为分配给产品的物料组定义计算公式，以在计算基于材料的制造开销时提高灵活性。

###### <a name="master-scheduling"></a>**master scheduling / 主计划编制**

为满足供求关系生产时间表的流程。

###### <a name="model"></a>**模型**

模型包含代码元素和引用元数据。 在 Visual Studio 中创建和检查。

###### <a name="model-element"></a>**模型元素**

模型是构成可分配软件解决方案的一组元素（源文件和元数据）。 模型是设计时概念。

### <a name="n"></a>**N**

###### <a name="nbv-net-book-value"></a>**NBV（帐面净值）**

按固定资产减去其累计折旧的原始成本之间的差值计算的固定资产的值。

### <a name="o"></a>**O**

###### <a name="object-permission"></a>**对象权限**

在安全对象中允许创建、阅读、更新、删除或执行工序。

###### <a name="operating-unit"></a>**运营单位**

将经济资源和实现的流程控制在人员之间的组织，有义务最大化对稀有资源的使用、改进流程和记录他们的业绩

###### <a name="output-product"></a>**产出产品**

从活动中获得的实际产品。

###### <a name="overlayering"></a>**覆盖**

通过提供覆盖默认源代码的新源代码来自定义源代码的方法。

### <a name="p"></a>**P**

###### <a name="package"></a>**套装**

包是一个或多个模型的可编译和可部署的单位。

###### <a name="party"></a>**当事方**

在参与契约关系时中假定法律约束力角色的实体。

###### <a name="party"></a>**当事方**

参与经济活动的人员或组织。

###### <a name="pegging"></a>**pegging / 限定标准**

跟踪需求物料数量到其来源的过程。

###### <a name="pegging-event"></a>**需求声明事件**

用信号通知产品需求的资源流事件。

###### <a name="phantom-item"></a>**虚拟物料**

要生产但不存储的物料。

###### <a name="planned-intercompany-demand"></a>**计划内部公司需求**

按假定供应商角色的法人的产品预测需求。 为从顺流法人中的产品，需求计划从计划需求生成。

###### <a name="post"></a>**过帐**

在特定的帐户记录经济事件的货币值，或者汇总并重新分类一般和子公司会计科目物料到一般和子公司会计科目物料。

###### <a name="preparer"></a>**准备人**

创建源文档以启动经济资源请求的人员。

###### <a name="privilege"></a>**特权**

用户需要执行任务的一个或多个权限。

###### <a name="procurement-catalog"></a>**采购目录**

产品类别分组的采购供应列表。 用于要求由组织内部使用的产品的采购目录。

###### <a name="product-component"></a>**产品组件**

产品的一部分。

###### <a name="product-dimension"></a>**产品维度**

用于..\财务维度\基于维度的配置的大小、颜色或配置产品属性。

###### <a name="product-family"></a>**产品系列**

参与同一生产或装运活动或为相同市场细分提供的唯一组物料、服务或权限。 在 Dynamics 365 for Operations 中，通过使用预测分配参数表示分组。

###### <a name="product-master"></a>**基础产品**

配置产品变型的基础的标准或功能产品表示。

###### <a name="product-receipt"></a>**产品收据**

论述托运中预定的产品收据、退货的产品收据或收到的产品收据的原始凭证。

###### <a name="product-variant"></a>**产品变型**

基础产品的配置。

###### <a name="production-flow"></a>**生产流**

使用“精益原则”设计的生产流程。

###### <a name="production-flow-model"></a>**生产流模型**

在生产流程进程中的一组工作单元提供的生产产能表示形式。

###### <a name="production-order"></a>**production order / 生产订单**

论述需要生产物料满足需求的源文档。

###### <a name="production-schedule"></a>**生产计划**

在指定时间由特定人力运营资源生成物料和物料数量的计划。

###### <a name="prospect"></a>**prospect / 目标客户**

具有现有和潜在能力来提供服务或可能的将来经济福利给法人的参与者。

###### <a name="purchase-agreement-policy"></a>**采购协议政策**

授权当事方来修改采购协议条款的策略。

###### <a name="purchase-order-policy"></a>**采购订单政策**

授权当事方修改采购订单条款和控制订单处理的策略。

###### <a name="purchase-quotation"></a>**采购报价单**

针对指定的价格以及截止到指定的日期响应采购流程中的报价请求而记录采购产品数量的报价的源文档。

###### <a name="purchase-requisition"></a>**purchase requisition / 采购申请**

提供产品请求以便可以供提交审核和按某一采购的组织用于授权采购的源文档。

### <a name="q"></a>**Q**

###### <a name="quantity-variance"></a>**数量差异**

在没有预计其他收据时的已订购产品数量和已接收产品数量之间的差别。

### <a name="r"></a>**R**

###### <a name="receipt-advice"></a>**收货通知**

提供采购员从供应商接收产品汇总的业务文档。

###### <a name="reconciliation"></a>**对帐**

调整两个或多个账户或报表的实践，以便审核。

###### <a name="reduction-key"></a>**缩减参数**

根据应用于特定期间的用户定义百分比，增加或缩减预测需求的一种方法。

###### <a name="register"></a>**收银机**

用于记录会计系统中记录运营的和资源流事件的财政影响的记录。

###### <a name="regulated-product"></a>**受管制产品**

要求符合性和审计报告的政府当局管制的产品。

###### <a name="rfq-request-for-quotation"></a>**RFQ（询价）**

针对指定价格中并按特定日期提供邀请以投标供应产品数量的源文档。

###### <a name="requester"></a>**申请人**

请求经济资源的人员。

###### <a name="restricted-product"></a>**受限制产品**

分配由政府当局限制或约束的产品。

### <a name="s"></a>**六**

###### <a name="sales-agreement"></a>**销售协议**

提供在两个或更多个当事方之间的协议的源文档基于销售当事方将确定在某一时间销售特定数量或值以交换有利价格和折扣的理解。

###### <a name="sales-agreement-policy"></a>**销售协议政策**

授权当事方来修改销售协议条款的策略。

###### <a name="sales-order-policy"></a>**销售订单政策**

授权当事方修改销售订单条款和控制处理和付款流程的策略。

###### <a name="sales-quotation"></a>**sales quotation / 销售报价单**

针对指定的价格以及截止到指定的日期响应销售流程中的报价请求而记录供应产品数量的报价的源文档。

###### <a name="security-role"></a>**安全角色**

定义的一组应用程序访问特权。 安全角色分配给用户以确定哪些任务用户可以执行，并且哪些部分用户界面用户可以进行查看。 为了访问系统所有用户必须分配到至少一个安全角色。

###### <a name="segregation-of-duties"></a>**职责划分**

设计用于分隔记录、验证、资产权限和参与、提供或记录经济交易财政影响的人员的缩减欺骗风险、不规则性和错误的原则。

###### <a name="sequencing"></a>**sequencing / 排序 **

处理作业或工序在生产设备执行以实现目标的订单。

###### <a name="shared-asset"></a>**公用资产**

用于多个现金产出单元（CGU）的资产。 一个示例就是配送中心，用于在物料运输到共享相同配送中心的不同市场之前对物料进行存储。

###### <a name="single-use-kanban"></a>**一次性看板**

使用固定数量看板规则满足格外高的要求的看板类型。 放弃时不触发新看板的一次性看板。

###### <a name="smart-rounding"></a>**智能舍入**

市场事件使用略微与少于其最近整数的奇数设置价格。

###### <a name="source-requirement"></a>**源需求**

在创建限定标准源文档行中记录的产品数量。

###### <a name="statistical-baseline-forecast"></a>**统计基准预测**

通过将预测算法应用于历史交易记录数据而创建的一种未来需求预测。

###### <a name="supermarket"></a>**超市**

具有有限产能并用于接近消耗量方面的批次物料的实时库存库位。

###### <a name="system-defined-financial-dimension-type"></a>**系统中定义的财务维度类型**

财务维度映射到某个表或在表示实体类型的数据库中查看，并且其值在该实体类型一个属性域中。

### <a name="t"></a>**二**

###### <a name="takt-time"></a>**单位产品生产时间**

所用生产产品的一个单位的时间。

###### <a name="task-guide"></a>**任务指南**

通过任务录制的步骤实现的受控式、指导式体验。 任务指南通过记录的经验引导用户。 所有安全、数据和申请行为在完成任务指南时是相同的，因为它们没有任务指南。 任务指南使用相同的仪器作为任务录制器以了解用户什么时候完成了预期步骤，以便可以提示用户执行录制中的下一步骤。

###### <a name="task-recorder"></a>**任务录制器**

在 Dynamics 365 for Operations 中预安装的工具。 在录制时，它记录用户在针对服务器执行的 Dynamics 365 for Operations 用户界面中输入的所有事件 - 包括添加的值、更改的设置、删除的数据等。

###### <a name="task-recording"></a>**任务录制**

包含任务录制器在 Microsoft Dynamics 365 for Operations 中运行时捕获的操作和注释的文件。

###### <a name="three-way-matching-policy"></a>**三向匹配政策**

要求一个或多个供应商发票价格与一个或多个采购订单价格匹配，并且要求一个或多个供应商发票数量与一个或多个产品收货数量匹配的匹配策略。

###### <a name="transfer-batch"></a>**转移批处理**

转移或可转移的一个或多个物料数量。

###### <a name="two-way-matching-policy"></a>**双向匹配政策**

要求一个或多个供应商发票价格与一个或多个采购订单价格匹配的匹配策略。

### <a name="u"></a>**U**

###### <a name="user-defined-financial-dimension-type"></a>**用户定义的财务维度类型**

财务维度值映射到用户定义的值列表。

### <a name="v"></a>**V**

###### <a name="value-stream"></a>**价值流**

控制一个或多个生产流的工序单位。

###### <a name="variant-configuration-technology"></a>**变型配置技术**

基础产品建模方法和搜索产品变型配置。

###### <a name="vat-value-added-tax"></a>**VAT（增值税）**

基于该阶段期间增加的值，在其生产的每一个阶段期间的一种税收。

###### <a name="vendor-catalog"></a>**供应商目录**

产品供应列表要从供应商购买才可用。

###### <a name="vendor-invoice"></a>**供应商发票**

论述供应商付款请求的源文档。 供应商发票可以引用一个或多个采购订单。 在授权供应商发票时，可以向供应商付款。

### <a name="w"></a>**三**

###### <a name="work-cell"></a>**工作单元**

参与生产流活动的资源组。

###### <a name="worker"></a>**工作人员**

假定员工或合同工的角色以及支付以交换服务的人员。

###### <a name="workspace"></a>**工作区**

提供用户执行的一个活动的概览的页面。 此页面使用磁贴、列表和图表显示待定工作和来自多个来源的相关数据，而且此页面显示与此数据相关的常见任务。 在工作区的汇总部分中显示的磁贴也在仪表板的相关部分显示。

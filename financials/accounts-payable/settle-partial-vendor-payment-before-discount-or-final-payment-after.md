# <a name="settle-a-partial-vendor-payment-before-the-discount-date-with-a-final-payment-after-the-discount-date"></a>在折扣日期之前结算后的一部分供应商付款，在最终付款的折扣日期。

本文向您介绍执行多个部分付款的情况，部分在现金折扣期间内，其他在现金折扣期间外。

Fabrikam 从采购供应商 3057 的货物。 如果发票在 14 天，支付 Fabrikam 接收 1% 的现金折扣。 发票必须在 30 天时支付。 供应商还允许 Fabrikam 在部分付款上采取现金折扣。 **结算参数在应付帐款参数**页。

## <a name="invoice-on-june-25"></a>6 月 25 日的发票
在 6 月 25 日，并且输入 4 月 1,000.00 过帐的发票为供应商 3057。 April 可以在“供应商交易记录”****页上查看该交易记录。

| 凭证   | 交易记录类型 | 日期      | 开票 | 交易币种借方金额 | 交易币种贷方金额 | 余额   | 货币 |
|-----------|------------------|-----------|---------|--------------------------------------|---------------------------------------|-----------|----------|
| Inv-10020 | 开票          | 6/25/2015 | 10020   |                                      | 1,000.00                              | -1,000.00 | USD      |

## <a name="partial-payment-on-july-2"></a>7 月 2 日的部分付款
7 月 2 日，April 想要结算此发票的 300.00。 付款可享受折扣，因为 Fabrikam 在部分付款上采取折扣。 因此，April 支付 297.00 并且有 3.00 折扣。 她为付款日志和输入供应商 3057 的一行。 她然后打开** **结算交易记录页，因此，她可以标记结算的发票。

| 标记     | 使用现金折扣 | 凭证   | 帐户 | 日期      | 到期日期  | 开票 | 交易记录币种金额 | 货币 | 要结算的金额 |
|----------|-------------------|-----------|---------|-----------|-----------|---------|--------------------------------|----------|------------------|
| 已选择 | 标准            | Inv-10020 | 3057    | 6/25/2015 | 7/25/2015 | 10020   | -1,000.00                      | USD      | -297.00          |

折扣信息显示在“结算未结交易记录”****页的底部。

|                              |           |
|------------------------------|-----------|
| 现金折扣日期           | 7/09/2015 |
| 现金折扣金额         | -10.00    |
| 使用现金折扣            | 标准    |
| 提取了现金折扣          | 0.00      |
| 要提取的现金折扣金额 | -3.00     |

然后，April 过帐该付款。 现在发票具有 700.00 余额。 April 可以在“供应商交易记录”****页上查看该交易记录。

| 凭证    | 交易记录类型 | 日期      | 开票 | 交易币种借方金额 | 交易币种贷方金额 | 余额 | 货币 |
|------------|------------------|-----------|---------|--------------------------------------|---------------------------------------|---------|----------|
| Inv-10020  | 开票          | 6/25/2015 | 10020   |                                      | 1,000.00                              | -700.00 | USD      |
| APP-10020  | 付款          | 7/1/2015  |         | 297.00                               |                                       | 0.00    | USD      |
| DISC-10020 | 现金折扣    | 7/1/2015  |         | 3.00                                 |                                       | 0.00    | USD      |

## <a name="remaining-payment-on-july-15-use-cash-discount--normal"></a>7 月 15 日的其余付款，使用现金折扣 = 常规
April 在 7 月 15 日（现金折扣期间后）支付此发票的剩余金额。 在“结算未结交易记录”****页上，“估计的现金折扣”****字段中不显示任何折扣金额，并且“现金折扣金额”****字段中的值为“0.00”****。 在 April 支付剩余的 700.00 时，将不会获得其他的折扣。

| 标记     | 使用现金折扣 | 凭证   | 帐户 | 日期      | 到期日期  | 开票 | 交易记录币种金额 | 货币 | 要结算的金额 |
|----------|-------------------|-----------|---------|-----------|-----------|---------|--------------------------------|----------|------------------|
| 已选择 | 标准            | Inv-10020 | 3057    | 6/25/2015 | 7/25/2015 | 10020   | -700.00                        | USD      | -700.00          |

折扣信息显示在“结算交易记录”****页的底部。 April 可以看到自己已获得 3.00 折扣。

|                              |           |
|------------------------------|-----------|
| 现金折扣日期           | 7/09/2015 |
| 现金折扣金额         | 0.00      |
| 使用现金折扣            | 标准    |
| 提取了现金折扣          | -3.00     |
| 要提取的现金折扣金额 | 0.00      |

然后，April 过帐付款。 在她打开“供应商交易记录”****页时，她将看到发票的余额为 0.00。 她还将具有两个付款。 一个付款针对 297.00 和 3.00 折扣；并且，另一个付款针对 700.00。

| APP-10020  | 付款          | 7/1/2015  |         | 297.00                               |                                       | 0.00    | USD      |
| DISC-10020 | 现金折扣    | 7/1/2015  |         | 3.00                                 |                                       | 0.00    | USD      |
| APP-10021  | 付款          | 7/15/2015 |         | 700.00                               |                                       | 0.00    | USD      |

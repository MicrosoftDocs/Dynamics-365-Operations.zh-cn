# <a name="consolidation-account-groups-and-additional-consolidation-accounts"></a>合并科目分组和其他合并的科目

此主题提供有关合并科目分组和合并其他的帐户的信息，并说明如何在 Microsoft Dynamics 365。工序。

<a name="consolidation-account-groups"></a>合并帐户组
----------------------------

合并科目分组允许您创建您想要使用合并数据科目的组。 通常，合并科目分组表示针对政府的会计科目表或将它们映射帐户分配到的公司由总部定义的组。 您可以找到**合并** **模块的区域设置**的合并科目分组。 在您添加新组时，您输入帐户组和名称的唯一标识符。

## <a name="additional-consolidation-accounts"></a>其他合并帐户
其他合并的帐户可以从分配现有会计科目表的帐户到合并科目分组。 可以指定帐户值合并和名称。 

您可以找到。**合并** **模块的区域设置**的其他合并科目。 在创建新的合并科目时，至少必须指定以下信息：

-   ** –主科目**此字段为显示所有主科目基于会计科目表您选择页面上的查找。 在您选择该科目时，主科目名称。**名称**字段中自动输入。
-   **合并科目分组** –使用此字段指定组帐户分配。 如果在将以两种不同的方式，您必须添加同一科目到四合并所有帐户组。
-   ** **合并科目–输入合并帐户的值。 此值在此不必须从会计科目表中的科目。 它可以是所需的值。
-   **合并科目名称** –输入帐户名称，您希望显示在查询和报表。
-   ** **此字段 SAT 级别–帐户对帐单墨西哥用于向税务主管机构申报。 

在您完成创建您的帐户合并组和附加的合并科目时可以通过选择某合并，联机处理的组。

# <a name="pay-when-paid-vendor-agreements"></a>即收即付的供应商协议

[!include[banner](../includes/banner.md)]


本文说明供应商协议的即收即付 (PWP) 条款。 PWP 条款让您可以部分或全部预扣对供应商的付款，直到您的项目的客户支付您的款项。 本文还提供一个真实示例显示如何为项目使用 PWP 条件。

当您批准供应商作为转包商参与项目时，可能想要预扣给供应商或转包商的付款，直到客户向您支付项目的付款。 为了保留对供应商的付款，在设置供应商的采购订单时，请设置即收即付 (PWP) 条款。 在您为供应商创建采购订单并且将项目 ID 分配给采购订单时，项目中的 PWP 条款与采购订单和供应商关联。 在**即收即付供应商发票**页上，您可以查看采购订单和关联的客户发票的未付款供应商发票列表。 使用此窗体确定是否向供应商付款，以及付多少款。 例如，客户支付项目发票上的金额时，您可以释放部分或全部相关供应商发票的付款。 从客户收到特定百分比的相关付款后，您可以设置 PWP 条款以指定对供应商付款。 从客户收到部分付款时，您可以手动释放部分相关供应商发票的付款。 以下示例显示您可以如何使用 PWP 条款来预扣给供应商或转包商的付款，直到客户给您付款。 您的组织与客户达成协议按客户要求要在其上安装软件的 100 台计算机。 您和客户一致同意每台计算机的价格为 150.00。 您聘请第三方供应商的服务为您提供计算机。 客户想在支付您的组织之前审核计算机的质量。 您的组织政策是预扣对第三方供应商的付款，直到客户就项目对您进行了付款。 因此，您将项目的 PWP 百分比设置为 100%，这样，您预扣对供应商的所有付款，直到收到客户发票的完全付款。 供应商为每台计算机支付 100.00。 供应商将计算机递送给您时，装运包括 100 台计算机价值 10,000.00 的发票。 由于您已设置项目的 PWP 条款，因此无需向供应商付款。 从供应商收到计算机后，您在计算机上安装所需的软件。 您将计算机递送给客户时，其中还包括 15,000.00 的发票。 客户检查计算机并审核产品质量。 然后，客户支付您的组织项目发票的全部金额。 从客户处收到完全付款后，您向供应商支付供应商发票的全部金额 10,000.00。

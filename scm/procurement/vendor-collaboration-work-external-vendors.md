# <a name="vendor-collaboration-with-external-vendors"></a>供应商与外部供应商协作

[!include[banner](../includes/banner.md)]


本主题说明采购代理如何与外部供应商协作交换关于采购订单和托运库存的信息。

**供应商协作** 模块针对的是没有与 Microsoft Dynamics 365 for Operations 进行电子数据交换 (EDI) 集成的供应商。 它允许供应商使用采购订单、发票和托运库存信息。 本主题介绍您可以如何与使用供应商协作界面的外部供应商协作处理采购订单和托运库存。 还介绍了如何允许特定供应商使用供应商协作，以及如何定义所有供应商在响应采购订单时将看到的信息。 有关外部供应商在供应商协作界面上可执行操作的详细信息，请参阅[供应商与客户协作](vendor-collaboration-work-customers-dynamics-365-operations.md)。  

有关供应商在开票流程中如何使用供应商协作的详细信息，请参阅[供应商协作开票工作区](/dynamics365/operations/financials/accounts-payable/vendor-portal-invoicing-workspace)。 有关如何提供新供应商协作用户的信息，请参阅 [管理供应商协作用户](manage-vendor-collaboration-users.md)。

## <a name="define-the-information-shown-to-vendors-when-they-respond-to-pos"></a>定义向响应采购订单的供应商显示的信息。
供应商响应您发送给他们的采购订单时，他们将看到一个对话框，需确认他们想要接受、拒绝或更改后接受采购订单。 此时需向供应商显示的信息可能具体针对您的业务，因此您可以指定三条确认消息中分别要显示的文本。 例如，文本可以向供应商介绍流程后续步骤或条款和条件。  

定义采购订单响应中显示的文本：

1.  打开“**响应采购订单的供应商的信息**”页面。
2.  选择其中一种响应类型。
3.  单击“**编辑**”。
4.  在“**信息消息**”框中输入您想要供应商看到的信息。

如果您需要添加多种语言的消息，则使用相应的语言代码创建单独的消息。 供应商将看到以他们使用的语言显示的消息。

## <a name="set-the-vendor-collaboration-options-for-a-specific-vendor"></a>设置特定供应商的供应商协作选项
Dynamics 365 for Operations 中的供应商协作一般设置由管理员配置。 例如，它们将确定哪些安全角色对与您协作的所有供应商可用。 还有某些设置可能对每个供应商帐户不同，您应当按照以下步骤进行设置：

-   启用供应商协作。
-   确定您是否希望供应商看到价格信息。

### <a name="enable-vendor-collaboration"></a>启用供应商协作

对外部供应商创建用户帐户前，您必须将供应商帐户配置为允许他们使用供应商协作。 为此，将在“**供应商**”页面上的“**常规**”选项卡上将“**协作激活**”字段设置为活动。 有两个选项供您选择：

-   **活动（自动确认采购订单）**- 供应商在不修改采购订单的情况下接受采购订单时，自动确认采购订单。
-   **活动（不自动确认采购订单）**- 供应商接受采购订单后，需由您的组织手动确认采购订单。

### <a name="decide-whether-you-want-the-vendor-to-see-price-information"></a>确定您是否希望供应商看到价格信息

如果您要通过协作界面共享单价、折扣和费用等价格信息，需将供应商帐户上的“**采购订单价格/金额**”选项设置为“**是**”。 此选项在“**采购订单默认**”选项卡上可用。

## <a name="work-with-pos-when-using-vendor-collaboration"></a>使用供应商协作处理采购订单
### <a name="sending-a-po-to-the-vendor"></a>向供应商发送采购订单

在 Dynamics 365 for Operations 中编制采购订单。 采购订单的状态为**已批准**时，使用**采购订单**页面上的 **发送以供确认** 操作将采购订单发送给供应商。 采购订单状态更改为“**正在进行外部审核**”。 发送采购订单后，供应商可以在供应商协作界面上的“**要审核的采购订单**”页面上看到订单，并且可以接受、拒绝或建议修改订单。 供应商还可以添加注释来沟通信息，如对 PO 的更改。 如果您希望供应商注意新的 PO，也可以使用打印管理系统通过电子邮件发送 PO。

### <a name="confirmation-and-acceptance-of-the-po-by-the-vendor"></a>供应商确认和接受采购订单

供应商接受采购订单后，采购订单可能会自动确认，也可能需要手动确认。 这取决于 **供应商激活** 字段是否对供应商设置为**活动（自动确认采购订单）**或**活动（不自动确认采购订单）**。

您可以使用**采购订单****准备**工作区监控供应商已经响应哪些采购订单。 此工作区包含两个含有状态为**正在进行外部审核**的采购订单的列表：

-   正在进行外部审核需要采取操作。
-   处于等待供应商响应的外部审核中。

### <a name="changing-a-po"></a>更改 PO

如果您需要更改已响应的采购订单，您将向供应商发送采购订单的新版本。 新采购订单还将具有版本后缀，表示是先前传递的采购订单的修订版本。 **采购订单供应商确认历史记录**页允许您和您的供应商跟踪每个订单的历史记录。 以前确认的采购订单版本将保留在已确认的采购订单列表中，直到新的采购订单确认。

### <a name="cancelling-a-po"></a>取消采购订单

取消采购订单后，状态更改为“**已批准**”。 您必须通过供应商门户将 PO 发送回供应商处，以便供应商可以确认或拒绝取消。 确认取消后，采购订单在供应商的已确认采购订单列表中显示为“**已取消**”。

### <a name="adding-attachments-to-a-po"></a>添加附加到采购订单

您可以使用文档管理系统将文件、图片和附注等添加到采购订单中。 您向供应商发送采购订单时，供应商将能够看到添加为限制类型“**外部**”的附件。

## <a name="purchase-order-statuses-and-versions"></a>采购订单状态和版本
本节介绍了采购订单在被确认之间可以拥有的各种状态，以及采购订单的新版本将在什么时候对供应商可用。 这里存在差异，具体取决于您是否对采购订单使用变更管理。 

### <a name="versions-and-statuses-if-you-dont-use-change-management"></a>不使用变更管理时的版本和状态

下表展示了一个 PO 可能会经过的状态和版本更改示例。

|                                                                          |                                                                                                                                                              |
|--------------------------------------------------------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **行动**                                                               | **状态和版本**                                                                                                                                       |
| PO 的初始版本是在 Dynamics 365 for Operations 中创建的。 | 状态为**已审核**。                                                                                                                                  |
| PO 已发送至供应商。                                            | 供应商协作界面上登记了一个版本，状态更改为**正在进行外部审核**。                                          |
| 供应商发送“**已接受更改**”响应。                  | 状态仍然为“**正在进行外部审核**”。                                                                                                                  |
| 您按供应商要求进行了一些更改。                  | 状态更改为**已批准**。                                                                                                                        |
| 您将 PO 的新版本发送给供应商。                        | 供应商协作界面上登记了一个新版本，状态更改为**正在进行外部审核**。                                      |
| 供应商接受了 PO 的新版本。                            | 状态仍然为“**正在进行外部审核**”，直到供应商帐户被配置为供应商接受 PO 时自动将 PO 设置为“**已确认**”。 |

供应商无需通过供应商协作界面确认 PO。 它们还可通过其他渠道发送电子邮件或传达对 PO 的接受。 然后您可以在 Dynamics 365 for Operations 中手动确认该订单。 在这种情况下，您将收到该订单正在确认的警告，即使没有供应商响应。 之后，PO 将在确认历史记录中显示为没有任何响应的未结已确认订单。 供应商将不能再选择确认或拒绝 PO。  

**注意：**对 Dynamics 365 for Operations 中的其他流程可用的 PO 版本始终是最新版本，即使该版本尚未在供应商协作界面中登记。

### <a name="versions-and-statuses-if-you-use-change-management"></a>使用变更管理时的版本和状态

如果您为 PO 启用了变更管理，PO 将通过审核工作流，然后到达**已审核**状态。 此流程对供应商不可见。  

下表展示了启用更改管理时一个 PO 可能会经过的状态和版本更改示例。 版本在 PO 通过审核时登记，而不是在将 PO 发送给供应商或已确认时。

|                                                                                                               |                                                                                                                                                                                                                                                                                                                                                                                             |
|---------------------------------------------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **行动**                                                                                                    | **状态和版本**                                                                                                                                                                                                                                                                                                                                                                      |
| PO 的初始版本是在 Dynamics 365 for Operations 中创建的。                                      | 状态为**草稿**。                                                                                                                                                                                                                                                                                                                                                                    |
| 将 PO 提交到审核流程。 （这是一个不涉及供应商的内部流程。） | 如果在审核流程中拒绝 PO，状态从“**草稿**”更改为“**正在审核**”，然后更改为“**批准**”。 已审核的 PO 将注册为一个版本。                                                                                                                                                                                                                     |
| PO 已发送至供应商                                                                                  | 供应商协作界面上登记了一个版本，状态更改为**正在进行外部审核**。                                                                                                                                                                                                                                                                       |
| 您按供应商要求进行了一些更改。                                                       | 状态更改回**草稿**。                                                                                                                                                                                                                                                                                                                                                    |
| 将 PO 再次提交到审核流程。                                                            | 如果在审核流程中拒绝 PO，状态从“**草稿**”更改为“**正在审核**”，然后更改为“**批准**”。 或者，也可以配置系统，这样特定字段更改时就不需要重新审核。 在这种情况下，状态将首先更改为**草稿**，然后将自动更新为**已审核**。 已审核的 PO 将注册为一个新版本。 |
| 您将 PO 的新版本发送给供应商。                                                             | 供应商协作界面上登记了一个新版本，状态更改为**正在进行外部审核**。                                                                                                                                                                                                                                                                   |
| 供应商批准了 PO 的新版本。                                                                | 状态将自动更改为**已确认**，或者在您收到供应商响应并确认 PO 时更改为“已确认”。                                                                                                                                                                                                                                                     |

## <a name="share-information-about-consignment-inventory"></a>共享关于托运库存的信息
如果正在使用托运库存，供应商可以使用供应商协作界面查看有关以下页面的信息：

-   **使用托运库存的采购订单** - 库存的所有权从供应商更改为您的公司时，生成托运库存的采购订单。 同时对产品收据过帐。 这些托运采购订单仅显示在“**使用托运库存的采购订单**”页面上。 它们包含在“**供应商协作**”模块的“**所有已确认的采购订单**”页面中。
-   **从托运库存接收的产品** - 此页列出产品所有权从供应商转移至您的公司的所有交易记录。 供应商可以使用此信息对客户开票。
-   **现有托运库存** - 此页显示了您的仓库已经收到的供应商拥有的现有托运库存。

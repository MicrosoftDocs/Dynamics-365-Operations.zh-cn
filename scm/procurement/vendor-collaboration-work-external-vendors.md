# <a name="vendor-collaboration-with-external-vendors"></a>供应商与外部供应商协作

[!include[banner](../includes/banner.md)]


本主题说明采购代理如何与外部供应商协作交换关于采购订单和托运库存的信息。

**供应商协作**模块针对的是没有与 Microsoft Dynamics 365 for Finance and Operations 进行电子数据交换 (EDI) 集成的供应商。 它允许供应商使用采购订单、发票和托运库存信息。 本主题介绍您可以如何与使用供应商协作界面的外部供应商协作处理采购订单和托运库存。 还介绍了如何允许特定供应商使用供应商协作，以及如何定义所有供应商在响应采购订单时将看到的信息。 有关外部供应商在供应商协作界面上可执行操作的详细信息，请参阅[供应商与客户协作](vendor-collaboration-work-customers-dynamics-365-operations.md)。  

有关供应商在开票流程中如何使用供应商协作的详细信息，请参阅[供应商协作开票工作区](/dynamics365/unified-operations/financials/accounts-payable/vendor-portal-invoicing-workspace)。 有关如何提供新供应商协作用户的信息，请参阅 [管理供应商协作用户](manage-vendor-collaboration-users.md)。

有关供应商在开票流程中如何使用供应商协作的详细信息，请参阅[供应商协作开票工作区](/dynamics365/operations/financials/accounts-payable/vendor-portal-invoicing-workspace)。 

有关如何提供新供应商协作用户的信息，请参阅 [管理供应商协作用户](manage-vendor-collaboration-users.md)。

## <a name="define-the-information-that-is-shown-to-vendors-when-they-respond-to-pos"></a>定义向响应采购订单的供应商显示的信息
供应商响应您发送给他们的采购订单时，他们将看到一个消息框，须确认他们想要接受、拒绝或更改后接受采购订单。 由于此时须向供应商显示的信息可能具体针对您的业务，因此您可以指定三条确认消息中分别要显示的文本。 例如，文本可以向供应商介绍流程后续步骤或条款和条件。  

定义采购订单响应中显示的文本：

1.  打开“**响应采购订单的供应商的信息**”页面。
2.  选择其中一种响应类型。
3.  单击“编辑”****。
4.  在“**信息消息**”框中输入您想要供应商看到的信息。

如果您必须添加多种语言的消息，则创建单独的消息并分别指定相应的语言代码。 向供应商显示的消息将采用供应商使用的语言。

## <a name="set-the-vendor-collaboration-options-for-a-specific-vendor"></a>设置特定供应商的供应商协作选项
Finance and Operations 中的供应商协作一般设置由管理员配置。 例如，管理员将确定哪些安全角色对与您协作的所有供应商可用。 还有某些设置可能对每个供应商帐户不同，您应当按照以下步骤进行设置：
-   启用供应商协作。
-   确定您是否希望供应商看到价格信息。

### <a name="enable-vendor-collaboration"></a>启用供应商协作

对外部供应商创建用户帐户前，您必须将供应商帐户配置为允许他们使用供应商协作。 为此，将在“**供应商**”页面上的“**常规**”选项卡上将“**协作激活**”字段设置为活动。 有两个选项供您选择：

-   **活动（自动确认采购订单）**- 供应商在不修改采购订单的情况下接受采购订单时，自动确认采购订单。
-   **活动（不自动确认采购订单）**- 供应商接受采购订单后，需由您的组织手动确认采购订单。

### <a name="decide-whether-you-want-the-vendor-to-see-price-information"></a>确定您是否希望供应商看到价格信息

如果您要通过协作界面共享单价、折扣和费用等价格信息，需将供应商帐户上的“**采购订单价格/金额**”选项设置为“**是**”。 此选项在“**采购订单默认**”选项卡上可用。

## <a name="work-with-pos-when-using-vendor-collaboration"></a>使用供应商协作处理采购订单
### <a name="sending-a-po-to-the-vendor"></a>向供应商发送采购订单

在 Finance and Operations 中编制采购订单。 采购订单的状态为“**已批准**”时，使用“**采购订单**”页面上的“**发送以供确认**”操作将采购订单发送给供应商。 采购订单状态更改为“**正在进行外部审核**”。 发送采购订单后，供应商可以在供应商协作界面中的**要审核的采购订单**上看到。 供应商可以接受、拒绝或建议更改订单。 供应商还可以添加注释来沟通信息，如对 PO 的更改。 如果您希望供应商注意新的 PO，也可以使用打印管理系统通过电子邮件发送 PO。

### <a name="confirmation-and-acceptance-of-the-po-by-the-vendor"></a>供应商确认和接受采购订单

供应商接受采购订单后，采购订单可能会自动确认，也可能需要手动确认。 这取决于**供应商激活**字段是否对供应商设置为**活动（自动确认采购订单）**或**活动（不自动确认采购订单）**。

您可以使用**采购订单****准备**工作区监控供应商已经响应哪些采购订单。 此工作区包含两个含有状态为**正在进行外部审核**的采购订单的列表：

-   正在进行外部审核需要采取操作。
-   处于等待供应商响应的外部审核中。

### <a name="changing-a-po"></a>更改 PO

要更改已响应的采购订单，必须向供应商发送采购订单的新版本。 新采购订单还将具有版本后缀，表示是先前传递的采购订单的修订版本。 **采购订单供应商确认历史记录**页允许您和您的供应商跟踪每个订单的历史记录。 以前确认的采购订单版本将保留在已确认的采购订单列表中，直到新的采购订单确认。

### <a name="cancelling-a-po"></a>取消采购订单

取消采购订单后，状态更改为“**已批准**”。 您必须通过供应商门户将 PO 发送回供应商处，以便供应商可以确认或拒绝取消。 确认取消后，采购订单在供应商的已确认采购订单列表中显示为“**已取消**”。

### <a name="adding-attachments-to-a-po"></a>添加附加到采购订单

您可以使用文档管理系统将文件、图片和附注等附件添加到采购订单中。 您发送采购订单时，供应商将能够看到类型为**外部**的附件。

## <a name="update-the-po-when-a-vendor-suggests-changes"></a>当供应商建议更改时更新采购订单
当供应商已响应采购订单且建议更改时，下一步是处理响应。
在**采购订单准备工作区**，正在进行外部审核要求具有操作列表，您可以将供应商已响应的采购订单标识为更改后接受。 正在进行外部审核要求具有操作列表，您还可以导航到供应商的响应。 在响应上，供应商可以更改标题上的以下信息。
 
-   供应商文档引用
-   交货方式
-   交货条款
-   确认的交货日期

供应商还可以添加附注或附件

在行上，供应商可以更改数量和交货日期、添加注释和附件、拒绝行、使用键入为文本的其他产品替代行以及将行拆分为多个交货。 根据供应商建议的更改，行状态将具有不同的行状态：
    
-   **已接受更改**
-   **已拒绝**
-   **已替代**在此情况下将添加状态为**替代**的额外行。
-   **已确认**拆分为计划。在此情况下将添加状态为**计划行**的额外行。

如果行没有更改，行状态为**已接受**。

在响应上，您可以看到之前提及的指示供应商做出的更改类型的行状态。 此外，所有更改的字段显示为粗体以帮助您识别更改。

您可以通过在响应上或一次在一行上单击**处理采购订单更新**操作的方式更新采购订单。 标题和行上的指示符**是否已处理采购订单更新？**让您能够看到系统是否已处理标题或行以使用源自响应的任何可能更改更新采购订单。 每个标题或行仅可以运行一次**处理采购订单更新**流程。

并非所有建议的更改都可以在采购订单上进行更新。 仅标题上的更新以及行上的日期和数量更新可以自动在采购订单上进行更新。 对于其他更改，您必须手动更新采购订单。 在这种情况下，**是否已处理采购订单更新？**指示符显示**手动更新**。 必须手动处理的一个更改示例可能是当供应商建议将行拆分到计划时。

状态为**已接受**的行将具有已确认的交货日期，该日期将在您执行**处理采购订单更新**时在采购订单上进行更新。 注释和附件不自动转移到当前采购订单。 请注意，如果您通过**处理采购订单更新**操作更新当前采购订单，将不会在采购订单行上重新评估贸易协议。

## <a name="share-information-about-consignment-inventory"></a>共享关于托运库存的信息
如果正在使用托运库存，供应商可以使用供应商协作界面查看有关以下页面的信息：

-   **使用托运库存的采购订单** - 库存的所有权从供应商更改为您的公司时，生成托运库存的采购订单。 同时对产品收据过帐。 这些托运采购订单仅显示在“**使用托运库存的采购订单**”页面上。 它们包含在“**供应商协作**”模块的“**所有已确认的采购订单**”页面中。
-   **从托运库存接收的产品** - 此页列出产品所有权从供应商转移至您的公司的所有交易记录。 供应商可以使用此信息对客户开票。
-   **现有托运库存** - 此页显示了您的仓库已经收到的供应商拥有的现有托运库存。

# <a name="configure-settlement"></a><span data-ttu-id="eadfb-104">配置结算</span><span class="sxs-lookup"><span data-stu-id="eadfb-104">Configure settlement</span></span>

[!include[banner](../includes/banner.md)]


<span data-ttu-id="eadfb-105">交易记录的结算方式和时间可能非常复杂，因此，您理解并正确定义参数以满足您的业务需求非常重要。</span><span class="sxs-lookup"><span data-stu-id="eadfb-105">How and when transactions are settled can be complex subjects, so it's essential that you understand and correctly define the parameters to meet your business requirements.</span></span> <span data-ttu-id="eadfb-106">本文介绍用于应付账款和应收账款的结算的参数。</span><span class="sxs-lookup"><span data-stu-id="eadfb-106">This article describes the parameters that are used for settlement for both Accounts payable and Accounts receivable.</span></span> 

<span data-ttu-id="eadfb-107">以下参数将影响如何在 Microsoft Dynamics 365 for Finance and Operations Enterprise Edition 中处理结算。</span><span class="sxs-lookup"><span data-stu-id="eadfb-107">The following parameters affect how settlements are processed in Microsoft Dynamics 365 for Finance and Operations, Enterprise edition.</span></span> <span data-ttu-id="eadfb-108">结算是对照付款或贷方通知单结算发票的流程。</span><span class="sxs-lookup"><span data-stu-id="eadfb-108">Settlement is the process of settling an invoice against a payment or credit note.</span></span> <span data-ttu-id="eadfb-109">这些参数位于**应收账款参数**和**应付账款参数**页的**结算**区域。</span><span class="sxs-lookup"><span data-stu-id="eadfb-109">These parameters are located in the **Settlement** area of the **Accounts receivable parameters** and **Accounts payable parameters** pages.</span></span>

-   <span data-ttu-id="eadfb-110">**自动结算** – 如果交易记录在过帐时应自动对照其他未结交易记录结算，则设置此选项为**是**。</span><span class="sxs-lookup"><span data-stu-id="eadfb-110">**Automatic settlement** – Set this option to **Yes** if a transaction should be settled automatically against other open transactions when it is posted.</span></span> <span data-ttu-id="eadfb-111">如果此选项设置为**否**，用户可以使用**结算交易记录**页在输入付款或之后手动结算交易记录。</span><span class="sxs-lookup"><span data-stu-id="eadfb-111">If this option is set to **No**, users can manually settle transactions when they enter payments, or later, by using the **Settle transactions** page.</span></span>

    -   <span data-ttu-id="eadfb-123">如果超额支付或支付不足产生的尾差大于**最大尾差**字段中定义的差额，该差异金额将过帐到为**自动交易记录帐户**页的**客户现金折扣**或**供应商现金折扣**选择的差额科目。</span><span class="sxs-lookup"><span data-stu-id="eadfb-123">If the overpayment or underpayment produces a difference that is more than the difference that is defined in the **Maximum penny difference** field, the difference amount is posted to the difference account that is selected for the **Customer cash discount** or **Vendor cash discount** posting type on the **Accounts for automatic transactions** page.</span></span>

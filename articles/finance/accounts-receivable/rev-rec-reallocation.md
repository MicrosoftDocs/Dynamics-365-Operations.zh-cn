# <a name="revenue-recognition-reallocation"></a><span data-ttu-id="4ff12-104">收入确认重新分配</span><span class="sxs-lookup"><span data-stu-id="4ff12-104">Revenue recognition reallocation</span></span>

[!include [banner](../includes/banner.md)]

<span data-ttu-id="4ff12-105">组织可以通过重新分配来在更改合同销售条款时重新计算收入价格。</span><span class="sxs-lookup"><span data-stu-id="4ff12-105">Reallocation enables organizations to recalculate revenue prices when the terms of a contractual sale are changed.</span></span> <span data-ttu-id="4ff12-106">出于收入确认目的，销售订单文档将视为合同。</span><span class="sxs-lookup"><span data-stu-id="4ff12-106">For the purpose of revenue recognition, the sales order documents are considered the contract.</span></span>

<span data-ttu-id="4ff12-107">您的组织必须自行确定是否需要重新分配。</span><span class="sxs-lookup"><span data-stu-id="4ff12-107">Your organization must determine for itself whether reallocation is required.</span></span> <span data-ttu-id="4ff12-108">对销售订单添加新行或为客户添加新销售订单可能不会视为对合同的更改。</span><span class="sxs-lookup"><span data-stu-id="4ff12-108">The addition of a new line to a sales order, or the addition of a new sales order for a customer, might not constitute a change to the contract.</span></span> <span data-ttu-id="4ff12-109">以下情况可能需要重新分配：</span><span class="sxs-lookup"><span data-stu-id="4ff12-109">The following scenarios might require reallocation:</span></span>

- <span data-ttu-id="4ff12-110">在对销售订单全部或部分开票后，客户向销售订单中添加了项目，或者从销售订单中删除了项目。</span><span class="sxs-lookup"><span data-stu-id="4ff12-110">A customer added items to a sales order, or removed items from the sales order, after the order was fully or partially invoiced.</span></span>
- <span data-ttu-id="4ff12-111">对于同一份协商合同，输入了处于确认状态或已开票状态的多个销售订单。</span><span class="sxs-lookup"><span data-stu-id="4ff12-111">Multiple sales orders, in either a confirmed state or an invoiced state, were entered for the same negotiated contract.</span></span>
- <span data-ttu-id="4ff12-112">在对原始销售订单全部或部分开票后，客户返回或收到项目的贷项。</span><span class="sxs-lookup"><span data-stu-id="4ff12-112">A customer returned or received a credit for an item after the original sales order was fully or partially invoiced.</span></span>

<span data-ttu-id="4ff12-113">重新分配流程存在一些重要限制：</span><span class="sxs-lookup"><span data-stu-id="4ff12-113">There are a few important limitations on the reallocation process:</span></span>

- <span data-ttu-id="4ff12-114">该流程只能运行一次。</span><span class="sxs-lookup"><span data-stu-id="4ff12-114">The process can be run only one time.</span></span> <span data-ttu-id="4ff12-115">因此，请务必注意，只有在完成所有更改后才能运行该流程。</span><span class="sxs-lookup"><span data-stu-id="4ff12-115">Therefore, it's important that you run it only after all changes are finalized.</span></span>
- <span data-ttu-id="4ff12-116">不能对项目销售订单运行该流程。</span><span class="sxs-lookup"><span data-stu-id="4ff12-116">The process can't be run on project sales orders.</span></span>
- <span data-ttu-id="4ff12-117">如果涉及多个销售订单，这些订单必须属于同一个客户帐户。</span><span class="sxs-lookup"><span data-stu-id="4ff12-117">If multiple sales orders are involved, they must be for the same customer account.</span></span>

- <span data-ttu-id="4ff12-119">该流程在运行后无法冲销或撤消。</span><span class="sxs-lookup"><span data-stu-id="4ff12-119">The process can't be reversed or undone after it's run.</span></span>

## <a name="set-up-reallocation"></a><span data-ttu-id="4ff12-120">设置重新分配</span><span class="sxs-lookup"><span data-stu-id="4ff12-120">Set up reallocation</span></span>

<span data-ttu-id="4ff12-121">此参数影响重新分配流程。</span><span class="sxs-lookup"><span data-stu-id="4ff12-121">One parameter affects the reallocation process.</span></span>

<span data-ttu-id="4ff12-122">由于可以对部分或全部开票的销售订单执行重新分配，因此必须使用新的、重新分配的收入价格来更正账单的任何先前会计条目。</span><span class="sxs-lookup"><span data-stu-id="4ff12-122">Because reallocation can be done on a sales order that is partially or fully invoiced, any previous accounting entries for the invoice must be corrected by using the new, reallocated revenue prices.</span></span> <span data-ttu-id="4ff12-123">执行此更正的方法是：冲销原始账单的会计条目，并过帐基于重新分配的收入价格的新会计条目。</span><span class="sxs-lookup"><span data-stu-id="4ff12-123">This correction is done by reversing the original invoice's accounting entry and posting a new accounting entry that is based on the reallocated revenue prices.</span></span>

<span data-ttu-id="4ff12-124">每个组织都必须决定更正应仅更新总帐，还是应更新应收帐款。</span><span class="sxs-lookup"><span data-stu-id="4ff12-124">Every organization must decide whether the correction should update only General ledger, or whether it should also update Accounts receivable.</span></span> <span data-ttu-id="4ff12-125">达成的决定确定 **总帐参数** 页面的 **收入确认** 选项卡上 **将账单更正过帐到应收帐款** 选项的合适设置（**收入确认 \> 设置 \> 总帐参数**）。</span><span class="sxs-lookup"><span data-stu-id="4ff12-125">The decision that is reached determines the appropriate setting of the **Post invoice corrections to Accounts receivable** option on the **Revenue recognition** tab of the **General ledger parameters** page (**Revenue recognition \> Setup \> General ledger parameters**).</span></span> <span data-ttu-id="4ff12-126">合适的设置取决于特定的方案。</span><span class="sxs-lookup"><span data-stu-id="4ff12-126">The appropriate setting depends on the specific scenario.</span></span> <span data-ttu-id="4ff12-127">有关可能的方案的详细信息，请使用本主题后面的[重新分配方案](#scenarios-for-reallocation)部分中的链接。</span><span class="sxs-lookup"><span data-stu-id="4ff12-127">For more information about possible scenarios, use the links in the [Scenarios for reallocation](#scenarios-for-reallocation) section later in this topic.</span></span>

<span data-ttu-id="4ff12-128">[![“总帐参数”页面上的“收入确认”选项卡](./media/01_RevRecScenarios.png)](./media/01_RevRecScenarios.png)</span><span class="sxs-lookup"><span data-stu-id="4ff12-128">[![Revenue recognition tab on the General ledger parameters page](./media/01_RevRecScenarios.png)](./media/01_RevRecScenarios.png)</span></span>

    - <span data-ttu-id="4ff12-131">贷方凭证将重用原始账单编号，但会在其后追加“-1”。</span><span class="sxs-lookup"><span data-stu-id="4ff12-131">The credit document reuses the original invoice number, but "-1" is appended to it.</span></span>

<span data-ttu-id="4ff12-148">如前所述，您可以仅更新总帐，也可以同时更新总帐和应收帐款。</span><span class="sxs-lookup"><span data-stu-id="4ff12-148">As has been mentioned, you can update only General ledger, or you can update both General ledger and Accounts receivable.</span></span> <span data-ttu-id="4ff12-149">这两种方法各有利弊。</span><span class="sxs-lookup"><span data-stu-id="4ff12-149">Both approaches have pros and cons.</span></span> <span data-ttu-id="4ff12-150">我们建议您评估组织的要求，以确定应该使用哪个选项。</span><span class="sxs-lookup"><span data-stu-id="4ff12-150">We recommend that you evaluate your organization's requirements to determine which option you should use.</span></span> <span data-ttu-id="4ff12-151">如果同时更新总帐和应收帐款，则正确的会计条目将显示在新账单上，并且可以从 **客户交易记录** 页面上的文档中查看。</span><span class="sxs-lookup"><span data-stu-id="4ff12-151">If you update both General ledger and Accounts receivable, the correct accounting entries will be shown on the new invoice and can be viewed from the document on the **Customer transactions** page.</span></span> <span data-ttu-id="4ff12-152">此外，结算流程将使用更新的会计条目以过帐任何现金折扣和损益。</span><span class="sxs-lookup"><span data-stu-id="4ff12-152">Additionally, the settlement process will use the updated accounting entries to post any cash discounts and gains or losses.</span></span> <span data-ttu-id="4ff12-153">另一方面，就像其他贷方凭证和客户账单一样，贷方凭证和新账单将显示在客户对帐单和帐龄报告中。</span><span class="sxs-lookup"><span data-stu-id="4ff12-153">On the other hand, the credit document and the new invoice will appear on customer statements and aging reports, just as other credit documents and customer invoices do.</span></span> <span data-ttu-id="4ff12-154">这些文档的说明将指示它们是通过会计更正而创建的。</span><span class="sxs-lookup"><span data-stu-id="4ff12-154">The description of those documents will indicate that they were created through an accounting correction.</span></span>

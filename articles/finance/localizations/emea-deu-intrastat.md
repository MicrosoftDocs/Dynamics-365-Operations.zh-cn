# <a name="german-intrastat"></a>德国内部统计

[!include [banner](../includes/banner.md)]

**内部统计** 页用于生成和报告有关欧盟 (EU) 国家/地区间的贸易信息。 德国内部统计申报包含有关要申报的货物贸易的信息。 此报表采用的格式依据德国主管机构制定的准则，[6.2 INSTAT/XML 格式的文件声明](https://www-idev.destatis.de/idev/doc/intra_en/hilfe6_2.html)页面提供了这些准则。

下表显示了德国内部统计申报中所包括的字段。

| 字段 | 发货 | 到达 |
|-------------------------|-------------------------|-------------------------|
| 参考期间 | X | X |
| 方向代码 | X | X |
| 内部统计申报的货币</br>**注意：** 此货币是<em>记帐币种</em>。 | X | X |
| 商品代码 | X | X |
| 商品名称 | X | X |
| 附属单位代码 | X | X |
| 托运/目的地的成员国 | X | X |
| 净批量 | X | X |
| 补充单位数量 | X | X |
| 原产国家/地区 |  | X |
| 发票金额 | X |  |
| 统计值 | X | X |
| 交易记录的性质 | X | X |
| 运输模式 | X | X |
| 区域代码</br>**注释:**</br><ul></br><li>如果原产国家或地区是德国，并且发票用于发货，则将显示原产国家的内部统计代码。</li></br><li>如果原产国家或地区不是德国，并且发票用于发货，则将显示代码 **99**。</li></br><li>如果发票用于到货，则将显示国家或地区的内部统计代码。</li></br></ul> | X | X |
| 港口/机场/内陆港口代码 | X | X |
| 交货条款 | X | X |


## <a name="set-up-intrastat"></a>设置内部统计

1. 设置公司的代码。

    1. 转至 **组织管理** > **组织** > **法人**。
    2. 在 **外贸和物流** 快速选项卡上 **标识** 部分内的 **DVR** 字段中，输入交换协议 ID。 此 ID 将包含在报表中。
    3. 在 **出口增值税免税编号** 字段中，输入贵公司的出口增值税 (VAT) 编号。
    4. 在 **进口增值税免税编号** 字段中，输入贵公司的进口增值税编号。
    5. 在 **内部统计代码** 字段中，输入分配给法人的内部统计代码。
    6. 在 **税务登记** 快速选项卡上，在 **税务登记编号** 字段中输入您公司的税务登记编号。

    > [!NOTE]
    > 如果您为附属公司生成内部统计报表，请在 **税务登记编号** 字段中输入上级公司的税务登记编号。

2. 在 [Microsoft Dynamics Lifecycle Services (LCS)](https://lcs.dynamics.com/Logon/Index) 中的共享资产库中，下载以下用于内部统计申报的电子报告 (ER) 配置的最新版本。

    - 内部统计模型
    - 内部统计报表
    - 内部统计 XML
    - 内部统计 XML（德国）

3. 设置外贸参数：

    1. 在 Dynamics 365 Finance 中，转到 **税务** > **设置** > **外贸参数**。
    2. 在 **内部统计** 选项卡的 **电子报告** 快速选项卡上，在 **文件格式映射** 字段中，选择 **内部统计 XML (DE)**。
    3. 在 **报表格式映射** 字段中，选择 **内部统计报表**。
    4. 在 **商品代码层次结构** 快速选项卡上，在 **类别层次结构** 字段中，选择 **内部统计**。
    5. 在 **交易代码** 字段中，选择财产转移的交易代码。 您将此代码用于导致实际或计划转移财产以进行补偿（财务或其他）的交易。 您还使用它进行更正。
    6. 在 **贷方通知单** 字段中，选择退货的交易代码。
    7. 在 **工作人员** 字段中，选择内部统计报表的联系人。 或者，在 **联系人** 选项卡上，在 **姓名**、**电话**、**传真**、**电子邮件** 和 **Internet 地址** 字段中输入或选择值。 报表中包含这些字段。
    8. 在 **主管机构** 字段中，选择内部统计主管机构。
    9. 转到 **税** > **间接税** > **销售税** > **销售税主管机构**，并输入您在上一步中选择的内部统计主管机构的以下信息：

       - 主管机构标识
       - 地址
       - 联系人信息

    10. 在 **国家/地区属性** 选项卡上的 **国家/地区** 字段中，列出与您的公司有业务往来的所有国家或地区。 对于每个国家或地区，在 **国家/地区类型** 字段中，选择 **欧盟**，以便国家或地区显示在内部统计报表中。

4. 设置区域代码。

    1. 转至 **组织管理** > **全球通讯簿** > **地址** > **地址设置**。
    2. 在 **省/市/自治区** 选项卡上的 **国家/区域** 字段中，选择 **DEU**，然后选择 **应用筛选器**。
    3. 在网格中，选择省/市/自治区。 然后，在 **内部统计代码** 字段中输入唯一的内部统计代码。

5. 设置产品的原产地的地址。

    1. 转到 **产品信息管理** > **产品** > **已发布产品**。
    2. 在网格中选择产品。
    3. 在 **外贸** 快速选项卡上的 **内部统计** 部分，在 **原产国家/地区** 字段中，选择 **DEU**。

6. 转到 **税务** > **设置** > **外贸** > **内部统计缩减**，选择汇总内部统计信息时应比较的字段。 对于德国内部统计，请选择以下字段：

    - 商品
    - 国家/地区
    - 冲销
    - 方向
    - 交货条款
    - 发票
    - 原产国家/地区
    - 港口
    - 发送方所在的国家/地区
    - 发送方所在的省/自治区/直辖市
    - 统计过程
    - 交易记录代码
    - 传输
    - 免税编号

### <a name="intrastat-transfer"></a>内部统计转移

在 **内部统计** 页面的操作窗格中，您可以选择 **转移** 从您的销售订单、普通发票、采购订单、供应商发票、供应商产品收据 **、** 项目发票和转移单自动转移有关集团内贸易的信息。 只有将欧盟国家/地区作为目的地（发货）或托运（到货）国家或地区的单据会被转移。

或者，您可以通过在操作窗格中选择 **新建** 手动输入交易。

### <a name="generate-an-intrastat-report"></a>生成内部统计报表

1. 转到 **纳税** > **申报** > **外贸** > **内部统计**。
2. 在操作窗格上，选择 **输出** > **报表**。
3. 在 **内部统计报表** 对话框中，设置以下字段。

    | 字段 | 说明 |
    |-------------------------|-------------------------|
    | 开始日期 | 选择报表的开始日期。 |
    | 截止日期 | 选择报表的结束日期。 |
    | 生成文件 | 将此选项设置为 **是** 可以生成 .xml 文件。 |
    | 文件名 | 保持此字段为空。 文件名是自动生成的，由 **&lt;envelopeId &gt;** XML 标记的值加上文件名扩展名 **.xml** 构成。</br>**&lt;envelopeId&gt;** 值由以下元素按以下顺序级联而成：</br><ol type="1"></br><li>**&lt;interchangeAgreementId&gt;** 标记的值</li></br><li>连字符 (-)</li></br><li>**&lt;YYYYMM 格式的 referencePeriod&gt;** 标记的值</li></br><li>连字符 (-)</li></br><li>**&lt;YYYYMMDD 格式的信封/日期/时间/日期&gt;** 标记的值 </li></br><li>连字符 (-)</li></br><li>**&lt;hhmm 格式的信封/日期/时间/时间&gt;** 标记的值 </li></br></ol> |
    | 方向 | <ul></br><li>选择 **到货** 以报告集团内部到货。</li></br><li>选择 **发货** 以报告集团内部发货。</li></br><li>选择 **两者** 以同时报告到货和发货。</li></br></ul> |
    | 税务登记编号 | 如果要用于生成发件人识别码的注册号与法人的税务注册号不同，请输入要用于生成发件人识别码的注册号。 |


## <a name="example"></a>示例

此示例显示了如何过帐内部统计的达货和发货。 使用 **DEMF** 法人。

1. 在 [LCS](https://lcs.dynamics.com/Logon/Index) 中的共享资产库中，下载以下用于内部统计申报格式的 ER 配置的最新版本：

    - 内部统计模型
    - 内部统计报表
    - 内部统计 XML
    - 内部统计 XML（德国）

2. 创建交易代码。

    1. 转到 **税务** > **设置** > **外贸** > **交易代码**。
    2. 在操作窗格上，选择 **新建**。
    3. 在 **交易代码** 字段中，输入 **21**。
    4. 在 **名称** 字段中，输入 **退货**。
    5. 在操作窗格上，选择 **保存**。

3. 设置主管机构标识号。

    1. 转到 **纳税** > **间接税** > **销售税** > **销售税主管机构**。
    2. 在列表中，选择 **TA**。
    3. 在 **主管机构标识** 字段中，输入 **123**。

4. 设置区域代码。

    1. 转至 **组织管理** > **全球通讯簿** > **地址** > **地址设置**。
    2. 在 **省/市/自治区** 选项卡上的 **国家/区域** 字段中，选择 **DEU**，然后选择 **应用筛选器**。
    3. 在网格中，选择 **BE**，然后在 **内部统计代码** 字段中输入 **11**。
    4. 在操作窗格上，选择 **保存**。

5. 设置外贸参数。

    1. 转到 **税务** > **设置** > **外贸** > **外贸参数**。
    2. 在 **内部统计** 选项卡上，在 **常规** 快速选项卡上，在 **交易代码** 字段中，选择 **11**。
    3. 在 **贷方通知单** 字段中，选择 **21**。
    4. 在 **主管机构** 字段中，选择 **TA**。
    5. 在 **电子报告** 快速选项卡上，在 **文件格式映射** 字段中，选择 **内部统计 XML（德国）**。
    6. 在 **报表格式映射** 字段中，选择 **内部统计报表**。
    7. 在 **商品代码层次结构** 快速选项卡上，在 **类别层次结构** 字段中，确保选择 **内部统计**。
    8. 在 **国家/地区属性** 选项卡上，选择 **新建**。
    9. 在 **当事方国家/地区** 字段中，选择 **FRA**。
    10. 在 **国家/地区类型** 字段中，选择 **欧盟**。

6. 将商品代码分配给产品，并设置产品的来源。 然后，当您选择产品时，将在销售订单和采购订单上自动设置 **商品代码**、**原产国** 和 **原产地** 字段。

    1. 转到 **产品信息管理** > **产品** > **已发布产品**。
    2. 在网格中，选择 **D0001**。
    3. 在 **外贸** 快速选项卡上的 **内部统计** 部分，在 **商品** 字段中，选择 **920 20 34**。
    4. 在 **来源** 部分内的 **国家/地区** 字段中，选择 **DEU**。
    5. 在 **管理库存** 快速选项卡上 **重量度量** 部分内的 **净重** 字段中，输入 **12**。
    6. 在操作窗格上，选择 **保存**。

7. 将运输分配给交货模式。 然后，当您选择交货模式时，将自动在订单上设置运输代码。

    1. 转到 **采购** > **设置** > **分配** > **交付方式**。
    2. 在列表中，选择 **10**。
    3. 在 **外贸** 快速选项卡上的 **运输** 字段中，选择 **01**。

8. 创建欧盟客户销售订单。

    1. 转到 **应收帐款** > **订单** > **所有销售订单**。
    2. 在操作窗格上，选择 **新建**。
    3. 在 **创建销售订单** 对话框的 **客户** 快速选项卡上，在 **客户** 部分的 **客户帐户** 字段中，选择 **DE-012**。
    4. 在 **常规** 快速选项卡的 **存储维度** 部分内的 **站点** 字段中，选择 **1**。
    5. 在 **仓库** 字段中，选择 **11**。
    6. 选择 **确定**。
    7. 在 **标题** 选项卡的 **外贸** 快速选项卡上的 **端口** 字段中，选择 **53**。
    8. 在 **统计过程** 字段中，选择 **31710**。
    9.  在 **选项** 选项卡的 **销售订单行** 快速选项卡上，在 **物料编号** 字段中，选择 **D0001**。 然后，在 **数量** 字段中，输入 **10**。
    10. 在 **行详细信息** 快速选项卡的 **外贸** 选项卡上，确保自动设置了 **交易代码**、**运输**、**商品** 和 **原产国家/地区** 字段。
    11. 在操作窗格上，选择 **保存**。
    12. 在操作窗格 **发票** 选项卡的 **常规** 部分中，选择 **发票**。
    13. 在 **过帐发票** 对话框的 **参数** 快速选项卡上，在 **参数** 部分的 **数量** 字段中，选择 **所有**。
    14. 选择 **确定** 以过帐发票。

9.  将交易记录转移到内部统计日记帐，并查看结果。

    1. 转到 **纳税** > **申报** > **外贸** > **内部统计**。
    2. 在操作窗格上，选择 **转帐**。
    3. 在 **内部统计(转移)** 对话框中，在 **参数** 部分，将 **客户发票** 选项设置为 **是**。
    4. 选择 **筛选器**。
    5. 在 **内部统计筛选器** 对话框中，在 **范围** 选项卡上，选择第一行，并确保 **字段** 字段设置为 **日期**。
    6. 在 **条件** 字段中，选择当前日期。
    7. 单击 **确定** 以关闭 **内部统计筛选器** 对话框。
    8. 选择 **确定** 以关闭 **内部统计(转移)** 对话框并查看结果。 此行表示您以前创建的销售订单。

        ![“内部统计”页面上代表销售订单的行](media/intrastat_deu_1.png)

10. 选择交易记录行，然后选择 **常规** 选项卡以查看更多详细信息。

    ![“内部统计”页面的“常规”选项卡上的销售订单详细信息](media/intrastat_deu_2.png)

11. 使用销售订单创建贷项通知单。

    1. 转到 **应收帐款** > **订单** > **所有销售订单**。
    2. 在操作窗格上，选择 **新建**。
    3. 在 **创建销售订单** 对话框的 **客户** 快速选项卡上，在 **客户** 部分的 **客户帐户** 字段中，选择 **DE-012**。
    4. 在 **常规** 快速选项卡的 **存储维度** 部分内的 **站点** 字段中，选择 **1**。
    5. 在 **仓库** 字段中，选择 **11**。
    6. 在 **标题** 选项卡的 **外贸** 快速选项卡上的 **端口** 字段中，选择 **53**。
    7. 在 **统计过程** 字段中，选择 **31710**。
    8. 在 **选项** 选项卡的 **销售订单行** 快速选项卡上，在 **物料编号** 字段中，选择 **D0001**。 然后，在 **数量** 字段中，输入 **-2**。
    9. 在 **行详细信息** 快速选项卡上，在 **外贸** 选项卡上，在 **交易代码** 字段中，选择 **21**。
    10. 确保自动设置了 **运输**、**商品** 和 **原产国家/地区** 字段。
    11. 在操作窗格上，选择 **保存**。
    12. 在操作窗格 **发票** 选项卡的 **常规** 部分中，选择 **发票**。
    13. 在 **过帐发票** 对话框的 **参数** 快速选项卡上，在 **参数** 部分的 **数量** 字段中，选择 **所有**。
    14. 选择 **确定** 以过帐发票。

12. 将交易记录转移到内部统计日记帐，并查看结果。

    1. 转到 **纳税** > **申报** > **外贸** > **内部统计**。
    2. 在操作窗格上，选择 **转帐**。
    3. 在 **内部统计(转移)** 对话框中，在 **参数** 部分，将 **客户发票** 选项设置为 **是**。
    4. 选择 **确定** 以关闭 **内部统计(转移)** 对话框并查看结果。 已添加 **方向** 字段设置为 **到货** 的新行。            
        此行表示实际退货。

        ![“内部统计”页面上实际退货的行](media/intrastat_deu_3.png)

13. 选择交易记录行，然后选择 **常规** 选项卡以查看更多详细信息。

    ![“内部统计”页面的“常规”选项卡上的实际退货详细信息](media/intrastat_deu_4.png)

14. 使用销售订单创建更正：

    1. 转到 **应收帐款** > **订单** > **所有销售订单**。
    2. 在操作窗格上，选择 **新建**。
    3. 在 **创建销售订单** 对话框的 **客户** 快速选项卡上，在 **客户** 部分的 **客户帐户** 字段中，选择 **DE-012**。
    4. 在 **常规** 快速选项卡的 **存储维度** 部分内的 **站点** 字段中，选择 **1**。
    5. 在 **仓库** 字段中，选择 **11**。
    6. 在 **标题** 选项卡的 **外贸** 快速选项卡上的 **端口** 字段中，选择 **53**。
    7. 在 **统计过程** 字段中，选择 **31710**。
    8. 在 **选项** 选项卡的 **销售订单行** 快速选项卡上，在 **物料编号** 字段中，选择 **D0001**。 然后，在 **数量** 字段中，输入 **-3**。
    9. 在 **行详细信息** 快速选项卡上，在 **外贸** 选项卡上，在 **交易代码** 字段中，选择 **11**。
    10. 确保自动设置了 **运输**、**商品** 和 **原产国家/地区** 字段。
    11. 在操作窗格上，选择 **保存**。
    12. 请确保 **交易记录代码** 字段设置为 11。
    13. 在操作窗格 **发票** 选项卡的 **常规** 部分中，选择 **发票**。

    15. 选择 **确定** 以过帐发票。

15. 将交易记录转移到内部统计日记帐，并查看结果：

    1. 转到 **纳税** > **申报** > **外贸** > **内部统计**。
    2. 在操作窗格上，选择 **转帐**。
    3. 在 **内部统计(转移)** 对话框中，在 **参数** 部分，将 **客户发票** 选项设置为 **是**。
    4. 选择 **确定** 以关闭 **内部统计(转移)** 对话框并查看结果。 已添加在 **更正** 列中显示了复选标记的新行。

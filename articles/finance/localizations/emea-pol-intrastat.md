# <a name="polish-intrastat"></a>波兰内部统计

[!include[banner](../includes/banner.md)]

**内部统计** 页用于生成和报告有关欧盟 (EU) 国家/地区间的贸易信息。 波兰内部统计申报包含有关要申报的货物贸易的信息。

波兰内部统计申报中包括以下字段。 除了 **RodzajTransportu**（运输方式）和 **KrajPochodzenia**（来源国家或地区）不包括在发货中，**IdKontrahenta**（客户的外国增值税号）不包括在到货中，所有字段都包括在到货和发货中。

| 字段名称 | Description |
|-------------------------|-------------------------|
| Deklaracja Data | 创建文档的日期。 |
| Miesiac | 申报的引用月份。 |
| Rok | 申报的引用年份。 |
| Numer | 引用期间的申报编号。 |
| Wersja | 申报的版本号。 |
| NrWlasny | 申报标识符。 此值会自动生成。 |
| Typ | 报告方向。</br><li>对于到货，将打印“P”。</li><li>对于发货，将打印“W”。</li> |
| Rodzaj | 申报的类型。 此值表示报告是原始申报还是更正申报。 |
| UC | 内部统计申报所指向的单位代码。 此值在 **外贸参数** 页面 **代理** 选项卡上 **销售税** 部分的 **免税编号** 字段中指定。 |
| Nazwa | 公司的名称。 |
| Miejscowosc, UlicaNumer, KodPocztowy | 法人的完整地址。 |
| Nip | 波兰纳税标识号（增值税 [VAT] ID）。 |
| Regon | 波兰统计标识号（企业编号）。 |
| LacznaWartoscFaktur | 发票值的总和。 |
| LacznaWartoscStatystyczna | 统计值的总和。 |
| LacznaLiczbaPozycji | 货物物料的总数。 |
| PozId | 给定货物物料的连续编号。 |
| OpisTowaru | 商品的商品名称。 |
| KrajPochodzeniaWysylki | 对方国家或地区的国际标准化组织 (ISO) 代码。 |
| WarunkiDostawy | 交货条款的内部统计代码。 |
| RodzajTransakcji | 指示交易性质的代码。 波兰公司使用两位数的交易代码。 |
| KodTowarowy | 根据组合命名法编码的八位商品代码。 |
| RodzajTransportu | 运输方式的内部统计代码。 |
| KrajPochodzenia | 生产或制造商品的国家或地区的 ISO 代码。 |
| MasaNetto | 以整公斤为单位的净质量。 |
| IloscUzupelniajacaJm | 以辅助度量单位计的数量，以整数表示。 |
| WartoscFaktury | 一个物料涵盖的所有交易的发票价值。 |
| WartoscStatystyczna | 统计值。 |
| Wypelniajacy: NazwiskoImie, Telefon, Faks, Email | 申报提交人员的姓名、电话号码、传真号码和电子邮件地址。 |
| IdKontrahenta | 客户在欧盟成员国的外国增值税号。 |


## <a name="set-up-intrastat"></a>设置内部统计

从全局存储库中，导入以下电子报告 (ER) 配置的最新版本：

   - 内部统计模型
   - 内部统计报表
   - 内部统计 (PL)

有关详细信息，请参阅[从 Configuration service 的全局知识库下载 ER 配置](../../fin-ops-core/dev-itpro/analytics/er-download-configurations-global-repo.md)。

## <a name="set-up-a-vat-id-and-an-enterprise-number-for-your-company"></a>为您的公司设置增值税 ID 和企业编号

### <a name="create-registration-types-for-company-codes"></a>为公司代码创建登记类型

您必须为公司代码创建两个登记类型：一个用于增值税 ID（NIP 代码），一个用于企业编号（登记号代码）。

1. 转到 **组织管理** > **全球通讯簿** > **登记类型** > **登记类型**。
2. 在“操作窗格”上，选择 **新建** 为增值税 ID 创建登记类型。
3. 在 **输入登记类型详细信息** 对话框的 **名称** 字段中，为新登记类型输入名称。 例如，输入 **NIP**。
4. 在 **国家/地区** 字段中，选择 **POL**。
5. 选择 **创建**。
6. 在“操作窗格”上，选择 **新建** 为企业编号创建登记类型。
7. 在 **输入登记类型详细信息** 对话框的 **名称** 字段中，为新登记类型输入名称。 例如，输入 **登记号**。
8. 在 **国家/地区** 字段中，选择 **POL**。
9. 选择 **创建**。

### <a name="match-the-registration-types-with-registration-categories"></a>将登记类型与登记类别匹配

1. 转到 **组织管理** > **全球通讯簿** > **登记类型** > **登记类别**。
2. 在操作窗格上，选择 **新建** 在您创建的每个登记类型和登记类别之间创建链接。

    - 对于增值税 ID（NIP 代码）的登记类型，选择 **增值税 ID** 登记类别。
    - 对于企业编号（登记号代码）的登记类型，选择 **企业 ID (COID)** 登记类别。

### <a name="set-up-a-vat-id-and-an-enterprise-number-for-your-company"></a>为您的公司设置增值税 ID 和企业编号

1. 转至 **组织管理** > **组织** > **法人**。
2. 在网格中，选择您的公司。
3. 在操作窗格上，选择 **登记 ID**。
4. 在 **登记 ID** 快速选项卡上，选择 **添加**。
5. 在 **登记类型** 字段中，选择前面创建的其中一个登记类型。
6. 根据您在上一步中选择的登记类型，输入您公司的增值税 ID（NIP 代码）或企业编号（登记号代码）。
7. 对您之前创建的其他登记类型重复步骤 4 到 6。

## <a name="set-up-a-company-address"></a>设置公司地址

1. 转至 **组织管理** > **组织** > **法人**。
2. 在网格中，选择您的公司。
3. 在 **地址** 选项卡上，选择 **编辑**。
4. 在 **编辑地址** 对话框中，在 **邮政编码** 字段中，选择您公司的邮政编码。
5. 在 **街道** 字段中，输入您的地址。
6. 在 **城市** 字段中，选择您的城市。

## <a name="set-up-foreign-trade-parameters"></a>设置外贸参数

1. 转到 **税务** > **设置** > **外贸参数**。
2. 在 **内部统计** 选项卡的 **电子报告** 快速选项卡上，在 **文件格式映射** 字段中，选择 **内部统计(PL)**。
3. 在 **报表格式映射** 字段中，选择 **内部统计报表**。
4. 在 **商品代码层次结构** 快速选项卡上，在 **类别层次结构** 字段中，选择 **内部统计**。
5. 在 **交易代码** 字段中，选择财产转移的交易代码。 您将此代码用于导致实际或计划转移财产以进行补偿（财务或其他）的交易。 您还使用它进行更正。 波兰的公司使用两位数的交易代码。
6. 在 **贷方通知单** 字段中，选择退货的交易代码。
7. 在 **国家/地区属性** 选项卡上的 **国家/地区** 字段中，列出与您的公司有业务往来的所有国家或地区。 对于欧盟的每个成员国家/地区，在 **国家/地区类型** 字段中选择 **欧盟**，以便国家/地区显示在内部统计报表中。 对于波兰，在 **国家/地区类型** 字段中选择 **国内**。
8. 在 **代理** 选项卡上，在 **代理** 快速选项卡上，在 **销售税** 部分，在 **免税编号** 字段中，输入 **420000** 指示内部统计申报所针对的单位代码。
9. 在 **联系人** 选项卡上，输入提交申报的人员的姓名、电话号码、传真号码和电子邮件地址。
10. 在 **编号规则** 选项卡上，在 **XML 文件编号** 引用的 **编号规则代码** 字段中，指定最多有九个字符的非连续数字序列。 此字段用于自动为内部统计报表中的 **申报标识符** 字段生成值。

## <a name="set-up-product-parameters-for-the-intrastat-declaration"></a>为内部统计申报设置产品参数

1. 转到 **产品信息管理** > **产品** > **已发布产品**。
2. 在网格中选择一个产品。
3. 在 **外贸** 快速选项卡的 **内部统计** 部分，在 **商品** 字段中，选择适当的商品代码。 商品名称将打印在内部统计报表的 **商品描述** 字段中。
4. 在 **来源** 部分的 **国家/地区** 字段中，选择产品的来源国家或地区。
5. 在 **管理库存** 快速选项卡上 **净重** 字段中，输入产品的重量（公斤）。

-   转到 **税务** > **设置** > **外贸** > **内部统计缩减**，选择汇总内部统计信息时应比较的字段。 对于波兰内部统计，请选择以下字段：

    - 商品
    - 交易记录代码
    - 原产国家/地区
    - 传输
    - 交货条款
    - 发送方所在的国家/地区
    - 国家/地区
    - 冲销
    - 免税编号
    - 方向
    - 发票

## <a name="set-up-the-transport-method-and-delivery-terms"></a>设置运输方法和交货条款

1.  设置运输代码。

    1. 转到 **税务** > **设置** > **外贸** > **运输方式**。
    2. 在操作窗格上，选择 **新建**。
    3. 在 **运输** 字段中，输入唯一代码。 波兰公司使用一位数的运输代码。

2.  设置交货方式内部统计代码。

    1. 转到 **采购** > **设置** > **分配** > **交付条款**。
    2. 在网格中，选择一组交货条款。
    3. 在 **常规** 快速选项卡上的 **内部统计代码** 字段中，输入唯一代码。

## <a name="intrastat-transfer"></a>内部统计转移

在 **内部统计** 页面的操作窗格中，您可以选择 **转移** 从您的销售订单、普通发票、采购订单、供应商发票、供应商产品收据、项目发票和转移单自动转移有关集团内贸易的信息。 只有将欧盟国家/地区作为目的地或托运国家或地区的单据会被转移。

您也可以通过在操作窗格中选择 **新建** 手动输入交易。

### <a name="generate-an-intrastat-report"></a>生成内部统计报表

1. 转到 **纳税** > **申报** > **外贸** > **内部统计**。
2. 在操作窗格上，选择 **输出** &gt; **报表**。
3. 在 **内部统计报表** 对话框中，设置以下字段。

    | 字段 | Description |
    |-------------------------|-------------------------|
    | 开始日期 | 选择报表的开始日期。 |
    | 生成文件 | 将此选项设置为 **是** 可为您的内部统计报告生成 .xml 文件。 |
    | 文件名 | 输入 .xml 文件的名称。 |
    | 生成报表 | 将此选项设置为 **是** 可为您的内部统计报告生成 .xlsx 文件。 |
    | 报表文件名称 | 输入 .xlsx 文件的名称。 |
    | 方向 | 选择 **到货** 可获取关于集团内部到货的报表。</br>选择 **发货** 可获取有关集团内部发货的报表。 |
    | 申报标识符 | 文档 ID 会自动生成，可以更新。 |
    | 申报类型 | 为原始申报选择 **申报**。</br>为用于完全替换现有的先前提交的原始申报或更正申报的更正申报选择 **申报更正 - 替换**。 |
    | 文档创建城市 | 输入应在内部统计申报的 **Miejscowosc** 字段中打印的值。 |
    | 文档创建日期 | 输入应在内部统计申报的 **Deklaracja Data** 字段中打印的值。 |
    | 单据编号 | 输入应在内部统计申报的 **Numer** 字段中打印的值。 |
    | 单据版本 | 输入应在内部统计申报的 **Wersja** 字段中打印的值。 |

4. 选择 **确定**，然后查看生成的报表。

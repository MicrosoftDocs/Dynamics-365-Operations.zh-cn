# <a name="inventory-posting"></a>库存过帐

**库存过帐模板** 页上的 **库存** 选项卡用于控制如何将库存交易过帐到总帐。 库存交易是未从销售订单、采购订单或生产订单创建的交易。 它们同时自动将实际和财务更新过帐到库存。 转移单是一个例外，当您装运和接收库存时，转移单会将实际更新和财务更新分开。

下表显示库存交易的一些示例。

| 交易记录类型 | 收货或发货 | 实际过帐、财务过帐或两者 | Description |
|---|---|---|---|
| 变动 | 两者都有 | 两者都有 | <p>移动日记帐是可用于添加或删除库存的库存日记帐。 其作用类似于库存调整日记帐。 但是，一个关键差异是它指定了抵销分录的主科目。</p><p>移动日记帐输入必须支出的期初余额和一次性调整。 例如，在为销售示例删除库存时会使用它。</p><p>过帐日记帐时，将同时进行实际更新和财务更新。</p><p>日记帐行上的正数量表示收货，负数量表示发货。</p> |
| 库存调整 | 两者都有 | 两者都有 | 库存调整日记帐用于添加或删除库存。 它不允许您选择抵销帐户。 只有在 **库存过帐模板** 页上指定的帐户用于更新总帐条目。 |
| 转移（日记帐） | 两者都有 | 两者都有 | <p>转移日记帐用于将库存从一个库位移至另一个库位（使用存储维度）。 它使用新产品或跟踪维度更新库存交易的产品信息。</p><p>转移日记帐会为物料移动创建一行。 此行具有库存维度的“自”和“至”字段。 转移日记帐中的每个行都会创建两个库存交易。 为“自”库存维度创建了一个交易。 它表示发货并具有负数量。 为“至”库存维度创建了另一个交易。 它表示收货并具有正数量。</p><p>如果库存变动被视为非财务转移，则转移日记帐可能不会创建凭证。 未使用 **存储维度组** 或 **跟踪维度组** 页面上的 **财务库存** 选项标记对“至”和“至”值不同的库存维度。 例如，如果未在 **库位** 维度上标记 **财务库存** 选项，并且您将库存从一个库位移动到同一站点和仓库中的另一个库位，则不会创建凭证。</p><p>转移日记帐不同于转移单，因为没有移动单据。 通过过帐日记帐，会在一个交易中完成更新。 相比之下，转移单可以生成领料和装运单据，并且需要两次更新来处理交易。</p> |
| 转移单装运 | 问题 | 两者都有 | <p>当您创建新转移单时，行和库存维度的每个组合都有两个库存交易：一个用于转移单装运，一个用于转移单收货。 装运转移单时，将更新两个库存交易，并为货物中转创建两个额外的库存交易，总共有四个库存交易。</p><ol><li>第一个库存交易用于从源仓库和库位中删除物料。 此交易的发货状态为 **已售出**，以指示该物料在仓库中不再可用。</li><li>第二个库存交易用于将物料添加到中转仓库，此中转仓库被选为要从中进行转移的仓库。 此交易的接收状态为 **已采购**。</li><li>第三个库存交易用于从中转仓库转移到目标仓库。 此交易的发货状态为 **实际预留**。 此状态可确保物料在仍在途时无法被另一个流程使用。</li><li>第四次库存交易用于在目标仓库收货。 此交易的接收状态为 **已订单**。</li></ol> |
| 转移单接收 | 收货 | 两者都有 | 在目标仓库中接收转移单后，在途仓库中的库存交易的库存状态（前面示例中的编号 3）将更新为 **已售出**，目标仓库的库存交易的库存状态将更新为 **已采购**。 |
| 物料清单 | 两者都有 | 两者都有 | 您可以使用物料清单 (BOM) 日记帐将产品报告为完工入库，并使用在不使用生产订单的情况下在流程中使用的原材料。 对于成品，物料清单日记帐通常包含至少一个正行；对于要使用的原材料，包含至少一个或多个负行。 成品行是库存收货，而负行是原材料库存发货。 创建物料清单日记帐时，第一行是物料清单标题，添加的后续行为物料清单行。 |
| 库存所有权更改 | 两者都有 | 两者都有 | 库存所有权更改日记帐用于将托运库存的所有权从供应商更改为您的组织。 库存所有权更改日记帐与库存转移日记帐相似，因为两个库存交易与行和库存维度的每个组合相关。 一个库存交易从 **所有权** 维度内的供应商中删除库存，另一个库存交易将物料添加到 **所有权** 维度内的法人。 |
| 物料到达 | 收货 | 实际 | 物料到达日记帐用于将库存的收货状态更新为 **已登记**。 它通常用于采购订单收货和销售订单退货。 |
| 生产输入 | 收货 | 实际 | 生产输入日记帐类似于物料到达日记帐。 生产输入用于在仓库中根据生产订单接收成品。 过帐日记帐时，库存交易的状态将更新为 **已登记**。 |
| 盘点 | 两者都有 | 两者都有 | 盘点日记帐用于记录为特定库存维度组合盘点的物料数量。 当日记帐的行有盘点差异时，将创建库存交易。 盘点数量较高的行会导致库存收货。 盘点数量较低的行会导致库存发货。 盘点数量与预期数量匹配的行没有库存记录。 |
| 标签盘点 | 不适用 | 不适用 | 标记盘点日记帐用于跟踪在完整库存盘点中分配和使用的库存标记。 您可以使用日记帐将标记编号分配给物料和库存维度的特定组合，并在完整库存期间跟踪该标记的状态。 标记盘点日记帐不创建库存交易。 |
| 质检订单 | 问题 | 实际 | <p>质检订单用于记录库存中给定批次的测试结果。 每个质检订单都与现有库存交易或库存交易的一部分相关。</p><p>质检订单将在以下两种情况下生成新的库存交易：</p><ul><li>质检订单在 **常规** 选项卡上标记为 **破坏性测试**。</li><li>质检订单在 **常规** 选项卡上标记为 **验证失败时进行检验**，并且测试未能通过验证。 在此情况下，会创建两个库存交易。 一个库存交易从原始库存维度组合和库位中删除物料，另一个库存交易将物料添加到检验仓库。</li></ul> |
| 检验单 | 两者都有 | 两者都有 | <p>检验单的库存交易就像转移单。 检验仓库用于跟踪库存。 有两个收货交易和两个发货交易。</p><p>当您最初创建订单时，将创建两个交易。 对于链接到物料所在仓库的检验仓库，收货交易具有 **已订购** 状态。 对于存储物料的仓库，发货交易具有 **在单** 状态。</p><p>当您启动检验单时，将创建另外两个交易，并更新现有交易。 检验仓库的收货交易的状态将更新为 **已接收**，存储仓库的发货交易的状态将更新为 **已扣除**。</p><p>这两个新交易用于表示最终移回存储仓库。 对于存储仓库，收货交易的状态为 **已订购**；对于检验仓库，该发货交易的状态为 **实际预留**。</p><p>当您将检验单报告为完工入库时，此操作表示检验单的实际更新。 存储仓库中的收据状态将更新为 **已接收**。</p><p>当您结束检验单时，此操作表示检验单的财务更新。 发货交易的状态将更新为 **已售出**，收货交易的状态将更新为 **已采购**。</p><p>或者，您可以报废检验单。 此操作会从库存中删除物料。 当您报废检验单时，仅保留三个交易。 存储仓库的收货交易已删除，剩余收货交易的状态将更新为 **已采购**。 两个收货交易的状态将更新为 **已售出**。 此操作是订单的实际和财务更新。</p> |

## <a name="fixed-receipt-price-posting"></a>固定收货价过帐

固定收货价使您可以使用产品的标准成本。 它是在物料的 **物料模型组** 页上选择 **标准成本** 选项的替代方法。 使用固定收货价时的主要区别是，过帐库存收货时，当前配置的成本价将用于物料。 没有固定收货价的成本重估流程。 过帐财务更新时，将在过帐时使用当前成本价。 如果收货时使用的成本价与发票不同，差异将过帐到固定收货价损益帐户。

若要选择性地对产品使用固定收货价，您必须完成以下配置：

- 对于在库存交易行中选择的物料，选择 **物料模型组** 页上的 **过帐实际库存** 复选框。
- 在 **物料模型组** 页面上选中 **固定收货价** 复选框。
- 在 **库存过帐模板** 页上，为以下过帐类型指定主科目：

    - 固定收货价收益
    - 固定收货价损失

有关详细信息，请参阅[固定收货价](/supply-chain/cost-management/fixed-receipt-price.md)。

## <a name="catch-weight-posting"></a>实际称重过帐

实际称重产品通常用于产品重量和/或大小有轻微差异的行业（如食品行业）。 非定重产品使用两种度量单位：库存单位和非定重单位。 库存重量在涉及到仓库时可能不同于库存从仓库发放时的重量。 实际称重产品处理会调整库存。

如果实际称重存在差异，则差异将过帐到 **库存过帐模板** 页上的 **实际称重损失** 和 **实际称重利润** 字段中指定的帐户。 通常，会在这两个字段中指定相同的主科目。

下表显示默认过帐类型的示例，并包括示例主科目和描述。

- “借方/贷方？” 列指示交易通常是借记还是贷记。 在某种情况下，交易可以过帐借项或贷项。
- “结算帐户”列指示过帐类型为结算帐户。 换言之，在过帐以后的交易时，将自动冲销此帐户中过帐的金额。
- “P/F”列指示过帐类型。 “P”表示实际过帐，“F”表示财务过帐。
- “跟随”列指示过帐类型的主科目通常是否与其他过帐类型的主科目相同。 具体来说，它指示通常用于过帐的过帐类型。

> [!NOTE]
> 表中的这些主科目和主科目名称是一些建议。 我们建议您与您的会计师合作，确定适合您的业务需求的最佳配置。

| 过帐类型 | 主帐户示例 | 主帐户名称示例 | 帐户类型 | 借方/贷方？ | 结算帐户 | P/F | 跟随 | Description |
|---|---|---|---|---|---|---|---|---|
| 实际称重损失帐户 | 510520 | 库存调整 | 支出 | | 否 | 两者都有 | 实际称重利润帐户 | 当您过帐实际称重额较低的库存交易时，将使用此帐户。 |
| 实际称重利润帐户 | 510520 | 库存调整 | 支出 | | 否 | 两者都有 | 实际称重损失帐户 | 当您过帐实际称重额较高的库存交易时，将使用此帐户。 |

有关实际称重产品的更多信息，请参阅[关于实际称重物料](/dynamicsax-2012/appuser-itpro/about-catch-weight-items.md)以及[使用仓库管理进行实际称重产品处理](/supply-chain/warehousing/catch-weight-processing.md)。

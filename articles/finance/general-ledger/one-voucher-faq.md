# <a name="one-voucher-faq"></a>一个凭证常见问题解答

[!include [banner](../includes/banner.md)]

本主题解答有关一个凭证功能的常见问题。 财务日记帐的一个凭证让您可以在单个凭证的上下文中输入多个子分类帐交易记录。 您可以在该凭证中包括的日记帐可以是普通日记帐、固定资产日记帐和供应商付款日记帐，等等。

## <a name="when-will-the-one-voucher-functionality-be-deprecated"></a>一个凭证功能会在什么时间弃用？

Microsoft 无法提供关于何时弃用一个凭证功能的准确估计，可能至少会在两年以后。

如“一个凭证”文档中所述，很多业务要求只能使用“一个凭证”才能满足。 Microsoft 必须确保在该功能弃用后，所有已确定的业务要求在系统中仍然能够得到满足。 因此，可能必须添加新功能来填补功能空白。

填补所有功能空白后，Microsoft 将通知您该功能将弃用。 但是，收到通知到弃用生效至少会间隔一年。

有关详细信息，请参阅[“一个凭证”文档](one-voucher.md)。

## <a name="what-will-the-solution-that-replaces-one-voucher-look-like"></a>替换“一个凭证”的解决方案将是什么样的？

Microsoft 无法提供具体的解决方案，因为每个功能空白都是不同的，必须根据业务要求进行评估。 一些功能上的空白可能会替换为帮助满足特定业务要求的功能。 但是，其他空白可能通过继续允许在日记帐中输入来填补，就像使用“一个凭证”时一样，但会根据需要增强系统来跟踪更多详细信息。

## <a name="where-can-i-track-the-progress-of-the-feature-gaps-being-filled"></a>在哪里可以跟踪功能空白的填补进度？

对于每项新功能，客户必须关注发布计划和“新增功能或更改”文档。 每项功能都会被添加到这些文档中，并且将有注释指示该功能是为了满足以前需要使用“一个凭证”功能的业务要求。

## <a name="when-the-deprecation-date-is-identified-where-will-it-be-communicated"></a>弃用日期确定后，将在哪里传达此信息？

弃用“一个凭证”是一项重大更改，会广泛传达。 在弃用生效之前，会通过产品文档、博客文章以及在合适的 Microsoft 会议上宣布来进行传达。

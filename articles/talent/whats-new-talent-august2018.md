# <a name="whats-new-or-changed-in-dynamics-365-talent---core-hr-august-2018"></a>Dynamics 365 Talent - Core HR（2018 年 8 月）中的新增功能或更改

[!include [banner](includes/banner.md)]

**内部版本 8.1.104**

此主题介绍了 Dynamics 365 Talent: Core HR 中的新增功能和更改的功能。

## <a name="view-expiring-records-in-manager-self-service"></a>在经理自助服务中查看即将过期的记录

现在可以在经理自助服务中查看即将过期的记录。 可通过新选项配置经理可查看哪些信息。 这些包括:

-   证书

-   标识号

-   试用期

-   筛选

-   测试

还可以通过此功能指定查找即将过期的记录的日期范围。

## <a name="configurable-transfer-actions"></a>可配置的转岗行为

可按角色配置录入壮岗请求时可使用的选项。 此功能可提高组织中的角色的灵活性。

例如，申请员工转岗的经理可能无法访问以建议或输入薪酬金额，或无法选择将与转岗请求关联的任务列表。 在这种情况下，经理可以创建并提交转岗请求，但不可以输入薪酬或任务列表分配。 在这个相同配置中，HR 可以分配新的薪酬值，还可以分配转岗完成后需要填写的其他任何核对清单。

默认情况下，新配置选项设置为不改变此更新之前的功能。

## <a name="leave-and-absence"></a>休假和缺勤

下面是休假和缺勤中新增的日期字段。

通过此功能，可以在计划级别设置应计期间基数以使用特定员工日期。 这样就可以在缺勤应计过程中使用非计划开始日期。 员工特定日期的选项中包含以下值：

-   自定义（此更新之前提供）

-   周年纪念日日期

-   原始雇用日期

-   受聘日期

-   工作人员的调整后开始日期

-   工作人员的开始日期

配置为使用一个员工特定的日期时，登记过程将使用指定日期计算应计期间。 也会在员工的计划登记中显示应计期间基数，以帮助了解应计过程中使用的信息。

## <a name="microsoft-word-integration"></a>Microsoft Word 集成

已在整个 Core HR 中启用了文档模板。 可通过此功能基于你创建的 Word 模板创建信函和报表。

[Office 集成教程](https://docs.microsoft.com/dynamics365/unified-operations/dev-itpro/office-integration/office-integration-tutorial?toc=/dynamics365/unified-operations/talent/toc.json)中提供了有关此功能的更多信息。


## <a name="other-fixes"></a>其他修复

此发行版中还包含一些缺陷修复，增加了新实体，修复了现有实体，并本地化了标签更改。

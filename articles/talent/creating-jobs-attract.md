# <a name="create-a-job"></a>创建职位

[!include [banner](includes/banner.md)]

此主题描述 Microsoft Dynamics 365 Talent: Attract 中的工作元素。 它还介绍了如何创建工作。

## <a name="job-creation"></a>工作创建

管理员、招聘人员和招聘经理可以创建工作。 当创建工作时，系统将提示您选择您在流程中的角色：招聘经理或招聘人员。 在选择角色之后，将提示您选择流程模板。 如果您选择**跳过**，将使用默认模板。 有关流程模板的详细信息，请参阅[在 Attract 中创建流程模板](./process-templates-attract.md)。

Attract 中的工作包含工作详细信息、招聘团队、招聘流程、工作发布和分析。

## <a name="job-details"></a>工作详细信息

**工作详细信息**选项卡包含有关工作职责和属性的详细信息。 职务、工作描述和工作位置字段是必填字段。 其他字段是可选的。

默认情况下，**空缺数量**字段设置为 **1**。 然而，您可以更改值。 当为工作准备好了聘约时，**可用空缺数量**字段的值将减少。

如果职位管理在管理员中心已打开，**更新职位**查找将可用。 此查找读取 Common Data Service 中的 JobPosition 实体并返回可为工作选择的职位列表。 如果选择的职位数量超出空缺职位数量，将收到警告。 如果一个职位用于多个工作，您也会收到警告。

> [!NOTE]
> 职位管理随综合招聘附件提供。

根据招聘流程的聘约活动的设置，职位编号可以在聘约中使用两次。 有关详细信息，请参阅[招聘流程](./activities-attract.md)。

Attract 包含一组默认**技能**。 在键入时，这些技能作为建议出现。 您可以通过在字段中输入新技能文本然后按 Enter 来添加更多技能。

Attract 包含一组默认的**工作职能**。 您可以通过在字段中输入新的工作职能然后按 Enter 来最多再添加三个工作职能。

Attract 包含一组默认的**公司行业**。 您可以通过在字段中输入新的公司行业然后按 Enter 来最多再添加三个公司行业。

## <a name="hiring-team"></a>招聘团队

**招聘团队**选项卡包含工作所涉及的个人的列表。 在用户被添加到招聘团队时，必须为他们分配在招聘团队中的角色。 角色确定用户有权访问的数据和他们将收到的通知。 可以选择的角色有**招聘人员**、**招聘经理**、**委托人**和**面试官**。 有关角色权限的详细信息，请参阅“角色管理”文档。 招聘人员和招聘经理可以指派一个或多个委托人来代表他们工作。 有关委托人的详细信息，请参阅 [Attract 中的安全和角色管理](./security-attract.md)。

招聘团队可以在激活工作后更新。

## <a name="process"></a>进程

有关招聘流程的默认信息基于创建工作时选择的流程模板。 如果当时未选择特定模板，将使用默认模板。 在您定义招聘流程时，您可以添加或删除各个阶段，“应聘者”、“申请”和“聘约”阶段除外。 虽然不能删除“应聘者”阶段，但可以将其关闭。 在每个阶段中，可以添加或移除一个或多个预定义的活动。

有关可以添加到招聘流程的活动的详细信息，请参阅 [Attract 中的招聘流程活动](./activities-attract.md)。

> [!NOTE]
> 招聘流程不能在激活工作后更新。

## <a name="postings"></a>过帐记录

在激活工作后，便可以发布工作。 只有招聘人员和管理员可以发布工作。 工作可以发布到 Talent Careers（Dynamics 365 Talent 求职站点）或 LinkedIn。 Attract 团队正在继续努力与工作面板整合者合作。 此列表一段时间后将扩展。 如果工作作为仅内部工作发布，应聘者需要 AAD 帐户才能查看和申请该工作。 如果工作作为公开工作列出，则应聘者可使用所有身份验证选项查看和申请工作。 

有关工作发布的详细信息，请参阅 [Attract 中的求职站点功能](career-site.md)。

> [!NOTE]
> 工作发布功能只随 Attract 的综合招聘附件提供。

## <a name="activate"></a>启用

在激活工作后，可以发布工作，并且可以将应聘者和申请人添加到工作中。 向工作添加应聘者的选项在招聘流程中的“应聘者”活动中设置。

> [!NOTE]
> 招聘流程不能在激活工作后更新。

## <a name="prospects-and-applicants"></a>应聘者和申请人

向工作添加应聘者的选项在招聘流程中的[应聘者活动](./activities-attract.md#prospect-activity)中设置。 此选项应在激活工作之前设置。 在激活工作后，可以将应聘者和申请人添加到工作中。

## <a name="approvals"></a>审核

Attract 工作可以提交以供审核。 并非所有工作都需要审核。 要求在模板级别设置。 默认情况下，审核在模板中是关闭的。 若要设置审核，请转到流程模板，将**审核**字段设置为“默认”。 然后在您创建工作时选择该模板。

在保存工作之后，可以提交工作以供审核。 下表列出了使用审核的文档的状态。

| 状态   | 状态                                                               |
|----------|---------------------------------------------------------------------|
| 草案    | 工作已保存，但未提交到工作流。 |
| 挂起  | 工作已提交给审核人。                            |
| 已审批 | 工作已审核，但未激活。            |
| 已拒绝 | 工作被拒绝，无法激活。               |
| 有效的   | 工作已审核并激活。                            |

在工作列表中，您可以筛选工作状态。

审核可以发送给公司中的任何 Microsoft Azure Active Directory (Azure AD) 用户。 审核可以同时发送给列出为审核人的所有人员。 工作只有在经过所有审核人的审核后，才能进入下一阶段。 如果一位审核人拒绝工作，该工作将显示**已拒绝**状态。 在审核工作后，便可以激活工作。

如果用户编辑已审核但未激活的工作，工作状态将重置为**草稿**，并且必须重新提交该工作以供审核。 不能编辑已审核且已激活的工作。

列出为审核人的人员将在 Attract 中收到通知和电子邮件，通知他们有要审核的项目。  审核人可单击电子邮件中的链接打开工作，查看详细信息，然后批准或拒绝。 工作的状态设置为**已批准**或**已拒绝**之后，提交者将在 Attract 中收到通知，并且还会收到一封电子邮件。 此外，如果审核人在 24 小时内尚未响应审核请求，审核人还会收到提醒电子邮件。

> [!NOTE]
> 您可以为审核电子邮件创建自定义电子邮件模板。 有关详细信息，请参阅[创建和管理电子邮件模板](https://docs.microsoft.com/dynamics365/unified-operations/talent/email-templates)。

## <a name="create-a-job"></a>创建工作

请遵从这些步骤创建工作。

1. 转至**工作**。
2. 选择**新建**。
3. 在**职务**字段中，输入职务。 在**角色**字段中，输入您的角色。
4. 在**模板**字段中，选择模板。 或者，选择**跳过**。 如果您选择**跳过**，将使用标记为默认模板的模板。

    如果文档应经过审核流程，请选择**审核流程**字段设置为**默认**的模板。

5. 在**详细信息**选项卡上，输入工作的详细信息。 **职务**、**工作描述**和**位置**字段是必填字段。
6. 选择**保存**。
7. 在**招聘团队**选项卡上，添加招聘经理、招聘人员或面试官。
8. 选择**保存**。
9. 在**流程**选项卡上，根据需要添加或删除阶段：

    - 若要添加阶段，请选择 **+ 新阶段**。
    - 若要删除阶段，将指针悬停在要删除的阶段上，然后选择出现的垃圾桶按钮。

        > [!NOTE]
        > 不能删除“应聘者”、“申请”和“聘约”阶段。

10. 根据需要添加或删除活动：

    - 要添加活动，请将其从右侧的列表拖到相应的阶段。 或者，双击活动，然后选择要添加到的阶段。
    - 若要删除活动，展开活动，然后选择活动标题上的垃圾桶按钮。

11. 选择**保存**。
12. 如果您选择了使用审核流程，请执行以下步骤：

    1. 选择 **+ 添加审核人**，然后输入具有 Azure AD 帐户的用户。 您可以添加多个审核人。
    2. 选择**发送给审核人**。

    工作的**工作状态**字段设置为**待处理**。 在**工作状态**字段的值更改为**已审核**后，可以激活该工作。

13. 若要激活工作，请选择**激活**。
14. 要发布工作，请转到**发布**，然后选择 Talent Careers 站点或 LinkedIn 下的**立即发布**。

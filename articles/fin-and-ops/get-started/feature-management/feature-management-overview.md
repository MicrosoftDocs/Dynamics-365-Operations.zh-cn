# <a name="feature-management-overview"></a>功能管理概述

[!include[banner](../../includes/banner.md)]
[!include [banner](../../includes/preview-banner.md)]

每个 Microsoft Dynamics 365 for Finance and Operations 版本中都会增加和更新功能。 功能管理体验提供一个工作区，可在其中查看各版本已交付的功能的列表。 默认情况下，新功能处于关闭状态。 可使用该工作区开启这些功能并查看其文档。

## <a name="the-feature-management-workspace"></a>功能管理工作区

可通过在仪表板中选择相应磁贴打开**功能管理**工作区。 将看到一个页面，其中显示功能管理体验支持的所有版本的功能列表。 Microsoft 将不断增强功能管理体验，使其包含更多功能以帮助您管理功能。

功能列表中包含以下信息：

- **功能名称** – 已添加的功能的说明。
- **启用状态** – 一个符号指示功能已启用（选中符号），未启用（空白），已计划启用（时钟）还是已强制启用（锁）。 此处显示的设置用于所有法人。 请注意，功能即使已开启，仍然受安全性控制。 因此，功能仅对根据安全角色具有其访问权的用户可用。 还仅对用户可访问的法人可用。
- **启用日期** – 功能的开启日期或将开启日期（如果该日期为将来日期）。
- **功能添加日期** – 功能添加到您的环境时的日期。 此日期是您在月度版本周期内更新环境时自动输入的。
- **模块** – 新功能影响的模块。

选择功能时，功能列表右侧的详细信息窗格内会显示更多信息。 可以在窗格顶部看到功能名称、功能的添加日期、功能影响的模块，以及**了解更多**链接。 选择此链接可查看功能的文档。 如果文档不可用，将回到临时页。 详细信息窗格中还包含**注释**字段，可在其中添加您自己有关功能的注释。

**功能管理**工作区中还包含多个选项卡，这些选项卡内有功能列表。 
- **新增** - 显示上次月度更新以来增加的所有功能。 如果跳过了任何月度更新，将包含上次更新以来的所有新功能。 最新的功能在列表顶部显示。 页面顶部的磁贴中还会显示新功能的总数。
- **未启用** - 显示所有尚未启用的功能。 最新的功能在列表顶部显示。 页面顶部的磁贴中还会显示新功能的总数。
- **已计划** - 显示所有已计划在将来某个日期启用的功能。 计划的日期最早的功能在列表顶部显示。 页面顶部的磁贴中还会显示新功能的总数。
- **全部** - 显示全部功能。 最新的功能在列表顶部显示。


## <a name="enable-a-feature"></a>启用功能

如果尚未启用某个功能，详细信息窗格中将显示**启用**按钮。 可使用此按钮启用该功能。

1. 选择要启用的功能，然后在详细信息窗格中选择**启用**。
2. 将显示一个滑块，可在此处指定应启用该功能的日期。 默认情况下，此日期设置为当天。
3. 选择**启用**以启用功能。

某些功能在启用后不能禁用。 如果尝试启用的功能不可禁用，您将收到警告。 此时可选择**取消**以取消操作并让功能保持禁用状态。 但是，如果选择**启用**并启用功能，以后不能禁用该功能。

启用该功能后，详细信息窗格的**了解更多**链接下将显示一条消息。 此消息说明已启用该功能或指出将在将来启用该功能。 如果选择将来的日期，将在**已计划**列表中显示该功能。 只要在功能列表中选择该功能，都会显示此消息。 计划将来启用的功能将由批处理根据系统日期提供的时区在将来的午夜启用。 

## <a name="reschedule-a-feature"></a>重新计划功能

如果某个功能已计划在将来启用，详细信息窗格中将显示**重新计划**按钮。 可使用此按钮将**启用日期**更改为其他日期。

1. 选择要重新计划的已计划功能，然后在详细信息窗格中选择**重新计划**。
2. 将显示一个滑块，可在此处指定应启用该功能的日期。 
3. 选择**启用**以重新计划该功能，或选择**禁用**以取消计划。

## <a name="disable-a-feature"></a>禁用功能

如果已启用某个功能，详细信息窗格中将显示**禁用**按钮。 可使用此按钮禁用该功能。 如果功能在启用后不可禁用，则**禁用**按钮不可用。

- 选择要关闭的功能，然后在详细信息窗格中选择**禁用**。

- 将禁用功能，并清除日期。

禁用该功能后，详细信息窗格的**了解更多**链接下将显示一条消息。 此消息说明尚未启用该功能，并将在**未启用**列表中显示该功能。 只要在功能列表中选择该功能，都会显示此消息。

## <a name="features-that-must-be-enabled"></a>必须启用的功能

可能会交付某个在您执行更新时必须自动启用的关键功能。 该功能将在**启用日期**自动启用。 将在详细信息窗格的**了解更多**链接下显示一条消息。 此消息说明已启用该功能或将在**启用日期**自动启用该功能。 只要在功能列表中选择该功能，此消息都会显示。

## <a name="assigning-roles"></a>分配角色

**功能管理**工作区可以由系统管理员打开，也可以由分配有为了支持功能管理体验而创建的功能管理员或功能查看者角色的用户打开。 功能管理员角色的用户可打开或关闭任何功能。 还可以更新功能的注释部分。 功能查看者角色的用户只能查看**功能管理**工作区。 不能打开或关闭功能。

功能管理员角色和功能查看者角色不能覆盖用户的现有安全性。 这些角色只能控制用于启用功能的访问权限。 不能提供功能本身的访问权限。

## <a name="using-feature-management-to-enable-isv-features-or-custom-features"></a>使用功能管理启用 ISV 功能或自定义功能

功能管理流程当前对 ISV 功能或自定义功能不可用。 我们将增加更多功能以增强功能管理，并在这些增强完成后，对所有功能开放功能管理，并提供有关如何更新功能以使用功能管理的具体说明。

## <a name="feature-management-and-flighting"></a>功能管理和外部测试

可通过功能管理控制每个版本中提供的功能。 Microsoft 团队可通过外部测试将功能发布给一小部分客户，这样测试和验证这些功能时不会影响全体客户。 功能管理不会控制任何功能的外部测试。

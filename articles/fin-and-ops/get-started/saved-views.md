# <a name="saved-views"></a>保存的视图

[!include [banner](../includes/banner.md)]
[!include [private preview banner](../includes/private-preview-banner.md)]

## <a name="introduction"></a>简介
个性化在让用户和组织优化 Microsoft Dynamics 365 for Finance and Operations 中的用户体验以满足其需求中扮演着重要的角色。 有关个性化的更多详细信息，请参阅[打造个性化的用户体验](personalize-user-experience.md)。

在传统个性化中，用户的每个窗体只有一组个性化设置。 保存的视图在若干重要方面扩展了个性化：

-    视图允许用户的每个窗体有多组有名称的个性化设置，可以根据需要快速切换。 这样用户就可以为一个页面创建多个经过优化的视图，其中的每个视图都经过定制，可以满足执行特定业务任务的需要。 

-    为特定页面类型创建的视图中也可以包含用户添加的筛选器或排序，用于快速回到经常筛选的数据集。 有关更多详细信息，请参阅[哪些页面支持视图](saved-views.md#what-pages-support-views)部分。 

-    可以将视图发布给安全角色，这样具有该角色的任何用户都可以访问和使用该视图，无论这些用户是否具备进行个性化的能力。 组织可通过此项发布功能定义针对自己的业务进行过优化的公司标准视图。 有关详细信息，请参阅[使用视图管理组织级个性化设置](saved-views.md#managing-personalizations-at-an-organizational-level-with-views)部分。

-    和传统个性化不同，用户在执行显式个性化设置或筛选列表时，不会自动保存视图。 需要显式保存，才能在执行与视图关联的更改前后创建视图时提供灵活性，以及确保不应长期使用的筛选器或个性化设置不会意外更改视图定义。  

## <a name="switching-between-views"></a>在视图之间进行切换
为环境启用视图之后，所有支持视图的页面中窗体顶部都将有一个折叠的视图选择器控件，该控件显示当前视图的名称。  

视图选择器有两种尺寸变体： 

-   **大视图选择器**：因为一些原因，明显拥有列表的页面的视图选择器较大。 最重要的是，较大视图选择器表示视图中可能包含用户定义的筛选器。 由于视图中包含筛选器，所以也有必要采用较大选择器，因为视图名称通常最适合描述屏幕中显示的数据，除了用户经常在这些页面类型中切换视图的情况。  
 
-   **小视图选择器**：其他所有整页窗体都采用在页面标题旁边显示的较小视图选择器。 这些页面中的视图内仅包含个性化设置（不包含用户定义的筛选器）。 在这些页面中，窗体标题或记录标题是窗体顶部最重要的信息。 较小大小还体现了预计这些页面中切换视图的频率较低。 
 
如果单击视图名称，视图选择器将打开，并显示该页面的可用视图列表

-    **经典视图**：经典视图是页面的现成视图，未应用显式个性化设置。  
-    **个人视图**：不带挂锁的视图表示您的个人视图。 这些视图是您创建的或管理员提供给您的。  
-    **锁定视图**：在视图选择器中，某些视图（如经典视图和发布给您的角色的所有视图）旁边有一个挂锁，表示不能对这些视图进行编辑。 但是，将自动保存对页面用法的隐式个性化设置，如更改网格列宽或展开或折叠快速选项卡。 不过，如果您有个性化权限，则可通过使用**保存副本**操作基于已锁定视图创建个人视图。
-    **新视图**：已发布但尚未打开的视图的视图名称旁边有一个火花。  

若要切换到其他视图，首先请打开视图选择器，然后选择要加载的视图。 

## <a name="creating-and-modifying-views"></a>创建和修改视图
和传统个性化不同，用户在执行显式个性化设置（或筛选列表）时，不会自动保存视图。 需要执行显式操作才能将这些更改保存到视图。 这样在执行与视图关联的更改前后创建视图时为用户提供了灵活性，还确保了一次性筛选器或个性化设置不会意外更改视图定义。 请注意，将把隐式个性化设置（如展开或折叠快速选项卡，或更改网格列宽）自动保存到当前视图，即使是已锁定视图也不例外。 

为了确保显示视图的当前状态，在您开始通过执行显式个性化或筛选对视图进行更改时，当前视图名称旁边将显示星号，表示您看到的是该视图已修改，但未保存的版本。

如果要保存这些更改，请执行以下步骤。
1.  选择视图名称打开视图选择器。
2.  若要修改现有视图：
     1. 选择**保存**。 请注意，不能为已锁定视图启用此操作。 
3.  若要新建视图：
     1.    选择**保存副本**。 
     2.    输入视图名称和描述（可选）。
     3.    选择**保存**。

## <a name="changing-the-default-view"></a>更改默认视图
默认视图是在您首次导航到页面时系统尝试打开的视图。 应该将此项设置为预期经常打开的视图。  

若要更改页面的默认视图，请执行以下步骤： 
1.  切换到要用作默认视图的视图。 
2.  选择视图名称打开视图选择器。 
3.  选择**更多**，然后选择**固定为默认值**。  

也可以在创建新视图（使用**保存副本**操作）时，通过在保存视图之前设置**固定为默认值**选项来将新视图设置为默认视图。  

请注意，在某些情况下，首次导航到页面时，不会执行与默认视图关联的查询。 例如，如果通过磁贴导航到某个页面，则无论与默认视图关联的是哪个查询，也会执行磁贴的查询。 此外，如果导航到其经典视图已定义有查询的页面，最初将执行原始查询，而不是默认视图的查询。 如果出现这种情况，加载视图时，将通过参考信息通知您。 要让视图查询按照预期执行，请在页面加载后切换视图。

## <a name="managing-personal-views"></a>管理个人视图 
**管理我的视图**对话框提供对个人视图和视图选择器中的视图顺序的基本维护功能。 若要打开此页面，请单击视图名称打开视图选择器下拉菜单，选择**更多**，然后选择**管理我的视图**。  

可对该页面的可用视图列表执行下面的一组操作。  
-    **更改默认视图**：可使用**固定为默认值**操作将当前突出显示的视图设置为该页面的默认视图。  
-    **为视图重新排序**：可使用**上移**和**下移**操作按照特定顺序重新排列视图。  
-    **重命名视图**：可使用**重命名**操作更改当前突出显示的个人视图的名称。 此操作对已锁定视图关闭。 
-    **删除视图**：可使用**删除**操作从页面中永久删除当前突出显示的视图。 不能恢复已删除的视图。  

此对话框中进行的任何更改都将在您选择**保存**按钮之后生效。

## <a name="managing-personalizations-at-an-organizational-level-with-views"></a>使用视图在组织级别管理个性化设置
若要了解组织级别管理个性化设置方面的改进，首先看看使用视图之前个性化管理的工作原理。  

在采用视图之前，管理员将页面的一组个性化设置应用于一个用户、一组用户或使用个性化窗体的用户。 如果这些用户具有个性化权限，将把个性化设置应用于该页面。 但是，不能阻止用户进一步个性化页面，这意味着组织无法确保其用户具有一致的用户界面。 如果这些用户中任何一位没有个性化权限，则不会加载管理员为其提供的个性化设置。 此外，如果组织中雇佣了新用户，管理员需要手动为该用户加载一组个性化设置。 没有用于指定一组应向该用户提供的特定个性化设置的自动机制。

采用保存的视图功能之后，大大简化了个性化设置的组织管理，主要是因为可以将视图发布给安全角色。 发布视图之后，具有该角色的任何用户都可以访问和使用该视图，无论这些用户是否具备进行个性化的能力。 由于每位用户都有已发布视图的副本，而该视图中会自动应用页面用法（即隐式个性化设置），所以所有用户都不能保存对已发布视图的查询的显式个性化设置或更新（也就是已发布视图已锁定）。 此外，如果为新用户提供了视图发布到的角色，无需管理员执行任何操作，这些用户将自动看到与其角色关联的视图。 同样，如果用户更改了在组织中的角色，则不能访问与其原角色关联的视图，这同样不需要管理员执行任何操作。 可以通过将发布的视图重新发布给相应安全角色，将对该视图的更新轻松分发给用户。

组织可通过此项发布功能以特定安全角色为目标，定义针对自己的业务进行过优化的公司标准视图。  

## <a name="publishing-views"></a>发布视图
发布过程中，可以将视图分配给一个或多个安全角色，这意味着具有该角色的所有用户尽管不能编辑该视图，但是都可以访问和使用该视图。 目前只有系统管理员具有视图选择器下拉菜单中的**发布**操作的权限。  

若要发布视图，请执行以下步骤： 
1.  创建并保存要发布的视图的个人副本。 
2.  当前加载该视图之后，选择视图名称打开视图选择器下拉菜单。 
3.  选择**更多**按钮，然后选择了**发布**。 将打开“发布”对话框。  
4.  为视图提供名称和描述（可选）。 这是收到此视图的用户将在自己的视图选择器中看到的名称。 请注意，不允许页面的已发布视图具有重复名称，即使采用这些视图的角色列表不同也不例外。  
5.  添加与将获取此视图的用户对应的任何安全角色。  
6.  选择**发布**。

请注意，在某些环境中，可能需要过一段时间（最多一小时）之后用户才能看到发布的视图。

## <a name="modifying-a-published-view"></a>修改发布的视图
发布视图后，您可能会发现需要对已发布视图进行更改。 尽管不能实时更改已发布的视图，因为已对所有用户（包括发布者）锁定了这些视图的编辑权限，但是您可以通过重新发布视图来进行更新。  

如果要对已发布视图进行的更改仅涉及发布参数（即视图的名称和描述，或视图要发布给的安全角色），请执行下列操作： 
1.  切换到已发布视图以查看要更新的参数。 
2.  从视图选择器下拉菜单中选择**发布**。 
3.  如果要更新现有视图，请选择**是**（如果要使用其他名称发布此视图，请选择**否**）。
4.  更新视图的名称、描述和/或安全角色。 
5.  选择**发布**。 
6.  如果您更新了已发布视图的名称，则还需要删除采用旧名称的已发布视图（有关更多详细信息，请参阅**管理已发布视图**部分）。 

如果对已发布视图的更改涉及修改与该视图关联的个性化设置或筛选器，请执行以下步骤： 
1.  切换到要修改的已发布视图。 
2.  保存已发布视图的副本以创建这个已发布视图的本地草稿。 
3.  根据所需更改修改本地草稿。
4.  使用原始名称发布视图。 

## <a name="managing-published-views"></a>管理已发布视图
与管理个人视图一样，**管理我的视图**对话框为具有发布权限的用户提供对该页的已发布视图（除了这些用户自己的个人视图）的基本维护功能。 若要打开此页面，请选择视图名称打开视图选择器下拉菜单，选择**更多**，然后选择**管理我的视图**。

虽然所有用户都可以看到其中显示自己的个人视图的**我的视图**选项卡，具有发布权限的用户还可以看到**已发布**选项卡，其中显示为该页发布的所有视图。 因为可能有多位用户在发布视图，所以无论您是不是视图的真正发布用户，可以管理完整的已发布视图列表至关重要。

可对该页面的所有已发布视图列表执行下面的一组操作。 

-    **发布**：可使用**发布**操作和修改后的发布参数（名称、描述、安全角色）重新发布视图。  
-    **删除**：可使用 **删除**操作永久删除已发布的视图。 此操作将删除系统中所有用户的该视图。  
 
此对话框中进行的任何更改都将在选择**保存**按钮之后生效。

## <a name="frequently-asked-questions"></a>常见问题
### <a name="how-do-i-enable-saved-views-in-my-environment"></a>如何启用我的环境中的保存的视图？ 
若要启用保存的视图，系统管理员应该执行以下操作： 
1.  通过导航搜索转到**个性化**页。 
2.  选择**设置**选项卡。
3.  将**启用保存的视图**选项设置为**是**。

启用此功能后，所有后续用户会话开始时都会启用视图。  

请注意，如果为环境关闭了个性化，即使执行上面的步骤，也将启用视图。 这是因为视图建立在个性化子系统的基础上。

### <a name="what-happens-to-existing-personalizations-when-views-are-enabled"></a>启用视图对现有个性化设置有何影响？ 
启用视图后，将把用户和窗体的所有现有个性化设置保存到一个称为**我的视图**的新视图中，这个新视图会自动设置为默认视图。 这是为了确保启用视图前后除了窗体上显示的视图选择器控件之外，用户体验保持一致。  

### <a name="what-pages-support-views"></a>哪些页面支持视图？ 
Finance and Operations 中的大多数页面支持视图，但不是所有页面都支持。 特别是除了仪表板和工作区之外的所有全屏页面现在都支持视图。 非全屏页面（包括对话框、下拉对话框、查找、增强预览）现在还是不支持视图。 将来的更新可能会考虑更多页面类型（如工作区和对话框）支持视图。   

### <a name="who-is-allowed-to-publish-views"></a>允许哪些人发布视图？
现在只有系统管理员有权发布视图。  计划了一种新的安全角色，这会为客户提供谁可以发布方面更大灵活性。  

### <a name="why-am-i-not-able-to-save-filters-with-this-view"></a>我为什么不能随该视图一起保存筛选器？ 
可能似乎无法随视图一起保存筛选器的原因有几个： 

- 页面可能不支持将筛选器保存为视图定义的一部分。 请注意，只有具有大视图选择器的页面才允许将个性化设置和查询修改保存为视图。 有关详细信息，请参阅“切换视图”部分。 

- 如果视图为默认视图，而页面的导航路径中包含查询，则最初可能不会应用该视图的查询。 下面是这种情况的两个主要场景： 
     - 如果从磁贴导航到某个页面，则无论与默认视图关联的是哪个查询，也会执行磁贴查询。 
     - 如果导航到页面，而入口点中包含查询，则最初将执行原始查询，而不是默认视图的查询。 
     
  加载视图时，将通过参考信息通知您发生了这些情况。 也可以通过在页面加载后切换到此视图予以确认，因为这样无论如何都将执行视图查询。  

- 有问题的页面可能不会正确支持视图，因为可能会完全忽略视图查询。 请通过**反馈**机制报告所有此类实例。 若要访问反馈页面，请单击**帮助和支持**，然后单击**反馈**。

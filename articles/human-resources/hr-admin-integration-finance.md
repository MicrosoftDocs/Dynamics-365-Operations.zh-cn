## <a name="integration-considerations"></a><span data-ttu-id="01934-364">集成注意事项</span><span class="sxs-lookup"><span data-stu-id="01934-364">Integration considerations</span></span>
<span data-ttu-id="01934-365">从 Human Resources 到 Finance 集成数据时，集成将尝试根据 ID 匹配记录。</span><span class="sxs-lookup"><span data-stu-id="01934-365">When integrating data from Human Resources to Finance, the integration will attempt to match records based on the ID.</span></span> <span data-ttu-id="01934-366">如果发生匹配，Finance 中的数据将被 Human Resources 中的值覆盖。</span><span class="sxs-lookup"><span data-stu-id="01934-366">If the match occurs, the data in Finance will be overwritten with the values in Human Resources.</span></span> <span data-ttu-id="01934-367">但是，如果逻辑上这些记录是不同的，但根据各自的编号规则在 Human Resources 或 Finance 中生成了相同的 ID，则可能发生问题。</span><span class="sxs-lookup"><span data-stu-id="01934-367">However, an issue may occur if logically these are different records and the same ID was generated in either Human Resources or Finance based on the respective number sequence.</span></span>

<span data-ttu-id="01934-368">可能发生这种情况的区域是“工作人员”和“职位”区域，前者使用人员编号进行匹配。</span><span class="sxs-lookup"><span data-stu-id="01934-368">The areas where this can occur are Worker, which uses Personnel number to make the match, and Positions.</span></span> <span data-ttu-id="01934-369">工作不使用编号规则。</span><span class="sxs-lookup"><span data-stu-id="01934-369">Jobs do not use number sequences.</span></span> <span data-ttu-id="01934-370">结果，如果 Human Resources 和 Finance 中同时存在同一个工作 ID，Human Resources 信息将覆盖 Dynamics 365 Finance 信息。</span><span class="sxs-lookup"><span data-stu-id="01934-370">As a result, if the same job ID is present in both Human Resources and Finance, the Human Resources information will overwrite the Dynamics 365 Finance information.</span></span> 

<span data-ttu-id="01934-371">为防止出现重复 ID 问题，您可以在[编号规则](https://docs.microsoft.com/dynamics365/unified-operations/fin-and-ops/organization-administration/number-sequence-overview?toc=/dynamics365/unified-operations/talent/toc.json)中添加前缀，或在编号规则中设置一个超出其他系统范围的开始编号。</span><span class="sxs-lookup"><span data-stu-id="01934-371">To prevent issues with duplicate IDs, you can either add a prefix on the [number sequence](https://docs.microsoft.com/dynamics365/unified-operations/fin-and-ops/organization-administration/number-sequence-overview?toc=/dynamics365/unified-operations/talent/toc.json), or set a beginning number on the number sequence that is beyond the range of the other system.</span></span>

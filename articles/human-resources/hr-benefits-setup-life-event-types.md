# <a name="configure-life-event-types"></a>配置生活事件类型


[!INCLUDE [PEAP](../includes/peap-2.md)]

[!include [Applies to Human Resources](../includes/applies-to-hr.md)]

Dynamics 365 Human Resources 使用 **生活事件类型** 来定义对更新员工福利登记有效的事件，如结婚或生子。 每个生活事件类型 ID 只能与一个生活事件类型相关联。 例如，如果您创建与生活事件类型 **员工地址更改** 相关联的名为 **地址更改** 的 **生活事件 ID**，您将无法创建另一个标注为 **员工地址更改** 的 ID 并将其与生活事件类型 **员工地址更改** 相关联。 如果生活事件类型未与计划类型关联，该生活事件类型将不会触发生活事件。 有关详细信息，请参阅[创建计划类型](hr-benefits-setup-plan-types.md)。

## <a name="create-a-life-event-type"></a>创建生活事件类型

1. 在 **福利管理** 工作区中，在 **设置** 下，选择 **生活事件类型**。

2. 选择 **新建**。

3. 为以下字段指定值：

   | 字段 | 说明 |
   | --- | --- |
   | **生活事件类型 ID** | 生活事件类型的唯一标识符。 |
   | **说明** | 生活事件类型的描述。 |
   | **生活事件类型** | 促进更新员工福利登记的催化剂。 有关生活事件的列表，请参阅下面的[生活事件](hr-benefits-setup-payment-frequencies.md?life-events)。 |

4. 选择 **保存**。

## <a name="view-attached-plans"></a>查看附加的计划

您可以查看附加到所选 **生活事件类型** 的计划的列表。 生活事件将附加到计划类型，计划类型将与计划相关联。

1. 在 **福利管理** 工作区中，在 **设置** 下，选择 **生活事件类型**。

2. 选择 **操作**。

3. 选择 **附加的计划**。

| **就业状况更改** |**工作人员 > 雇用 > 雇用历史记录页** | 对于具有现有雇用详细信息的工作人员，创建具有不同雇用状态的新雇用详细信息将触发生活事件。  更新具有不同雇用状态的现有雇用详细信息也会触发生活事件。  |
| **员工地址更改** |**工作人员 > 个人资料 > 地址**</li><li>**工作人员 > 个人信息 > 个人联系人 > 地址**</li></ul> | 更改地址。 地址必须是 **主要** 地址才能触发生活事件。 |
| **依赖方更改** |<br><ul><li>**工作人员 > 个人资料 > 个人信息 > 个人联系人**/li><li>**员工自助服务**</li></ul> | 添加个人联系人，将其指定为依赖方并定义 **生效日期**。 更新个人联系人的依赖方 **失效日期** 信息。 个人联系人关系必须是子女、配偶、家庭伴侣或前配偶。  |

| **休假(配偶/家庭伴侣)** | **工作人员 > 个人资料 > 个人信息 > 个人联系人 > 依赖方详细信息 > 休假** | 创建个人联系人和定义 **休假生效日期**。 更新个人联系人 **休假**。 更新个人联系人 **休假生效日期**。  |
| **覆盖范围(职位)更改** |<br><ul><li>**工作人员 > 职位分配 > 工作人员职位分配**</li><li>**职位 > 职位**</li></ul>| 对工作人员职位分配记录中的职位的更改。 职位中工作人员分配的更改。 |
| **覆盖范围(薪金)更改** |<br><ul><li>**工作人员 > 薪酬 > 固定计划**</li><li>**工作人员 > 个人信息 > 福利年薪**</li></ul>| 如果未启用 **福利管理 > 人力资源共享参数 > 福利 > 福利年薪**，更新 **工作人员 > 薪酬 > 固定计划** 将创建生活事件。 如果启用了 **福利管理 > 人力资源共享参数 > 福利 > 福利年薪**，更新 **工作人员 > 个人信息 > 福利年薪** 将创建生活事件。 |
| **医疗保险(员工/依赖方)** | **工作人员 > 个人资料 > 个人信息 > 个人联系人 > 依赖方详细信息 > 医疗保险生效日期** | 为个人联系人添加或更新 **医疗保险生效** 日期会创建此生活事件。 |
| **法院裁定的支持** | **工作人员 > 个人资料 > 个人信息 > 个人联系人 > 依赖方 > 法院裁定的支持** (QMSCO/QDRO) 和生效日期 | 创建个人联系人时，如果 **法院裁定的支持** 为 **是**，将创建生活事件。 更新 **法院裁定的支持** 或 **法院裁定的到期日期** 也会触发生活事件。 |
| **死亡** | **工作人员 > 个人资料 > 个人信息 > 死亡日期** | 输入或更新 **死亡日期**。 |

| **受益人** | **工作人员 > 个人资料 > 个人信息 > 个人联系人** | 个人联系人已添加，**受益人** 和 **生效日期** 字段已填充。 个人联系人必须为 **Child**、**Spouse**、**DomesticPartner**、**Sibling**、**FamilyContact**、**OtherContact** 或 **Parent** 类型。 |
| **员工医疗保险** | **工作人员 > 工作人员 > 版本 > 雇用历史记录 > 日期管理器 > 福利详细信息** | **符合医疗保险资格** 设置为 **是**。 **医疗保险资格日期** 更改。 |

| **资格规则覆盖更改（不特定于美国）** | **人力资源(高级) > 福利 > 计划 > 福利 >资格规则覆盖** | 更新福利计划资格替代中的 **生效日期** 或 **失效日期** 将触发此生活事件。 |

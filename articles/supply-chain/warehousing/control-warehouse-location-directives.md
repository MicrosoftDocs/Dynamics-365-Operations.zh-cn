## <a name="work-templates"></a><span data-ttu-id="fc89e-108">工作模板</span><span class="sxs-lookup"><span data-stu-id="fc89e-108">Work templates</span></span>
<span data-ttu-id="fc89e-109">**工作模板**页可以定义必须在仓库中执行的工作工序。</span><span class="sxs-lookup"><span data-stu-id="fc89e-109">The **Work templates** page lets you define the work operations that must be performed in the warehouse.</span></span> <span data-ttu-id="fc89e-110">通常，仓库工作工序包括一个对操作：仓库工作人员在一个库位领取现有库存量，然后将领取的库存放到另一个库位。</span><span class="sxs-lookup"><span data-stu-id="fc89e-110">Typically, warehouse work operations consist of a pair of actions: a warehouse worker picks up on-hand inventory in one location and then puts the picked inventory down in another location.</span></span> 

<span data-ttu-id="fc89e-111">工作模板包括标题和关联的行。</span><span class="sxs-lookup"><span data-stu-id="fc89e-111">Work templates consists of a header and associated lines.</span></span> <span data-ttu-id="fc89e-112">每个工作模板针对特定*工作订单类型*。</span><span class="sxs-lookup"><span data-stu-id="fc89e-112">Each work template is for a specific *work order type*.</span></span> <span data-ttu-id="fc89e-113">许多工作订单类型与源文档关联，例如采购订单或销售订单。</span><span class="sxs-lookup"><span data-stu-id="fc89e-113">Many work order types are associated with source documents, such as purchase or sales orders.</span></span> <span data-ttu-id="fc89e-114">但是，其他工作订单类型表示单独的仓库流程，例如周期盘点。</span><span class="sxs-lookup"><span data-stu-id="fc89e-114">However, other work order types represent separate warehouse processes, such as cycle counting.</span></span> <span data-ttu-id="fc89e-115">*工作池 ID* 使您可组织工作组。</span><span class="sxs-lookup"><span data-stu-id="fc89e-115">The *work pool ID* lets you organize work into groups.</span></span> 

<span data-ttu-id="fc89e-116">在工作标题定义中的设置可用于确定什么时候应创建一项新的工作。</span><span class="sxs-lookup"><span data-stu-id="fc89e-116">The settings in the work header definition can be used to determine when a new piece of work should be created.</span></span> <span data-ttu-id="fc89e-117">例如，您可以设置最大领料行数和最大预期领料时间。</span><span class="sxs-lookup"><span data-stu-id="fc89e-117">For example, you can set a maximum number of pick lines and a maximum expected pick time.</span></span> <span data-ttu-id="fc89e-118">然后，如果销售订单领料流程的工作超过这些值之一，工作将拆分为两项工作。</span><span class="sxs-lookup"><span data-stu-id="fc89e-118">Then, if the work for a sales order picking process exceeds either of those values, that work is split into two pieces of work.</span></span> 

<span data-ttu-id="fc89e-119">工作行表示处理该工作所需的实际任务。</span><span class="sxs-lookup"><span data-stu-id="fc89e-119">The work lines represent the physical tasks that are required in order to process the work.</span></span> <span data-ttu-id="fc89e-120">例如，对于出货仓库流程，可能存在领取仓库内的物料的工作行，以及将那些物料放入暂存区域的另一个行。</span><span class="sxs-lookup"><span data-stu-id="fc89e-120">For example, for an outbound warehouse process, there might be a work line for picking up the items within the warehouse and another line for putting those items into a staging area.</span></span> <span data-ttu-id="fc89e-121">然后可能存在从暂存领取物料的附加行，以及将物料放入卡车（作为装载流程的一部分）的另一行。</span><span class="sxs-lookup"><span data-stu-id="fc89e-121">There can then be an additional line for picking the items from staging, and another line for putting the items into a truck as part of the loading process.</span></span> <span data-ttu-id="fc89e-122">您可以在工作模板行设置*指令代码*。</span><span class="sxs-lookup"><span data-stu-id="fc89e-122">You can set a *directive code* on work template lines.</span></span> <span data-ttu-id="fc89e-123">指令代码链接到库位指令，因而帮助确保仓库工作在正确的仓库库位中处理。</span><span class="sxs-lookup"><span data-stu-id="fc89e-123">A directive code is linked to a location directive and therefore helps guarantee that the warehouse work is processed in the correct location in the warehouse.</span></span> 

<span data-ttu-id="fc89e-124">您可以设置查询来控制什么时候使用一个特定的工作模板。</span><span class="sxs-lookup"><span data-stu-id="fc89e-124">You can set up a query to control when a particular work template is used.</span></span> <span data-ttu-id="fc89e-125">例如，您可以设置限制，以便特定模板只能在特定仓库中用于工作。</span><span class="sxs-lookup"><span data-stu-id="fc89e-125">For example, you can set a limitation so that a particular template can be used for work only in a specific warehouse.</span></span> <span data-ttu-id="fc89e-126">或者，您可以根据销售来源，具有用于创建出站销售订单处理的工作的若干模板。</span><span class="sxs-lookup"><span data-stu-id="fc89e-126">Alternatively, you might have several templates that are used to create work for outbound sales order processing, depending on the sales origin.</span></span> <span data-ttu-id="fc89e-127">系统使用“**序列号**”字段确定可用工作模板在其中评估的订单。</span><span class="sxs-lookup"><span data-stu-id="fc89e-127">The system uses the **Sequence number** field to determine the order that the available work templates are assessed in.</span></span> <span data-ttu-id="fc89e-128">因此，如果您具有特定工作模板的非常详细的查询，应为其指定小序列号。</span><span class="sxs-lookup"><span data-stu-id="fc89e-128">Therefore, if you have a very specific query for a particular work template, you should give it a low sequence number.</span></span> <span data-ttu-id="fc89e-129">该查询然后在其他更为一般的查询前进行评估。</span><span class="sxs-lookup"><span data-stu-id="fc89e-129">That query will then be evaluated before other, more general queries.</span></span> 

<span data-ttu-id="fc89e-130">若要停止或暂停工作流程，您可以在工作行使用**停止工作**设置。</span><span class="sxs-lookup"><span data-stu-id="fc89e-130">To stop or pause a work process, you can use the **Stop work** setting on the work line.</span></span> <span data-ttu-id="fc89e-131">在这种情况下，执行工作的工作人员将不会被要求执行下一工作行步骤。</span><span class="sxs-lookup"><span data-stu-id="fc89e-131">In that case, the worker who is performing the work won't be asked to perform the next work line step.</span></span> <span data-ttu-id="fc89e-132">若要移至下一个步骤，该工作人员或其他工作人员必须再次选择该工作。</span><span class="sxs-lookup"><span data-stu-id="fc89e-132">To move on to the next step, that worker or another worker must select the work again.</span></span> <span data-ttu-id="fc89e-133">您还可以通过在工作模板行使用不同的*工作类 ID* 来在一项工作内分离任务。</span><span class="sxs-lookup"><span data-stu-id="fc89e-133">You can also separate the tasks within a piece of work by using a different *work class ID* on the work template lines.</span></span>

## <a name="location-directives"></a><span data-ttu-id="fc89e-134">位置指令</span><span class="sxs-lookup"><span data-stu-id="fc89e-134">Location directives</span></span>
<span data-ttu-id="fc89e-135">库位指令是帮助标识领料和使位置用于库存移动的规则。</span><span class="sxs-lookup"><span data-stu-id="fc89e-135">Location directives are rules that help identify pick and put locations for inventory movement.</span></span> <span data-ttu-id="fc89e-136">例如，在销售订单交易记录中，位置指令决定在何处领物料，及将已领的物料放置在何处。</span><span class="sxs-lookup"><span data-stu-id="fc89e-136">For example, in a sales order transaction, a location directive determines where the items will be picked, and where the picked items will be put.</span></span> <span data-ttu-id="fc89e-137">库位指令包括标题和关联行，您在**库位指令**页上创建它们。</span><span class="sxs-lookup"><span data-stu-id="fc89e-137">Location directives consist of a header and associated lines, and you create them on the **Location directives** page.</span></span> 

<span data-ttu-id="fc89e-138">在标题上，必须将每个库位指令与指定指令将用于的库存交易记录类型的*工作订单类型*关联，如销售订单、补货或原材料领取。</span><span class="sxs-lookup"><span data-stu-id="fc89e-138">On the header, each location directive must be associated with a *work order type* that specifies the type of inventory transaction that the directive will be used for, such as sales orders, replenishment, or raw material picking.</span></span> <span data-ttu-id="fc89e-139">*工作类型*指定库位指令是否将用于领取或放置工作，或者用于某些其他仓库流程，如盘点或库存状态更改。</span><span class="sxs-lookup"><span data-stu-id="fc89e-139">The *work type* specifies whether the location directive will be used for picking or putting work, or for some other warehouse process, such as counting or inventory status changes.</span></span> <span data-ttu-id="fc89e-140">您还必须指定*站点*和*仓库*。</span><span class="sxs-lookup"><span data-stu-id="fc89e-140">You must also specify a *site* and a *warehouse*.</span></span> <span data-ttu-id="fc89e-141">您在标题上指定的*指令代码*可以用于将库位指令与一个或多个工作模板链接。</span><span class="sxs-lookup"><span data-stu-id="fc89e-141">A *directive code* that you specify on the header can be used to link the location directive to one or more work templates.</span></span>

<span data-ttu-id="fc89e-145">库位指令行设置对库位查找规则应用的其他限制。</span><span class="sxs-lookup"><span data-stu-id="fc89e-145">The location directive lines set additional restrictions on the application of the location finding rules.</span></span> <span data-ttu-id="fc89e-146">您可以指定该指令应该用于的最小数量和最大数量，并且，您可以指定该指令应该针对特定库存单位。</span><span class="sxs-lookup"><span data-stu-id="fc89e-146">You can specify a minimum quantity and a maximum quantity that the directive should apply to, and you can specify that the directive should be for a specific inventory unit.</span></span> <span data-ttu-id="fc89e-147">例如，如果度量单位为托盘，则可将托盘中的物料放入指定位置中。</span><span class="sxs-lookup"><span data-stu-id="fc89e-147">For example, if the unit of measure is pallets, the items in pallets can be put in a specific location.</span></span> <span data-ttu-id="fc89e-148">您还可以指定数量是否可以跨多个库位拆分。</span><span class="sxs-lookup"><span data-stu-id="fc89e-148">You can also specify whether the quantity can be split across multiple locations.</span></span> <span data-ttu-id="fc89e-149">与库位指令标题相似，每个库位指令具有一个确定在其中评估行的订单的序列号。</span><span class="sxs-lookup"><span data-stu-id="fc89e-149">Like the location directive header, each location directive line has a sequence number that determines the order that the lines are assessed in.</span></span>

### <a name="example-of-the-use-of-location-directives"></a><span data-ttu-id="fc89e-155">库位指令使用示例</span><span class="sxs-lookup"><span data-stu-id="fc89e-155">Example of the use of location directives</span></span>

<span data-ttu-id="fc89e-156">对于本示例，我们将考虑采购订单流程，在此流程中库位指令必须为已在收货台登记的库存物料在仓库内查找可用容量。</span><span class="sxs-lookup"><span data-stu-id="fc89e-156">For this example, we will consider a purchase order process where the location directive must find free capacity within a warehouse for inventory items that have just been registered at the receiving dock.</span></span> <span data-ttu-id="fc89e-157">首先，我们希望尝试通过与现有库存量合并来在仓库内查找可用容量。</span><span class="sxs-lookup"><span data-stu-id="fc89e-157">First, we want to try to find free capacity within the warehouse by consolidating with existing on-hand inventory.</span></span> <span data-ttu-id="fc89e-158">如果合并是不可能的，我们然后要查找空库位。</span><span class="sxs-lookup"><span data-stu-id="fc89e-158">If consolidation isn't possible, we then want to find an empty location.</span></span> 

<span data-ttu-id="fc89e-159">对于此情况，我们必须定义两个库位指令操作。</span><span class="sxs-lookup"><span data-stu-id="fc89e-159">For this scenario, we must define two location directive actions.</span></span> <span data-ttu-id="fc89e-160">序列中的第一个操作必须使用**合并**策略，第二个应使用**没有传入工作的空库位**策略。</span><span class="sxs-lookup"><span data-stu-id="fc89e-160">The first action in the sequence must use the **Consolidate** strategy, and the second should use the **Empty location with no incoming work** strategy.</span></span> <span data-ttu-id="fc89e-161">除非我们定义第三个操作来处理溢出情况，否则在仓库中没有更多容量时，有两个可能的结果：即使未定义库位也可以创建工作，或者创建工作流程可能失败。</span><span class="sxs-lookup"><span data-stu-id="fc89e-161">Unless we define a third action to handle an overflow scenario, two outcomes are possible when there is no more capacity in the warehouse: work can be created even though no locations are defined, or the work creation process can fail.</span></span> <span data-ttu-id="fc89e-162">结果将由**库位指令失败**页上的设置确定，在此页面您可以决定是否为每个工作订单类型选择**在库位指令失败时停止工作**选项。</span><span class="sxs-lookup"><span data-stu-id="fc89e-162">The outcome is determined by the setup on the **Location directive failures** page, where you can decide whether to select the **Stop work on location directive failure** option for each work order type.</span></span>

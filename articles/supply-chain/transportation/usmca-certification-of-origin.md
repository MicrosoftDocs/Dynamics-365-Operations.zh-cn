# <a name="usmca-certification-of-origin"></a>USMCA 原产地认证

[!include [banner](../includes/banner.md)]

使用此功能，您可以打印 United States-Mexico-Canada Agreement (USMCA) 要求的原产地认证文档。

*USMCA 原产地认证文档* 包含申报所需要的最基本的数据元素。 一些数据元素可以在打印之前预填充，另一些则必须在打印之后手动填充。 为了获得优惠的关税待遇，必须完成此文档，并在进行申报时此文档必须为进口商所有。 此文档可以由进口商、出口商或生产商完成。

您可以从 **所有装运** 列表页或 **装运详细信息** 页为单次装运打印此文档。

仅当法人的主要地址中的国家/地区是美国时，此文档才能够访问。

根据文档打印选择，此文档可以使用系统中的数据预填充。 可以通过将打印文档导出为可编辑格式（如 Microsoft Word）更改或向打印文档添加数据。 导出后，您可以在申报之前进行任何必需的更改。

## <a name="turn-on-the-usmca-feature"></a>开启 USMCA 功能

USMCA 功能只有在系统中开启之后才能使用。 管理员可以使用[功能管理](../../fin-ops-core/fin-ops/get-started/feature-management/feature-management-overview.md)设置检查功能状态和开启功能。 在 **功能管理** 工作区中，此功能按照下面的方式列出：

- **模块**：*运输管理*
- **功能名称**：*USMCA 原产地认证文档*

## <a name="document-content"></a>文档内容

USMCA 原产地认证文档包含以下数据元素：

- 地址元素
- 认证方的角色
- 单次装运
- 账单
- 总期间
- 项目详细信息
- 认证人签名
- 页数

有关这些元素中每个元素以及如何找到元素的值的详细信息，请参阅本主题的其余几节内容。

## <a name="print-a-usmca-certification-of-origin-document"></a>打印 USMCA 原产地认证文档

要为装运打印 USMCA 原产地认证文档，请执行以下操作：

1. 执行以下选项之一：
    - 转到 **运输管理 > 装运 > 所有装运**，选择要为其打印文档的装运。
    - 打开您要为其打印文档的装运的 **装运详细信息** 页（有多种方法可以访问此页，包括从 **所有装运** 页）。
1. 在操作窗格上，打开 **装运** 选项卡，从 **打印** 组，选择 **USMCA 原产地证书**。
1. **原产地证书** 对话框将打开。 进行以下小节中所述的设置，然后选择 **确定** 生成文档。
1. 文档预览将打开。 根据需要使用操作窗格上提供的命令打印或导出文档。

### <a name="certifying-party"></a>认证方

使用 **认证方** 下拉列表确定在打印文档的当事方的类型。 指定认证方是 *出口商*、*出口商和生产商*、*生产商* 还是 *进口商*；如果都不是，则留为空白。 您选择的选项确定在文档的地址部分打印的内容。

您选择的 **认证方** 将包含在打印文档中。

入站和出站装运均可打印此文档。 入站装运只能选择 *进口商* 作为 **认证方**。

### <a name="has-various-producers"></a>具有不同的生产商

**认证方** 下拉列表控制要在文档中用于生产商详细信息的文本。 选择以下各项之一：

- *多个生产商* - 在生产商详细信息中打印文本“多个”。
- *按要求提供* - 在生产商详细信息中打印文本“按进口机构的要求提供”。

当 **认证方** 设置为 *出口商和生产商* 或 *生产商* 时，**有多个生产商** 设置将被取消，生产商地址详细信息将与验证人相同。

### <a name="blanket-period"></a>总期间

使用 **总期间起始** 和 **总期间截止** 设置建立总期间，在此期间，即使文档仅为一个装运打印，也会包含相同商品的多个装运。 您可以无任何约束地设置总期间日期，它将被添加到文档中。 您也可以将这些设置留为空白或设置在过去。

### <a name="is-single-shipment"></a>单次装运

在 **原产地证书** 对话框中，将 **单次装运** 设置为以下值之一：

- *是* - 在发票编号旁边添加“单次装运: 是”。
- *否* - 不添加任何内容。

## <a name="other-information-included-in-the-document"></a>文件中包含的其他信息

除了您使用 **原产地证书** 对话框选择的可选元素之外，USMCA 原产地认证文档还将包括以下小节中总结的信息和自定义字段。 有些信息必须在您生成文档后手动输入。

### <a name="invoice-number"></a>发票编号

不论总期间如何，与装运相关的销售发票的 ID 都会打印在文档上。 不论如何选择 **单次装运**，都会打印发票编号。

### <a name="item-details"></a>项目详细信息

文档提供几个列出特定物料详细信息的部分，它们是：

- **SKU 编号**：打印已发布产品的物料编号。

- **描述**：打印已发布产品的描述或名称。 如果存在使用用户语言的描述，将打印此描述。 如果不存在这样的描述，将以用户语言打印名称。 如果此名称不存在，将打印物料名称。

- **协调制度 (HS) 关税分类**：打印与产品相关的协调关税表。 您可以转到 **运输管理 \> 设置 \> 运输标准 \> 协调关税表** 来设置这些表。

- **原产地标准：** 首次发布文档时，必须在此部分中手动输入数据。

- **原产国家/地区：** 打印原产国家/地区，您可以转到 **产品信息管理 \> 设置 \> 产品合规 \> 原产国家/地区** 来应用（另请参阅[原产国家/地区](../pim/country-of-origin.md)）。 原产国家/地区的 ISO 代码根据装运交货地址和物料中的目的地国家/地区打印。 如果未设置原产国家/地区数据，此值将恢复为 **已发布产品 \> 外贸 \> 原产地** 处的设置。 如果仍然找不到原产国家/地区数据，则必须在生成文档后手动输入原产国家/地区。

### <a name="certifier-signature-and-date"></a>认证人签名和日期

您必须在生成文档后手动输入此信息。

### <a name="consists-of-number-of-pages"></a>包含页数

签署认证的用户必须在生成文档后手动输入页数（用于验证）。

### <a name="page-numbers"></a>页码

打印在文档底部的当前页和页数。

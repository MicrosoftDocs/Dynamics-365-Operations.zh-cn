# <a name="manage-changes-to-engineering-products"></a>管理工程产品的更改

[!include [banner](../includes/banner.md)]

工程更改管理提供用于管理工程产品的更改的结构化流程。 您可以使用 *工程更改请求* 流程以提出和请求更改，然后使用 *工程更改订单* 路程以实际作出这些更改。 用户可以创建工程更改请求或工程更改订单，然后有一个流程来审查和批准它们，评估它们对现有交易的影响，并对其进行跟进。

## <a name="engineering-change-requests"></a>工程更改请求

工程更改请求流程可让您捕获来自公司中所有相关部门的更改请求。 无论是作为工程师工作还是在制造、采购、仓库或销售部门中工作都无关紧要：任何人都可以使用工程更改请求来请求更改。 此更改可能是关于新产品的想法，您在使用现有产品时发现的问题，对现有产品进行改进的建议等等。

有人提交更改请求后，*审查和审批* 流程由确定谁必须批准更改（例如产品所有者）的工作流管理。

若要为工程更改订单或工程更改请求设置工作流，请转到 **工程更改管理 \> 工程工作流**。 选择 **新建**，选择工作流将用于检查工程更改订单还是工程更改请求，然后配置工作流。

有关工作流的详细信息，请参阅[工作流系统概述](../../fin-ops-core/fin-ops/organization-administration/overview-workflow-system.md)。 有关产品所有者的详细信息，请参阅[产品所有者](product-owner.md)。

### <a name="create-a-new-engineering-change-request"></a>创建新的工程更改请求

若要创建工程更改请求，请执行以下步骤之一。

- 转到 **工程更改管理 \> 通用 \> 工程更改管理 \> 工程更改请求**，然后在操作窗格上选择 **新建**。
- 打开现有工程产品的 **产品详细信息** 页面。 然后，在操作窗格的 **工程师** 选项卡上，在 **工程更改管理** 组中，选择 **工程更改请求 \> 新工程更改请求**。

创建新的更改请求。 现在，您可以在每个快速选项卡上设置字段，如以下子部分中所述。

#### <a name="general-fasttab"></a>常规快速选项卡

**常规** 快速选项卡可让您提供更改请求的基本描述。 下表介绍此快速选项卡上的字段。

| 字段 | 说明 |
|---|---|
| 更改请求 | 输入工程更改请求的名称。 |
| 岗位名称 | 输入简短描述或标识请求中的更改的文本。 |
| 优先级 | 选择一个值来指示更改的优先级有多高。 您可以根据需要自定义公司的可用值。 （有关详细信息，请参阅[为工程更改管理建立通用值](engineering-change-management-setup.md)。） |
| 类别 | 选择一个值来描述您所请求的更改类型。 您可以根据需要自定义公司的可用值。 （有关详细信息，请参阅[为工程更改管理建立通用值](engineering-change-management-setup.md)。） |
| 严重性 | 选择一个值来指示应通过实施请求解决的问题的严重性。 您可以根据需要自定义公司的可用值。 （有关详细信息，请参阅[为工程更改管理建立通用值](engineering-change-management-setup.md)。） |
| 申请者 | 创建请求的用户的名称。 |
| 在值 | 请求的创建日期。 |
| 状态 | 请求的状态。 首次创建请求时，状态为 *已创建*。 批准请求后，状态更改为 *已批准*。 如果已为请求创建了相关的更改订单，状态将更改为 *已跟进*。 |
| 更改订单 | 更改订单编号（如果已通过更改订单跟进更改请求）。 |

#### <a name="information-fasttab"></a>信息快速选项卡

**信息** 快速选项卡可让您添加有关请求的详细信息。

若要将行添加到网格，请选择网格上方工具栏上的 **新建**，然后选择以下选项之一：

- **文件** – 上传文件。
- **图像** – 上传图像文件。
- **注释** – 直接在网格中输入注释。
- **URL** – 直接在网格中输入 URL。

下表介绍每个行上的字段。

| 字段 | 说明 |
|---|---|
| 创建日期和时间 | 创建行的日期和时间。 |
| 类型 | 为该行创建的信息类型（文件、图像、注释或 URL）。 |
| 说明 | 输入行的描述。 |
| 限制 | 指示已添加的信息是来自内部还是外部源的值。 |
| 已附加 | 选中的复选框表示该行包含附件（文件或图像）。 若要下载附件，请选择该行，然后选择网格上方工具栏上的 **打开**。 |

#### <a name="products-fasttab"></a>产品快速选项卡

**产品** 快速选项卡可让您列出受更改请求影响的每个产品。 您可以使用工具栏上的按钮将产品添加到网格或删除产品。

此列表仅供参考。 因此，您可以添加尽可能多的您认为相关的产品。 如果您从现有产品的 **产品详细信息** 页面中更改请求，在保存请求记录后，该产品应在 **产品** 快速选项卡上列出。

#### <a name="source-fasttab"></a>源快速选项卡

**源** 快速选项卡可让您跟踪更改请求的起点。 例如，如果您想查看更改请求是否是从销售订单中创建的，创建者是谁以及创建它的公司，这很有用。

### <a name="evaluate-the-business-impact-of-a-change-request"></a>评估更改请求的业务影响

审查更改请求时，可以搜索依赖项。 这样，您可以评估请求的更改对未结交易（例如销售订单、生产订单和现有库存）的影响。

1. 转到 **工程更改管理 \> 通用 \> 工程更改管理 \> 工程更改请求**。
1. 打开现有的更改请求，或在操作窗格上选择 **新建** 以创建新的更改请求。
1. 在操作窗格的 **更改请求** 选项卡上，在 **业务影响** 组中，选择以下按钮之一：

    - **搜索** – 扫描所有未结交易，然后打开 **对未结交易的业务影响** 对话框，其中列出了将受更改影响的所有交易。
    - **查看上一个搜索** – 打开 **对未结交易的业务影响** 对话框，其中列出了上一个搜索的结果。 （新搜索未完成。）

1. 如果发现了需要更改的关键问题，您可以在 **对未结交易的业务影响** 对话框中使用工具栏上的按钮阻止未结交易或通知负责的用户。

### <a name="create-a-change-order-from-a-change-request"></a>根据更改请求创建更改订单

审查工程更改请求的工程师可以直接从 **工程更改请求** 页面创建工程更改订单。 在操作窗格的 **更改请求** 选项卡上，在 **工程更改订单** 组中，选择 **复制链接和产品**。

## <a name="engineering-change-orders"></a>工程更改订单

工程更改订单提供用于对工程产品作出更改的结构化流程。 您使用与工程相关的数据的副本提出更改。 实际主数据不受影响。 有关与工程相关的数据的详细信息，请参阅[工程版本和工程产品类别](engineering-versions-product-category.md)。

您可以创建基于批准的工程更改请求的工程更改订单。 工程师还可以从头开始创建工程更改订单。 通过以下任何步骤，您可以在单个工程更改订单上包括多个产品：

- 手动选择产品。
- 使用物料清单 (BOM) 来包括产品结构中较低的产品（即子产品）。
- 使用一个使用位置搜索来包括产品结构中较高的产品（即父产品）。

完成更改提出后，将通过工作流处理审查和审批流程。 您可以根据优先级和严重性设置不同的工作流。

若要为工程更改订单或工程更改请求设置工作流，请转到 **工程更改管理 \> 工程工作流**。 选择 **新建**，选择工作流将用于检查工程更改订单还是工程更改请求，然后配置工作流。

有关工作流的详细信息，请参阅[工作流系统概述](../../fin-ops-core/fin-ops/organization-administration/overview-workflow-system.md)。

以下是一些可能需要批准工程更改订单的典型利益干系人：

- **产品所有者** – 有关产品所有者的详细信息，请参阅[产品所有者](product-owner.md)。
- **团队负责人** – 工程更改订单的 **标题** 视图中的 **工程师** 字段显示创建了工程更改订单的工程师。 如果工程师属于在系统中定义的团队，**负责** 字段将显示该团队的负责人。
- **财务部门** – 财务部门可能需要审查更改涉及高成本的情况。

您可以选择是在工程更改订单获得批准后直接对其进行处理（作为工作流的一部分），还是应在以后进行处理（作为手动步骤）。 在处理工程更改订单期间，将更新实际产品上的工程数据。

在审查更改请求时，在操作窗格的 **更改请求** 选项卡上，在 **业务影响** 组，选择 **搜索** 以评估提出的更改对未结交易（例如销售订单、生产订单和现有库存）的影响。 结果显示在 **对未结交易的业务影响** 对话框中，您可以在其中选择受影响的交易，然后使用工具栏中的命令查看详细信息，通知负责的用户或阻止交易。

### <a name="engineering-change-orders-in-engineering-or-operational-companies"></a>工程或运营公司中的工程更改订单

如[工程公司和数据所有权规则](engineering-org-data-ownership-rules.md)中所述，您可以编辑的产品数据会有所不同，具体取决于您所工作的法人类型（是工程公司还是运营公司）。 数据所有权规则也适用于工程更改订单。 因此，根据您创建工程更改订单的法人，可以作出不同类型的更改。 下面举了一些示例加以说明：

- 对于 **工程公司** 中的工程更改订单，您可以对工程数据进行基本更改。 例如，您可以创建产品的新版本，通过 BOM 更改产品的结构，以及更改工程属性值。 对于每个受影响的产品，请在 **影响** 字段中选择以下值之一：

    - **无** – 更新现有产品版本（版本内更新）。
    - **新版本** – 创建基于所选产品版本的新版本。
    - **新产品** – 创建全新的产品或基于所选产品版本的产品变型。

- 对于 **经营公司** 中的工程更改订单，您可以更改产品的物流数据。 例如，您可以使用用于采购的设置扩充现有 BOM，添加本地工艺路线或本地 BOM，甚至可以通过添加用于本地包装材料、润滑液或本地语言说明的新 BOM 行来扩充 BOM。 当工程公司发送新更新时，将保留用户在运营公司中进行的扩充。 有关详细信息，请参阅[工程公司和数据所有权规则](engineering-org-data-ownership-rules.md)。

    在工程公司中处理工程更改订单时，仅在工程公司中创建和/或更新产品。 因此，如果还应更新产品主数据，则还必须将产品发布给运营公司。

- 您可以直接从工程更改订单发布产品。 打开更改订单，然后在操作窗格的 **更改订单** 选项卡上，在 **产品发布** 组中，选择 **发布产品结构**。 该流程的工作原理与您从 **已发布产品** 页面发布产品一样。 有关详细信息，请参阅[发布产品结构](release-product-structure.md)。
- 您可以根据以下因素从工程更改订单中自动发布产品：

    - 向以前发布了产品的公司重新发布。 选择 **搜索** 以扫描所有以前的版本，然后选择 **视图** 以查看结果。 **视图** 页面显示以前发布的产品，您可以选择要重新发布的产品。 然后关闭 **视图** 页面，选择 **处理** 以重新发布所选产品。
    - 工程产品类别的发布控制中的自动发布设置。 您可以执行此发布作为工作流的一部分。 在使用 **收集发布建议** 块时，如果在工程产品类别的发布控制中选中 **自动发布** 复选框，将使用重新发布建议填充发布建议（请参阅前面的列表项）并将产品发布到公司。 您可以选择 **视图** 以查看结果，如上一个列表项所述。 在使用 **处理发布建议** 块时，也将发布产品。 如果您仅选择收集发布建议作为工作流的一部分，可以通过选择 **处理** 手动启动发布，如上一个列表项所述。

## <a name="follow-up-on-an-engineering-change-request-via-an-engineering-change-order"></a>通过工程更改订单跟进工程更改请求

一旦工程更改请求获得批准，您可以立即通过工程更改订单跟进它。 您可以将多个工程更改请求合并到单个工程更改订单中。 单个工程更改订单甚至可以包含多个产品。 （通常，在必须将相同的更改应用到多个产品时使用此方法。）但是，不能从单个工程更改请求中创建多个工程更改订单。

若要通过更改订单跟进更改请求，请打开更改请求，然后在操作窗格的 **更改订单** 选项卡上，在 **工程更改订单** 组中，选择 **复制链接和产品**。 然后，您可以选择更改请求要连接到的现有工程更改订单，或者可以为该特定请求创建新的工程更改订单。

## <a name="engineering-change-order-report"></a>工程更改订单报表

工程更改订单报告描述了工程更改订单中所作的更改。 它们在审查和审批期间以及之后都非常有用。

若要查看工程更改订单报告，请打开相关的更改订单，然后在操作窗格的 **更改订单** 选项卡上，在 **视图** 组中，选择 **工程更改订单报告**。

## <a name="fields-on-an-engineering-change-order"></a>工程更改订单上的字段

工程更改订单上的大多数字段与已发布产品、工程版本、凭证、BOM（行）和工艺路线（工序）的字段相同。 但是，下表中的字段对于更改订单是唯一的。

| 字段 | 说明 |
|---|---|
| 工程更改原因 | 选择更改受影响产品的原因。 |
| 更改描述 | 输入更改的描述。 |
| 必需特殊工具 | 指定是否需要特殊工具来应用更改。 |
| 工程材料处置 | 选择应用更改后产生的任何废物的材料处置代码。 |
| 需要客户审核 | 指定在应用更改之前是否需要客户审批。 |
| 收到的客户审核 | 指定客户审批的状态。 |
| 环境健康和安全 | 指定环境健康和安全规则是否适用于更改。 如果适用，则可以选择适用的规则。 |

您可以使用 **维护/复制更改信息** 按钮在受影响的产品之间复制更改信息。

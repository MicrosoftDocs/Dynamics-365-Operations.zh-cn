<span data-ttu-id="d1bc6-109">设置成本计算单涉及两个目标。</span><span class="sxs-lookup"><span data-stu-id="d1bc6-109">Setting up the costing sheet involves two objectives.</span></span> <span data-ttu-id="d1bc6-110">作为第一个目标，您定义用于显示与制造物料或生产订单有关的所售货物成本信息的格式。</span><span class="sxs-lookup"><span data-stu-id="d1bc6-110">As the first objective, you define the format for displaying cost of goods sold information about a manufactured item or production order.</span></span> <span data-ttu-id="d1bc6-111">采用此格式的显示称作成本计算单。</span><span class="sxs-lookup"><span data-stu-id="d1bc6-111">The formatted display is termed a costing sheet.</span></span> <span data-ttu-id="d1bc6-112">作为第二个目标，您定义用于计算间接成本的基数。</span><span class="sxs-lookup"><span data-stu-id="d1bc6-112">As the second objective, you define the basis for calculating indirect costs.</span></span> <span data-ttu-id="d1bc6-113">成本计算单设置建立在用于显示信息和用于间接成本计算公式的成本组功能的基础上。</span><span class="sxs-lookup"><span data-stu-id="d1bc6-113">The costing sheet setup builds on the cost group feature for displaying information and for the indirect cost calculation formulas.</span></span> <span data-ttu-id="d1bc6-114">本文介绍两个成本计算单设置的目标。</span><span class="sxs-lookup"><span data-stu-id="d1bc6-114">The two objectives of costing sheet setup are described in this article.</span></span> 

<span data-ttu-id="d1bc6-115">成本计算单是格式化的信息显示，有关为制造物料或生产订单销售的货物的成本。</span><span class="sxs-lookup"><span data-stu-id="d1bc6-115">A costing sheet is the formatted display of information about the cost of goods that are sold for a manufactured item or a production order.</span></span> <span data-ttu-id="d1bc6-116">在您设置成本计算单时，您可以定义信息的格式并定义用于计算间接成本的基础。</span><span class="sxs-lookup"><span data-stu-id="d1bc6-116">When you set up a costing sheet, you define the format for the information and also define the basis for calculating indirect costs.</span></span> <span data-ttu-id="d1bc6-117">成本计算单设置建立在用于显示信息和用于间接成本计算公式的成本组功能的基础上。</span><span class="sxs-lookup"><span data-stu-id="d1bc6-117">The costing sheet setup builds on the cost group features for displaying information and for the formulas that are used to calculated indirect cost.</span></span> <span data-ttu-id="d1bc6-118">下面是有关成本计算单设置的两个目的的详细信息：</span><span class="sxs-lookup"><span data-stu-id="d1bc6-118">Here is more information about the two objectives of costing sheet setup:</span></span>
-   <span data-ttu-id="d1bc6-119">**定义成本计算单的格式。**</span><span class="sxs-lookup"><span data-stu-id="d1bc6-119">**Define the format for the costing sheet.**</span></span> <span data-ttu-id="d1bc6-120">成本计算单的用户定义的格式标识包含制造物料的所售货物成本的成本细分。</span><span class="sxs-lookup"><span data-stu-id="d1bc6-120">The user-defined format for a costing sheet identifies the segmentation of costs that contain a manufactured item’s cost of goods sold.</span></span> <span data-ttu-id="d1bc6-121">例如，物料的所售货物成本信息可以基于成本组细分为材料、人工和开销。</span><span class="sxs-lookup"><span data-stu-id="d1bc6-121">For example, the information about an item’s cost of goods sold can be segmented into material, labor, and overhead, based on cost groups.</span></span> <span data-ttu-id="d1bc6-122">将这些成本组分配给物料、工艺路线工序的成本类别以及间接成本计算公式。</span><span class="sxs-lookup"><span data-stu-id="d1bc6-122">These cost groups are assigned to items, cost categories for routing operations, and indirect cost calculation formulas.</span></span> <span data-ttu-id="d1bc6-123">在已定义多个成本组时，成本计算单的格式通常需要中间合计。</span><span class="sxs-lookup"><span data-stu-id="d1bc6-123">The format for the costing sheet typically requires intermediate totals when multiple cost groups have been defined.</span></span> <span data-ttu-id="d1bc6-124">例如，与材料相关的多个成本组可以聚合。</span><span class="sxs-lookup"><span data-stu-id="d1bc6-124">For example, multiple cost groups that are related to material can be aggregated.</span></span> <span data-ttu-id="d1bc6-125">成本计算单格式的定义是可选的，但如果将计算间接成本，则必须定义成本计算单格式。</span><span class="sxs-lookup"><span data-stu-id="d1bc6-125">The definition of a costing sheet format is optional, but a costing sheet format must be defined if indirect costs will be calculated.</span></span>
-   <span data-ttu-id="d1bc6-126">**为计算间接成本定义基数。**</span><span class="sxs-lookup"><span data-stu-id="d1bc6-126">**Define the basis for calculating indirect costs.**</span></span> <span data-ttu-id="d1bc6-127">间接成本反映与生产某一制造物料相关联的制造开销。</span><span class="sxs-lookup"><span data-stu-id="d1bc6-127">Indirect costs reflect manufacturing overhead that is associated with the production of a manufactured item.</span></span> <span data-ttu-id="d1bc6-128">间接成本计算公式可以表示为附加费或费率。</span><span class="sxs-lookup"><span data-stu-id="d1bc6-128">An indirect cost calculation formula can be expressed as either a surcharge or a rate.</span></span> <span data-ttu-id="d1bc6-129">附加费表示某一值的百分比，而费率表示某一工艺路线工序的每工时金额。</span><span class="sxs-lookup"><span data-stu-id="d1bc6-129">A surcharge represents a percentage of value, whereas a rate represents an amount per hour for a routing operation.</span></span> <span data-ttu-id="d1bc6-130">成本组定义用于计算公式的基数；例如，100 个百分比的附加费针对人工成本组，而 USD 50.00 的每工时费率针对机器成本组。</span><span class="sxs-lookup"><span data-stu-id="d1bc6-130">A cost group defines the basis for the calculation formula, such as a 100-percent surcharge for a labor cost group or a USD 50.00 hourly rate for a machine cost group.</span></span> <span data-ttu-id="d1bc6-131">如果您想要定义计算公式及其成本组基数，成本计算单设置要求您标识表示开销的成本组，并选择是否使用附加费或费率方法。</span><span class="sxs-lookup"><span data-stu-id="d1bc6-131">If you want to define a calculation formula and its cost group basis, the costing sheet setup requires that you identify the cost group that represents the overhead, and select whether a surcharge or rate approach is used.</span></span>

<span data-ttu-id="d1bc6-132">每个计算公式都必须作为某一成本记录输入。</span><span class="sxs-lookup"><span data-stu-id="d1bc6-132">Each calculation formula must be entered as a cost record.</span></span> <span data-ttu-id="d1bc6-133">该成本记录由指定的成本计算版本、附加费百分比（或费率金额）、成本组基数、状态和生效日期构成。</span><span class="sxs-lookup"><span data-stu-id="d1bc6-133">The cost record consists of a specified costing version, a surcharge percentage or a rate amount, the cost group basis, a status, and an effective date.</span></span> <span data-ttu-id="d1bc6-134">当初次输入成本记录时，它具有 **“待定”** 状态和预期生效日期。</span><span class="sxs-lookup"><span data-stu-id="d1bc6-134">When a cost record is first entered, it has **Pending** status and an effective date.</span></span> <span data-ttu-id="d1bc6-135">当您激活成本记录时，状态将更新以便该记录是当前有效的记录，而且生效日期将更新为激活日期。</span><span class="sxs-lookup"><span data-stu-id="d1bc6-135">When you activate the cost record, the status is updated to so that the record is the current active record, and the effective date is updated to the activation date.</span></span> <span data-ttu-id="d1bc6-136">成本记录还可以为特定于站点的计算公式指定站点。</span><span class="sxs-lookup"><span data-stu-id="d1bc6-136">The cost record can also specify a site for a site-specific calculation formula.</span></span> <span data-ttu-id="d1bc6-137">另外，您可以将 **“站点”** 字段留空以指示计算公式是公司范围的公式。</span><span class="sxs-lookup"><span data-stu-id="d1bc6-137">Alternatively, you can leave the **Site** field blank to indicate that the calculation formula is a company-wide formula.</span></span> <span data-ttu-id="d1bc6-138">在计算公式已标记为按物料计算的公式时，成本记录可以（可选）由指定的物料或物料组构成。</span><span class="sxs-lookup"><span data-stu-id="d1bc6-138">The cost record can optionally consist of a specified item or item group when the calculation formula has been marked as a per-item formula.</span></span> 

<span data-ttu-id="d1bc6-139">间接成本计算公式的当前有效成本记录用于评估生产订单成本。</span><span class="sxs-lookup"><span data-stu-id="d1bc6-139">The current active cost records for indirect cost calculation formulas are used to estimate production order costs.</span></span> <span data-ttu-id="d1bc6-140">它们还用于计算与时间和材料实际消耗量有关的实际成本。</span><span class="sxs-lookup"><span data-stu-id="d1bc6-140">They are also used to calculate actual costs that are related to actual consumption of time and material.</span></span> <span data-ttu-id="d1bc6-141">未决成本记录用于将来日期的物料清单计算中。</span><span class="sxs-lookup"><span data-stu-id="d1bc6-141">Pending cost records are used in bill of materials (BOM) calculations for a future date.</span></span> 

<span data-ttu-id="d1bc6-142">用于成本计算版本的两个冻结策略确定是否可维护未决成本以及是否启用未决成本。</span><span class="sxs-lookup"><span data-stu-id="d1bc6-142">Two blocking policies for a costing version determine whether pending costs can be maintained, and whether the pending cost can be started.</span></span> <span data-ttu-id="d1bc6-143">使用冻结策略可以允许数据维护，然后为某一成本计算版本内的成本数据阻止数据维护。</span><span class="sxs-lookup"><span data-stu-id="d1bc6-143">Use the blocking policies to permit data maintenance, and then to prevent data maintenance for the cost data in a costing version.</span></span> 

<span data-ttu-id="d1bc6-144">在您为间接成本定义成本计算单格式和计算后，必须执行单独的步骤来验证和保存该信息。</span><span class="sxs-lookup"><span data-stu-id="d1bc6-144">After you define the costing sheet format and calculations for indirect costs, you must perform a separate step to validate and save the information.</span></span> <span data-ttu-id="d1bc6-145">成本计算单表示公司范围的格式，以便一致地显示有关所售货物成本的信息。</span><span class="sxs-lookup"><span data-stu-id="d1bc6-145">The costing sheet represents a company-wide format for consistently displaying information about the costs of goods sold.</span></span> 

<span data-ttu-id="d1bc6-146">成本计算单为 **“计算物料成本”** 页的一部分。</span><span class="sxs-lookup"><span data-stu-id="d1bc6-146">The costing sheet is displayed as part of the **Calculate item cost** page.</span></span> <span data-ttu-id="d1bc6-147">成本计算单可以针对制造物料的已计算成本记录显示在 **“物料价格”** 页上，或者针对订单特定的计算记录显示在 **“物料清单计算结果”** 页上。</span><span class="sxs-lookup"><span data-stu-id="d1bc6-147">The costing sheet can be displayed for a manufactured item’s calculated cost record on the **Item price** page or for an order-specific calculation record on the **BOM calculation results** page.</span></span> <span data-ttu-id="d1bc6-148">它还可以针对生产订单显示为 **“价格计算”** 页的一部分。</span><span class="sxs-lookup"><span data-stu-id="d1bc6-148">It can also be displayed as part of the **Price calculation** page for a production order.</span></span>

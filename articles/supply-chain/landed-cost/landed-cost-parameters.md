# <a name="landed-cost-parameters-setup"></a>登陆成本参数设置

[!include [banner](../../includes/banner.md)]

您将使用 **登陆成本参数** 页设置在整个 **登陆成本** 模块中用于过帐、状态更新、编号规则和行为的常规信息和配置设置。 参数的设置在法人之间共享，可以由管理员修改。

## <a name="open-the-landed-cost-parameters-page"></a>打开登陆成本参数页

要使用参数，请转到 **登陆成本 \> 设置 \> 登陆成本参数**，然后按照以下小节中的说明设置字段。

![登陆成本参数页](media/landed-cost-parameters.png "登陆成本参数页")

## <a name="general-tab"></a>常规选项卡

### <a name="general-fasttab"></a>常规快速选项卡

下表描述了 **登陆成本参数** 页 **常规** 选项卡的 **常规** 快速选项卡上的可用字段。

| 设置 | 说明 |
|---|---|
| 使用装运汇率 | 为定义的时间段设置装运费用，它用于使用多个货币的商品的估计成本。 将此选项设置为 *是* 可以使用装运费用。 |
| 汇率类型 | 用于航行和航行成本的多币种计算的默认汇率集合。 |
| 更新采购订单数量 | 选择用户更改采购订单行上的数量时将发生什么行为：<ul><li>**接受** – 航行数量自动调整。</li><li>**警告** – 如果行附加到航行，将显示警告，但航行数量会更新。</li><li>**错误** – 如果行附加到航行，将显示错误消息，采购订单无法更新。 因此，必须首先从航行中删除订单行。</li></ul> |
| 自动更新纸箱数量 | 当此选项设置为 *是* 时，所有纸箱将加在一起，并在航行和集装箱级别显示。 设置为 *否* 时，纸箱数量最初设置为 0（零），之后可以根据需要手动进行编辑。 |

### <a name="costing-fasttab"></a>成本计算快速选项卡

下表描述了 **登陆成本参数** 页 **常规** 选项卡的 **成本计算** 快速选项卡上的可用字段。

| 设置 | 说明 |
|---|---|
| 过帐说明 | 在分类帐中选择值调整：<ul><li>**总计** – 总值将过帐到分类帐中。</li><li>**物料组** – 调整按物料组指定。</li><li>**物料编号** – 调整按物料编号指定。 此值提供最详细的信息。</li></ul> |
| 允许零成本 | 如果用户尝试过帐不包含预期航行成本值的航行发票或采购订单的成本估计，将此选项设置为 *否* 将显示错误。 错误消息指明无法分配成本 0（零），发票过帐将失败。 在这种情况下，用户可以手动更新估计（或重新配置自动成本），然后拉入更新后的值或删除不适用的成本。<p>将此选项设置为 *是* 将允许航行成本为空。 在这种情况下，将根据成本区域分配价格 0（零）。 然后，可以在接收航行时对照零价格成本处理供应商成本发票。</p><p>我们建议您在自动成本记录上配置估计，以阻止零价格成本出现。 虽然此估计不完全准确，但应该仍会比假定的零成本更准确。</p> |
| 调整过帐日期 | 当您过帐应付帐款航行成本发票时，结算表（库存调整）也将更新。 当您在发票日记帐中时，选择 **选择航行成本** 页上默认设置的日期：<ul><li>**交易日期** – 使用日记帐的日期（过帐日期）。</li><li>**采购订单发票日期** – 使用存货（采购订单）发票的财务过帐日期。</li><li>**所选日期** – 用户可以指定过帐日期。 日期可以留为空白，但是如果它在成本发票过帐时仍为空白，用户将收到错误消息。</li></ul> |
| 使用采购账单凭证 | 当此选项设置为 *是* 时，成本应计交易将使用与用于采购发票相同的凭证号。 设置为 *否* 时，系统将为 **成本应计凭证** 编号规则使用下一个可用数字，该编号规则在 **登陆成本参数** 页的 **编号规则** 选项卡上设置。<p>此选项仅在将 **应付帐款参数** 的 **发票** 选项卡上的 **过帐到分类帐中的费用帐户** 选项设置为 *是* 时有效。</p> |
| 锁定到结算帐户的手动过帐 | 将此选项设置为 *是*，将通过在供应商发票日记帐的操作窗格中选择 **功能 \> 选择航行成本** 来阻止过帐到交易未链接到航行的清算帐户中。 我们建议您将此选项设置为 *是*，以可以正确结算航行和清算帐户。 |
| 使用成本类型费用应计帐户 | 当此选项设置为 *是* 时，在 **成本类型代码** 页上为相关成本类型代码配置的费用应计帐户将用于将成本应计为费用。<p>此选项仅在将 **应付帐款参数** 的 **发票** 选项卡上的 **过帐到分类帐中的费用帐户** 选项设置为 *是* 时有效。 |
| 将调整过帐为差异 | 当此选项设置为 *是* 时，它将替代标准功能，强制将与估计成本和实际成本之间的差异相关的库存调整交易过帐到差异科目中。<p>当此选项设置为 *否* 时，将根据成本计算方法和成本类型代码的配置处理与差异相关的库存调整交易。 对于标准成本，差异仍将过帐到差异科目。 对于移动加权平均 (WMA)，差异将过帐到差异科目或库存。</p><p>此选项仅在将 **应付帐款参数** 的 **发票** 选项卡上的 **过帐到分类帐中的费用帐户** 选项设置为 *是* 时有效。</p> |

### <a name="validation-fasttab"></a>验证快速选项卡

下表描述了 **登陆成本参数** 页 **常规** 选项卡的 **验证** 快速选项卡上的可用字段。

| 设置 | 说明 |
|---|---|
| 多张成本账单 | 选择如果针对同一个杂项费用的航行、帐页或集装箱处理多个发票将发生什么行为。<ul><li>**接受** – 系统应允许多个成本发票。</li><li>**警告** – 将显示警告。</li><li>**错误** – 将显示错误消息。</li></ul> |
| 每帐页多个供应商 | 选择如果将供应商的多个采购订单添加到帐页时发生什么行为。<ul><li>**接受** – 系统应允许该操作。</li><li>**警告** – 将显示警告，但仍然可以执行操作。</li><li>**错误** – 将显示错误消息，操作将被阻止。</li></ul><p>您的关税代理或当地法律可能会为此字段指定特定值。</p> |
| 多个交货方式容差 | 选择将来自使用不同于航行的交货方式的采购订单中的货物添加到该航行时发生什么行为。<ul><li>**接受** – 系统应允许该操作。</li><li>**警告** – 将显示警告，但仍然可以执行操作。</li><li>**错误** – 将显示错误消息，操作将被阻止。</li></ul> |
| 多个仓库容差 | 选择航行包含多个必须传递到不同仓库的订单行时发生什么行为。 这些订单行可以分布在一个或多个采购订单中。<ul><li>**接受** – 系统应允许该操作。</li><li>**警告** – 将显示警告。</li><li>**错误** – 将显示错误消息。</li></ul> |
| 缺少体积 | 选择如果用户在航行中添加没有体积的物料将发生什么行为。<ul><li>**接受** – 系统应接受物料。</li><li>**警告** – 将显示警告。</li><li>**错误** – 将显示错误消息。</li></ul><p>如果体积用于计算和分摊成本，我们建议您选择 *警告* 或 *错误*。</p> |
| 不含航行成本的服务提供商 | 选择如果用户尝试为未链接到航行的服务提供商处理发票将发生什么行为。 <ul><li>**接受** – 系统应允许该操作。</li><li>**警告** – 将显示警告。</li><li>**错误** – 将显示错误消息。</li></ul><p>我们建议您选择 *警告*。</p> |

### <a name="defaults-fasttab"></a>默认快速选项卡

下表描述了 **登陆成本参数** 页 **常规** 选项卡的 **默认** 快速选项卡上的可用字段。

| 设置 | 说明 |
|---|---|
| 日记帐名称 | 选择 *创建到达日记帐* 功能应使用的默认日记帐。 |
| 航行状态 | 选择在用户可以在系统中为航行设置租赁装运集装箱前航行必须具有的状态。 此操作通常在货物在运输途中或在码头时发生。 |
| 行程模板 | 选择要用于新的租赁装运集装箱的默认行程模板。 通常，您将选择包含租赁成本的行程模板。 |
| 变动 | 如果交货的超交/欠交数量在定义的容差内，将自动处理变动日记帐。 选择要使用的默认变动日记帐。 所选变动日记帐名称的 **对方科目** 字段必须有值。 |
| 转让 | 处理欠交时，欠收数量将转移到欠交仓库中。 选择要使用的默认转移日记帐。 |
| 禁用非航行采购订单 | 为未附加到航行的采购订单关闭登陆成本超交/欠交功能。 |
| 禁用非在途货物采购订单 | 为不使用在途货物功能的采购订单关闭登陆成本超交/欠交功能。 |
| 在途货物超收宽限期 | 指定在首次接收装运集装箱后仍可以为该装运集装箱完成其他超收的天数。 |

## <a name="status-updates-tab"></a>状态更新选项卡

系统使用状态值指示每个航行的状态。 航行状态值可以通过航行跟踪或定期批处理作业自动应用于航行。 或者，您可以通过打开航行，然后在操作窗格 **管理** 选项卡上的 **航行更新** 组中选择状态来手动应用。 

您可以根据需要创建任意多的航行状态值。 但是，必须在 **登陆成本参数** 页的 **状态更新** 选项卡上将其中四个值定义为用于特殊目的。 下表描述了在那里可用的字段。

| 设置 | 说明 |
|---|---|
| 已计算成本 | 选择标识航行已完成的航行状态。 |
| 在途 | 选择标识航行正在途中的航行状态。 |
| 成本计算就绪 | 选择标识航行已准备好进行成本计算的航行状态。 当已为航行处理所有将 **航行成本上的贷方** 字段设置为 *供应商* 的存货采购发票和航行成本发票时，将使用此状态。 成本计算流程失败的航行状态将为 *成本计算就绪*。|
| 预成本计算 | 选择标识航行正在进行预成本计算的航行状态。 当新成本交易在进行成本计算后被添加到航行时，将使用此状态。 新成本交易在接收第二个货运发票或意外滞期费时，可能会被添加到以前进行成本计算的航行。 当新航行成本被添加到已进行成本计算的航行时，将自动应用此状态。 |

## <a name="voyage-creator-tab"></a>航行创建者选项卡

下表描述了 **登陆成本参数** 页的 **航行创建者** 选项卡上的各个部分。

| 部分 | 说明 |
|---|---|
| 容差 | **外部体积容差** 和 **外部重量公差** 字段定义阈值，在该阈值之上，货物将被视为超体积和超重。 当用户在 **航行编辑器** 页上添加货物时，如果体积或重量超过您在此处设置的值，系统将显示警告。 每个字段的值表示为为相关装运集装箱类型设置的最大体积或重量的百分比。 我们建议此值在最大体积或重量的 5% 到 10% 之间。 |
| 帐页创建设置 | 系统可以在航行创建流程中创建多个帐页。 使用此部分定义何时应创建新帐页。 对于本节中的每一行，系统将检查指定的表和字段，并为每个唯一字段值创建帐页。 |

## <a name="cost-estimates-tab"></a>成本估计选项卡

**登陆成本参数** 页的 **成本估计** 选项卡仅提供一个字段：**默认成本计算版本**。 此字段仅在成本计算方法为 *标准成本计算* 时应用。 选择用于 *物料成本价格更新* 定期任务的默认成本计算版本。 您可能必须在每次新的财政年度开始时更改此设置。

## <a name="inventory-dimensions-tab"></a>库存维度选项卡

您将使用 **登陆成本参数** 页的 **库存维度** 选项卡控制默认情况下在使用维度的每个登陆成本页上应显示哪些可用的库存维度。

选择一个维度，然后将默认应显示该维度的每一页的 **航行行**、**在途货物** 和/或 **成本估计** 选项设置为 *是*。 根据需要对其他维度重复此步骤。

此选项卡上的设置跨各个法人为每个指定页面建立默认维度。 但是，在其中一个指定页面上工作的用户可以通过在操作窗格上选择 **库存 \> 显示维度** 来替代默认维度。

## <a name="number-sequences-tab"></a>编号规则选项卡

**登陆成本参数** 页的 **编号规则** 选项卡列出登陆成本所需的但不在法人之间共享的每个参考编号规则类型。 对于列表中的每个参考，选择一个编号规则代码。

> [!NOTE]
> 在多公司配置中，必须为每个公司（法人）创建不同的编号规则。

## <a name="shared-number-sequences-tab"></a>共享编号规则选项卡

**登陆成本参数** 页的 **共享编号规则** 选项卡列出在登陆成本的法人之间共享的每个参考编号规则类型。 当前，列表中只有一个编号规则。 此编号规则用于航行 ID。

在 **所有航行** 页上，用户可以查看所有法人的所有航行。 但是，要编辑和处理航行，用户必须位于所选记录的法人中。

## <a name="feature-visibility-tab"></a>功能可见性选项卡

登陆成本将功能（字段和功能）添加到 Microsoft Dynamics 365 Supply Chain Management 中的几个常用页面。 这些页面包括与供应商主数据、已发布产品、采购订单、转移单和仓库设置相关的页面。 如果您使用登陆成本，则必须让这些功能在每个位置可见，然后才能从中受益。 如果您不使用登陆成本，可以隐藏这些功能，使其不产生干扰。

在 **登陆成本参数** 页的 **功能可见性** 选项卡上，将 **激活** 选项设置为 *是*，以使登陆成本功能在任何提供的位置可见。 设置为 *否* 将在登陆成本外部的常用页面上隐藏这些功能。 但是，即使此选项设置为 *否*，模块本身（包括 **登陆成本参数** 页）对于具有正确访问权限的用户也仍然可用。

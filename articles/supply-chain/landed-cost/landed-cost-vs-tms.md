# <a name="landed-cost-vs-transportation-management"></a>登陆成本与运输管理

[!include [banner](../../includes/banner.md)]
[!include [preview banner](../includes/preview-banner.md)]

Microsoft Dynamics 365 Supply Chain Management 提供两个不同的模块来处理运输：**运输管理** (TMS) 和 **登陆成本**。 本主题总结了两个模块的共同功能，并着重说明了它们之间的区别。 您可以使用此信息来确定最适合您的业务实践的模块。 您可能会发现，有些业务实践与 TMS 一起使用效果更好，而其他实践与登陆成本一起使用效果最好。 然后，根据您的业务要求，您可以选择仅使用一个模块，也可以将两个模块组合使用。

本主题没有对任何一个模块的所有功能进行全面回顾。 而是着重介绍可用功能，因为它与将货物从供应商运输到企业的仓库有关，功能可以在此场景中使用。

## <a name="terminology-reference-data-and-reporting-differences"></a>术语、参考数据和报告差异

### <a name="terminology-comparison"></a>术语比较

TMS 和登陆成本为相似概念使用不同术语。 下表总结了这些术语及其在两个模块中的使用。

| TMS 术语 | 登陆成本术语 | 说明 |
|----------|------------------|-------|
| **装入**<p>*负荷* 是在同一个运输单元（如卡车或船只）上同时运输的装运的集合。 负荷可以是入站或出站。</p> | **航行**<p>通常，*航行* 是沿着一个 *行程* 行驶的一个船只。 一个航行可以包含多个 *装运集装箱*，还可以包括来自组织中不同法人的入站订单。 航行只能是入站。</p> | TMS 也使用术语 *航行*，指的是您可以在其中输入标识符的信息字段。 但是，没有功能与 TMS 字段关联，它与登陆成本中的 *航行* 概念无关。 |
| **运输路线**<p>*路线* 是从起始点到目的地的运输的物理路径。</p> | **行程**<p>*行程* 是从起始点到目的地的运输的物理路径。 每个航行都必须分配到一个行程。</p> | |
| **路线段**<p>一个路线可以有多个 *路线段*，每个代表行程的一个步骤。 一个路线可以包括多个承运人或服务，每个承运人或服务被视为一个路线段。</p> | **支线**<p>每个行程可以有多个 *支线*，每个代表行程的一个步骤。 支线可以是运输、关税处理或另一个活动的一部分。</p> | |
| **集装箱**<p>*集装箱* 可以是 TMS 使用的任何一种包裹或包装。</p> | **装运集装箱**<p>*装运集装箱* 是从集装箱船上得知的字面意义上的物理装运集装箱。</p> | |
| **商品代码**<p>在 TMS（以及 Supply Chain Management 中的其他位置）中，*商品代码* 标识商品类型。 它们可能会影响关税、危险物料的处理等。</p> | **商品代码**<p>在登陆成本中，*商品代码* 在法人级别建立。 它们主要用于报告。</p> | 登陆成本还可以使用用于 TMS 和 Supply Chain Management 中其他位置的商品代码。 但是，登陆成本商品代码仅由登陆成本使用。 |

### <a name="some-reference-data-isnt-shared"></a>有些参考数据不共享

TMS 和登陆成本不共享实体的参考数据，如成本设置、行程和支线。 如果同时使用两个模块，您必须重新创建未共享的参考数据。

### <a name="intercompany-reports-dont-work-with-goods-in-transit"></a><a name="intercompany-reports"></a>内部公司报表不适用于在途货物

以下报表无法与登陆成本提供的在途货物功能一起使用：

- [内部公司在途货物合计报表](https://docs.microsoft.com/dynamicsax-2012/appuser-itpro/intercompany-goods-in-transit-totals-report-intercompanygoodsintransittotals)
- [内部公司在途货物合计报表](https://docs.microsoft.com/dynamicsax-2012/appuser-itpro/intercompany-goods-in-transit-totals-report-intercompanygoodsintransittotals)

这些报表假定货物在您发出销售装箱单后即进入在途状态，在收货后即进入库存。 但是，在途货物不是以这种方式处理。 因此，如果您将在途货物与内部公司功能一起使用，这两个报表的结果都会不正确。

## <a name="using-the-two-modules-together"></a>将两个模块一起使用

您可以将 TMS 同时用于入站和出站操作。 登陆成本为入站操作提供大部分与 TMS 相同的基本功能，它另外还增加了一些功能。 因此，您可能需要考虑将 TMS 用于出站操作，将登陆成本用于入站操作。

通常，我们不建议您将两个模块一起用于入站操作。 您应该使用最能满足您的要求的模块。 如果您确实将两个模块一起使用，则不能在它们之间共享源文档。 例如，不要对 TMS 中的负荷和登陆成本中的航行使用相同的采购订单。 特别是，您必须确保不将系统设置为自动创建入站负荷。 如果采购订单中的物料包含在航行中，则不能将它们作为负荷的一部分处理。

## <a name="goods-in-transit"></a>在途货物

登陆成本的主要功能之一是能够处理在途货物。 在途货物处理让您可以在物料在目标仓库实际接收之前对装运的物料具有财务所有权。 国际贸易通常需要在途货物处理。

### <a name="tms-goods-in-transit-features"></a>TMS 在途货物功能

TMS 当前不支持在途货物处理。

### <a name="landed-cost-goods-in-transit-features"></a>登陆成本在途货物功能

在途货物处理是登陆成本的一个主要功能，提供以下功能：

- **在途货物仓库** – 在途货物仓库可以与 Supply Chain Management 中的每个仓库关联。 货物在为原始采购订单开票后转移到在途货物仓库。 在那里实际预留的物料只有在物料在实际仓库中接收后才可以使用。 因此，您可以通过为采购订单开票来获取货物的财务所有权。
- **在途货物总帐帐户** – 登陆成本将特定的总帐过帐帐户添加到采购订单过帐模板中来处理在途货物处理。 此在途货物总帐帐户充当“开票未收货”类型的帐户。 为原始采购订单开票并创建在途货物订单时，直接采购订单成本将过帐到在途货物总帐帐户中，直到在最终的实际位置接收货物。
- **跟踪控制更新** – 在途货物订单支持登陆成本中的跟踪控制功能。 跟踪控制让您可以在货物在途中时更新货物的预期到达日期。 跟踪控制然后更新航行和关联的采购订单行。 然后预期交货日期可用于主计划和物流。
- **触发超过/低于交易** – 如果航行行上的预期货物与仓库中接收的实际货物之间出现差异，登陆成本将通过创建超过和/或低于交易来解决。 然后，您可以根据要处理这些交易的方式，通过采购订单或变动日记帐来管理这些交易。 超过和低于交易提供返回航行、原始采购订单以及新变动日记帐或采购订单以处理差异的链接。
- **在途货物订单** – 登陆成本引入了一个新的源单据，称为 *在途货物订单*。 通过在途货物订单，您可以为原始采购订单开票来获取物料的财务所有权。 为采购订单开票时，将为每个具有库存维度组合的采购订单行创建新的源单据。 然后货物将实际转移到在途货物仓库，在那里针对在途货物订单实际预留货物，除非在途货物订单已接收，否则货物不能使用。

## <a name="miscellaneous-charges-and-shipment-costs"></a>杂项费用和装运成本

货物装运和装运管理最重要的方面之一是识别与装运关联的间接成本。 这些间接成本不是与采购订单以及装运或航行关联的直接库存成本。 而是由于货物从供应商转移到您的组织而引入的额外成本。 这些成本可能包括与从一个物理位置到另一个位置的货物装运关联的运费成本，或与从外国供应商处进口货物关联的关税成本。

登陆成本通过创建称为 *自动成本* 的新成本结构类型来处理这些成本。 TMS 使用杂项费用功能来处理这些成本。

### <a name="tms-charge-and-cost-features"></a>TMS 费用和成本功能

TMS 使用杂项费用管理分配到相关源单据行的负荷的额外成本。 这些杂项费用可以在采购订单行或采购订单头级别设置。

在 TMS 中，杂项费用可以按库存的重量、体积或数量分摊或划分。 费用可以按运费或附属费用划分。 在 TMS 中使用其他分摊方法需要进一步的开发。

### <a name="landed-cost-charge-and-cost-features"></a>登陆成本费用和成本功能

登陆成本提供一组成本功能来处理与货物装运关联的额外成本。 这些成本称为自动成本，它们在初始装运开票时通过使用估计成本金额来应用。 每个成本类型在其自己的过帐中进行管理。 过帐间接成本的实际发票后，估计成本将自动更新以反映实际成本。

另外，与货物装运关联的间接成本可以在从始发港口开始的航行期间或在接收货物后的任何时间开票。 此功能为货物的使用提供了更大的灵活性。

以下列表概括了登陆成本中费用和成本功能的一些概念：

- **成本区域** – 成本和费用可以在航行中分配到不同级别，或叫 *成本区域*。 成本可以在航行级别应用，可以分解到航行中的每个物料。 另外，成本可以在集装箱、采购订单、物料或转移单级别应用。 每个成本费用可以跨各个区域单独管理，可以分解到按物料划分的成本。
- **分摊方法** – 登陆成本中有多个分摊方法。 成本费用可以按数量、美元金额、价值、重量、体积、度量或用户定义的容量度量进行分摊。
- **清算/差额控制** – 成本费用在登陆成本中设置为其自己的成本代码类型。 每个成本代码类型让您可以为估计成本费用定义清算帐户，还可以为估计成本与实际成本之间的差异定义应计和采购价格差异帐户。 估计成本在最初过帐时，有清算帐户的贷方。 过帐实际发票后，实际成本费用已过帐，估计成本已记入清算帐户的借方。

## <a name="matching-freight-bills-and-invoices"></a>匹配货运帐单和发票

### <a name="tms-bill-and-invoice-matching-features"></a>TMS 帐单和发票匹配功能

TMS 可以将包含估计成本和发票的货运单与货运的实际成本进行匹配。 可以电子方式或使用纸张接收发票。 估计成本与实际成本之间的匹配差异不会影响库存评估。

有关详细信息，请参阅[在运输管理中对运费进行对帐](../transportation/reconcile-freight-transportation-management.md)。

### <a name="landed-cost-bill-and-invoice-matching-features"></a>登陆成本帐单和发票匹配功能

登陆成本可以将货运帐单与估计成本费用相匹配，并可以将实际成本费用开票到估计成本。 在开票时，运费在发票日记帐中，估计成本从清算帐户中清除。 另外，实际成本费用将对照物料的所售货物成本过帐，估计费用和实际费用之间的差额也会一同过帐。 此行为可以在开票流程中提供灵活性。

## <a name="tracking"></a>跟踪

TMS 和登陆成本都有的一个重要功能是能够跟踪货物，以及确定从起始点到最终目标仓库的行程中货物所在的位置。 通过跟踪，您可以跟随和更新货物所在的位置以及货物应会在何时到达仓库以供使用。 除了货物在行程期间的状态外，登陆成本还提供行程每个步骤的预期日期和实际日期。

### <a name="tms-tracking-features"></a>TMS 跟踪功能

TMS 提供有限的跟踪传入负荷功能。 它显示日期，支持构建集成来查找确切位置（例如，在 **运输状态** 页上）。

对于每个路线段，您可以输入估计计划和到达时间。

### <a name="landed-cost-tracking-features"></a>登陆成本跟踪功能

登陆成本可以为每个航行提供从始发港口到最终目的地的跟踪控制。 跟踪控制设置航行的状态。 此状态指示航行是在航行中、在海关进行检查，还是在到达最终仓库的途中在当地交货。 此状态可以在采购订单行、集装箱和航行的标头级别设置。 因此，您可以进行更精细的控制。

此外，基于指定提前期的确认预期日期与行程的每个步骤关联。 确认和预期交货日期将添加到采购订单行和在途货物订单中，可用于主计划和物流。 除预期日期外，实际日期也可以更新。 行程中的后续步骤然后将相应更新。

## <a name="multi-leg-journeys"></a>多支线行程

行程概念确定货物在流程中的位置，控制货物在运输过程中的状态。 货物可能会经过行程的多个支线，您可以将各个成本与特定支线关联。 例如，船只航行的运费成本，从港口到目的地的货物运输的当地运输成本。

### <a name="tms-multi-leg-journey-features"></a>TMS 多支线行程功能

在 TMS 中，可以将路线分解为代表多支线行程的路线段。 TMS 可以将即期费率和附属费用分配到各个路线段。

有关详细信息，请参阅[计划具有多个停止点的货运运输路线](../transportation/plan-freight-transportation-routes-multiple-stops.md)。

### <a name="landed-cost-multi-leg-journey-features"></a>登陆成本多支线行程功能

登陆成本提供一个框架，通过一系列支线将货物从起始点运至目的地，支线是行程中的停止点或步骤。 支线将组合在一起来创建行程模板，模板定义货物如何从供应商转移到您的组织。

在行程的每个支线中，您可以进一步定义每个采购订单行的状态以及与之关联的提前期。 合并的所有支线的提前期用于确定估计交货日期。 但是，要应用多支线行程，需要在途货物处理。 航行中货物的行程在特定于登陆成本的表中管理。

## <a name="receiving-by-container"></a>按集装箱接收

管理货物在船只上如何拆分和储存可能很重要。 在船只上，货物可以保留在一个或多个集装箱中。 接收货物时，将在集装箱中接收，航行中的不同集装箱可能在不同时间或不同日期接收。

TMS 和登陆成本都提供用于管理集装箱中货物收货的功能。 TMS 使用负荷概念管理与装运集装箱关联的货物、采购订单和转移单。 TMS 支持根据通过发货通知 (ASN) 接收的包装结构进行接收。 登陆成本使用装运集装箱概念处理采购订单和控制与船只上集装箱关联的间接成本。

### <a name="tms-receiving-by-container-features"></a>TMS 的按集装箱接收功能

TMS 支持入站 ASN，使用仓库应用的所有接收变体，以及使用 Supply Chain Management 客户端的所有接收方法。

### <a name="landed-cost-receiving-by-container-features"></a>登陆成本的按集装箱接收功能

为支持按集装箱接收，登陆成本将创建装运集装箱记录，并使用集装箱 ID 将采购订单与特定的装运集装箱关联。 然后，间接成本可以应用于该装运集装箱并分解，以与相关的采购订单关联。

登陆成本中的集装箱可以通过一种称为 *在途货物收货* 的新收货类型，通过到达日记帐或移动设备接收来接收。 使用到达日记帐时，可以从在途货物订单或集装箱中的原始采购订单行初始化数量。 登陆成本为通过仓库应用接收提供了两个工作类型。

登陆成本不为电子货物收货提供 ASN。 此外，它不支持处理负荷接收、牌照接收或混合牌照接收的仓库应用流。

## <a name="rate-shopping-by-vendor"></a>按供应商进行费率查询

费率查询让公司可以根据最低价格、最快路线或同时考虑两者来选择运输供应商。

### <a name="tms-rate-shopping-features"></a>TMS 费率查询功能

TMS 让您可以为入站和出站订单确定供应商和路线选择解决方案。 例如，您可以标识装运的最快路线或最便宜费率。

TMS 通过费率路线工作台提供全面的费率查询和货运优化，通过费率引擎提供灵活的费率选项，提供用于与外部当事方集成的费率引擎 API，并支持容重。

有关详细信息，请参阅[运输管理引擎](../transportation/transportation-management-engines.md)。

### <a name="landed-cost-rate-shopping-features"></a>登陆成本费率查询功能

登陆成本仅对供应商费率查询提供有限支持。 您可以输入货运转运商值，但登陆成本不会跨多个供应商进行比较。

## <a name="driver-check-incheck-out-with-appointment-scheduling"></a>使用预约计划的驾驶员签入/签出

驾驶员签入/签出功能让系统能够监视到达情况并防止装货码头出现拥挤。

### <a name="tms-driver-check-incheck-out-features"></a>TMS 驾驶员签入/签出功能

TMS 允许您创建 *预约*。 预约代表特定日期和特定时间在码头发生的接收采购订单、装运销售订单或处理入站或出站负荷的事件。 预约可确保码头可用于装货和卸货，它们帮助防止出现多个承运人同时到达一个位置的情况。

系统支持允许驾驶员在特定码头大门签入。

### <a name="landed-cost-driver-check-incheck-out-features"></a>登陆成本驾驶员签入/签出功能

登陆成本可以存储将要交付集装箱的日期和时间的估计。

## <a name="transfer-orders-and-additional-costs"></a>转移单和额外成本

通常，企业必须能够在仓库之间转移货物。 发生这种类型的转移时，成本与转移关联，您可能需要确认这些成本。 在登陆成本中，可以将转移单添加到航行或船只中，以跟踪货物从一个仓库或站点到另一个仓库或站点的移动，估计交货时间和预期交货日期，以及将间接成本添加到货物转移中。

### <a name="tms-transfer-order-cost-features"></a>TMS 转移单成本功能

TMS 支持创建连接到转移的运费。 这些费用可以从转移单查看，但登陆成本不会被添加到物料成本中。 通过基于这些费用创建货运帐单来支持货运对帐。 然后将此货运帐单与供应商货运发票进行匹配。

### <a name="landed-cost-transfer-order-cost-features"></a>登陆成本转移单成本功能

登陆成本让您可以将转移单添加到船只或航行中。 这样，您可以在接收转移单时将装运成本作为库存结算添加到航行中。 可以为实际成本的间接成本开票，并根据航行跟踪间接成本来更新所售货物成本。 转移单在标准版本中不使用在途货物处理，但可以添加到航行中。 登陆成本将被添加到每个物料的总成本中。

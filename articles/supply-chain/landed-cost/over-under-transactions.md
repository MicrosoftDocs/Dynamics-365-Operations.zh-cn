# <a name="overunder-transactions"></a>超过/低于交易记录

[!include [banner](../../includes/banner.md)]

处理航行中的订单时，系统预期最终目标仓库接收的用于使用的物料数量，与在与航行关联的采购订单行上指定的数量匹配。 但是，由于仓库中并非总是收到采购订单行上的确切数量，因此 **登陆成本** 模块定义了一组规则，用于处理货物的超收和收货不足。 这些规则特别重要，因为原始采购订单已经开票，无法再进行修改。 通过设置超过/低于交易策略的详细信息，您可以让系统确定收货时如何管理货物的超过处理和低于处理。 您还可以使用 **超过/低于交易** 页手动管理超过和不足的库存。

## <a name="overunder-tolerances"></a>超过/低于容差

您可以设置超交和欠交容差来指定可以在航行中处理而不会引起错误的超过和低于数量或体积。 如果航行行收货超出这些容差，必须进行修改和解决，然后才能关闭航行行，进行进一步处理。

要配置容差，转到 **登陆成本 \> 超过/低于设置 \> 超过/低于容差**。 在那里，您可以查看、编辑、添加和删除容差记录。 下表说明了可用于每个记录的字段。

| 字段 | 说明 |
|---|---|
| 帐户编码 | <p>通过选择以下值之一定义容差要应用的供应商范围：</p><ul><li>**表**– 超过/低于容差仅应用于为行选择的供应商。</li><li>**组**– 超过/低于容差应用于为行选择的供应商容差组的所有供应商。</li><li>**所有** – 超过/低于容差应用于所有供应商。</li></ul> |
| 帐户关系 | 根据您在 **帐户代码** 字段中选择的值，选择超过/低于容差要应用的供应商或供应商组。 |
| 物料代码 | <p>通过选择以下值之一定义容差要应用的物料范围：</p><ul><li>**表**– 超过/低于容差仅应用于为行选择的物料。</li><li>**组**– 超过/低于容差应用于为行选择的物料容差组的所有物料。</li><li>**所有** – 超过/低于容差应用于所有物料。</li></ul> |
| 物料关系 | 根据您在 **物料代码** 字段中选择的值，选择超过/低于容差要应用的物料或物料组。 |
| 金额容差 | 输入应该应用于整个采购订单的金额容差。 |
| 百分比容差 | 输入应该应用于单个采购订单行的百分比容差。 |

## <a name="overunder-reasons"></a>超过/低于原因

当超过或低于数量与接收的航行行有关时，您可能必须确定数量超过或低于的原因。 在这种情况下，您可以在接收货物时在接收行上选择超交或欠交原因。

要设置超交和欠交原因，转到 **登陆成本 \> 超过/低于设置 \> 超过/低于原因**。 在那里，您可以查看、编辑、添加和删除可用的超交和欠交原因。 下表说明了可用于每个原因的字段。

| 字段 | 说明 |
|---|---|
| 超过/低于原因 | 为超收或收货不足交易的原因输入唯一名称。 |
| 说明 | 输入超过/低于原因的描述。 |
| 变动 | 选择与超过/低于原因关联的变动日记帐。 此字段在 **库存日记帐名称** 页（**库存管理设置 \> 日记帐名称 \> 库存**）列出了与 *变动* 日记帐类型关联的所有可用日记帐。 |

## <a name="item-overunder-tolerance-groups"></a>高出/低于容差物料组

具有相似容差的物料可以分组在一起。 这样，您可以同时为该组中的所有物料设置超过/低于容差。

要设置物料超过/低于容差组，转到 **登陆成本 \> 超过/低于设置 \> 物料超过/低于容差组**。 在那里，您可以查看、编辑、添加和删除超过/低于容差组记录。 下表说明了可用于每个记录的字段。

| 字段 | 说明 |
|---|---|
| 超过/低于容差组 | 为组输入唯一名称。 选择描述容差的名称，如 *1% 差异*。 |
| 说明 | 输入组的描述。 |

## <a name="vendor-overunder-tolerance-groups"></a>供应商超过/低于容差组

您可以将定期超交和欠交的供应商分到一组。 然后，可以按组设置超过/低于容差。

要设置供应商超过/低于容差组，转到 **登陆成本 \> 超过/低于设置 \> 供应商超过/低于容差组**。 在那里，您可以查看、编辑、添加和删除超过/低于容差组记录。 下表说明了可用于每个记录的字段。

| 字段 | 说明 |
|---|---|
| 超过/低于组 | 为组输入唯一名称。 选择描述属于此组的供应商的类型的名称。 |
| 说明 | 输入组的描述。 |

## <a name="view-and-process-overunder-transactions"></a>查看和处理超过/低于交易

当储存的货物数量与初始化数量不匹配时，将生成超过和低于交易。 到达日记帐中的数量仅应使用储存的数量进行更新。

处理航行行的接收后，可以为供应商设置超过/低于容差。 然后将查看和验证物料。 容差可以设置为百分比、当地金额或都设置。

如果接收的物料在容差范围内，系统将生成变动日记帐。 可以在航行参数页面指定此日记帐。 另外，还将创建超过/低于交易。 例如，订单值为 100 美元，收货值为 99 美元，这些值符合容差规则。 在这种情况下，将为收货不足的数量创建负变动日记帐。

如果接收的物料超出容差，系统将为数量差异生成超过/低于交易。

要查看和处理超过/低于交易，转到 **登陆成本 \> 查询 \> 超过/低于交易**。 在那里，超过/低于交易将与航行中所有超收或收货不足的物料关联。 我们建议您使用 **超过/低于交易** 页来解决与航行关联的所有超过/低于交易。 我们还建议您 **不要** 使用变动或盘点日记帐来手动解决欠交仓库交易。 而应该使用 **超过/低于交易** 页管理欠交仓库数量。

### <a name="upper-overview-tab"></a>上部的“概览”选项卡

要查看超过/低于交易，选择 **超过/低于交易** 页上部的 **概览** 选项卡。 下表描述了网格中的可用字段。

| 字段 | 说明 |
|---|---|
| 超过/低于交易记录编号 | 处理到达日记帐时自动创建的超过/低于交易记录的唯一名称。 |
| 航行 | 采购订单行所附加到的航行。 |
| 装运集装箱 | 采购订单行所附加到的集装箱。 |
| 到达日记帐 | 从其生成超过/低于行的到达日记帐。 |
| 参考 | 对与超过/低于交易关联的采购订单或转移单的引用。 |
| 数值 | 超过/低于交易中引用的采购订单。 |
| 供应商帐户 | 与超过或低于相关的供应商。 |
| 在途货物编号 | 在途货物编号（如果适用）。 |
| 物料编号 | 与超过或低于相关的物料编号。 |
| 原始数量 | 附加到装运的采购订单行的原始数量。 |
| 收货数量 | 实际接收的数量。 |
| 差额 | 预期到达日记帐金额与到达日记帐中过帐的金额之间的差额。 负值表示货物超交。 |
| 剩余数量 | 仍在到达日记帐行上的数量。 |
| 超交/欠交 | 指示接收的数量是超过还是不足的值。 |
| 状态 | 超过/低于交易的状态。 根据 **[登陆成本参数](landed-cost-parameters.md)** 页上的设置，超交可以自动为超交量创建变动日记帐并将其过帐。 在这种情况下，超过/低于交易的状态将为 *已完成*。 如果需要执行操作将货物移出欠交仓库，记录的状态将为 *进行中*。 |
| 超过/低于原因 | 与超过/低于交易关联的原因。 |
| 其他库存维度 | 您可以根据需要在网格中显示其他维度的列。 这些维度包括批处理号、库存状态和仓库。 在操作窗格上，选择 **库存 \> 显示维度** 打开一个对话框，您可以在其中选择要包括的维度。 |

### <a name="process-overunder-transactions"></a>处理超过/低于交易

**超过/低于交易** 页的在操作窗格提供以下命令来处理在页面上选择的交易。 每个命令让您可以选择如何处理每个交易。

- **创建 \> 变动** – 为所选交易创建和过帐变动日记帐。 对于低于交易，将自动为预期和实际收到数量之间的差异创建和过帐变动日记帐。 如果您希望供应商了解成本差异，请为超过交易选择此命令。
- **创建 \> 采购订单** – 为所选交易的预期和实际接收数量之间的差异创建采购订单。 如果您的组织将了解成本差异，请为超过交易选择此命令。
- **创建 \> 转移到目标** – 此命令仅适用于转移单。 如果要将超过或低于数量转移到目标仓库，请选择此命令。
- **创建 \> 转移到原始** – 此命令仅适用于转移单。 如果要将超过或低于数量转移到原始仓库，请选择此命令。

# <a name="product-collection-modules"></a>产品集合模块


[!include [banner](includes/banner.md)]

此主题概述 Microsoft Dynamics 365 Commerce 中的产品集合模块。

## <a name="overview"></a>概览

产品发现是零售商在电子商务网站中主要用于联系客户的工具。 产品集合模块通过提供可用于快速创作产品集合的直观视觉界面，帮助零售商生成出色的购物体验。

产品集合模块表示网站中的物理产品和服务。 产品集合模块通常链接到详细信息页，客户可在此页中购买产品或服务，或了解其详情。 

产品集合的来源可以是以下四种类型的列表：

- Dynamics 365 Commerce 中手动定义为产品的相关产品的产品的编辑列表，或产品列表
- 算法列表，如新品列表、最畅销产品或热门产品
- 基于机器学习的建议列表
- 支持个性化的客户结果的个性化列表。 客户必须登录到电子商务站点才能查看个性化结果。 来宾用户看不到个性化结果。 客户可以从[帐户管理页面](account-management.md)选择退出个性化。

下图显示电子商务站点中正在使用的不同产品集合类型。

![电子商务站点中不同产品集合类型的示例](./media/ProductCollectionsAcrossTheSiteUseProductPlacement.png)

> [!NOTE]
> 始终使用产品集合模块显示相似类型的一组产品。

## <a name="product-collection-modules-and-types"></a>产品集合模块和类型

下表介绍 Dynamics 365 Commerce 中的各种产品集合模块。

| 产品集合模块  | 类型 | 说明 |
|----------------------------|------|-------------|
| 类别                   | 类别 | 此模块根据零售商为渠道创建的导航类别层次结构的定义，显示一个类别的产品列表。 |
| 相关产品           | 评论 | 此模块显示推销经理在 Commerce 中为作者选择的关系类型配置为相关产品的产品的列表。 |
| 搜索结果             | 搜索查询 | 这种产品集合模块显示最符合客户输入的搜索查询的产品的列表。 |
| 精选产品列表      | 评论 | 此模块显示推销者和编辑者已在 Commerce 中创建的自定义列表。 |
| 全新                        | 算法 | 此模块显示已经分类为渠道和目录的最新产品的列表。 此列表可以为登录用户显示个性化结果（如果站点作者选择该选项）。 |
| 最畅销               | 算法 | 此模块显示按销售数最高排名的产品的列表。 此列表可以为登录用户显示个性化结果（如果站点作者选择该选项）。 |
| 热门                   | 算法 | 此模块显示给定期间表现最出色的产品的列表。 此列表可以为登录用户显示个性化结果（如果站点作者选择该选项）。 |
| 经常搭配购买的产品 | 人工智能/机器学习 | 此模块使用机器学习分析客户购买模式，并推荐经常与给定产品搭配购买的相关商品的列表 此列表可以为登录用户显示个性化结果（如果站点作者选择该选项）。 |
| 人们还喜欢           | 人工智能/机器学习 | 此模块使用机器学习分析客户购买模式，并推荐与给定产品有关的商品。 此列表可以为登录用户显示个性化结果（如果站点作者选择该选项）。 |
| 为您推荐              | 人工智能/机器学习 | 此模块使用机器学习分析登录用户的购买模式，并基于这些购买模式提供个性化推荐。 对于来宾用户，此列表将被折叠。 |

## <a name="add-a-product-collection-module-to-a-category-page"></a>向类别页添加产品集合模块

若要向类别页添加产品集合模块，请执行以下步骤。

1. 在 Dynamics 365 Commerce 中，转到您的站点，然后创建一个与默认类别页使用同一个模板的页面。
1. 在页面大纲中，选择**子页脚**插槽，选择省略号按钮 (**...**)，然后选择**添加模块**。
1. 在**添加模块**对话框中，选择**容器**，然后选择**确定**。
1. 在容器模块中，选择省略号按钮，然后选择**添加模块**。
1. 在**添加模块**对话框中，选择**产品集合**，然后选择**确定**。  
1. 通过为产品集合选择相应数据源和输入来配置设置。
1. 在产品集合模块的属性窗格中，选择**添加产品列表**。
1. 在**选择产品列表配置**对话框中，选择列表类型，输入商品数，然后选择列表类型的其他任何可用选项。 有关列表类型的详细信息，请参见下表。 
1. 选择**确定**。
1. 保存页面，然后签入。

下表显示**选择产品列表配置**对话框中可供选择的列表类型。

| 类型                       | 说明 | 用法 | 页面上下文 | 特定上下文 | 个性化 |
|----------------------------|-------------|-------|--------------|------------------|-----------------|
| 产品(按类别)       | 属于给定类别的产品的列表。 通过页面上下文或作者提供的上下文确定此类别。 | 这种类型的列表可以在任何页面上（例如，主页、类别页面、市场营销页面或产品详细信息页面 \[PDP\]）用来推广特定类别的产品。 | 来自页面上下文的类别（如果有，例如，类别页面） | 作者可以提供特定类别作为列表的上下文。 | 不适用 |
| 相关产品           | 推销经理已在 Commerce 中为关系类型配置为相关产品的产品的列表。 | 这种类型的列表主要用于 PDP，但如果提供了父产品，则可以在任何页面上使用。 | 来自页面的产品，关系类型（必需） | 可以在选择器中选择产品，使用关系类型。 | 不适用 |
| 策划                    | 推销者和编辑者已在 Commerce 中创建的自定义列表。 | 扩充类别页、主页、结帐与购物车页，以及产品页 | 不适用 | 不适用 | 不适用 |
| 算法                | <ul><li>**新商品** – 已经分类为渠道和目录的最新产品的列表。</li><li>**最畅销** – 按销售数最高排名的产品的列表。</li><li>**热门** – 给定期间表现最出色的产品的列表。</li></ul> | 主页、扩展类别页和结帐与购物车页 | 来自页面上下文的类别（例如，类别页面） | 由站点作者确定的类别 | 受支持 |
| 经常搭配购买的产品 | 使用机器学习分析客户购买模式和推荐经常与给定产品搭配购买的相关商品的列表。 | 这种类型的列表仅适用于购物车页面。 | 购物车 | 不适用 | 受支持 |
| 人们还喜欢           | 使用机器学习分析客户购买模式和推荐与给定产品有关的商品的列表。 | 在 PDP 上使用此类型的列表来显示其他客户已购买的产品。 | 来自页面的产品上下文 | 由站点作者提供的产品 | 受支持 |
| 为您推荐              | 使用机器学习确定客户偏好的列表。 | 这种类型的列表可以在任何页面上使用。 | 不适用| 不适用 | 受支持 |

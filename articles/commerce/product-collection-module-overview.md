# <a name="product-collection-modules"></a>产品集合模块  

[!include [banner](includes/preview-banner.md)]
[!include [banner](includes/banner.md)]

此主题概述 Microsoft Dynamics 365 Commerce 中的产品集合模块。

## <a name="overview"></a>概览

产品发现是零售商在电子商务网站中主要用于联系客户的工具。 产品集合模块通过提供可用于快速创作产品集合的直观视觉界面，帮助零售商生成出色的购物体验。

产品集合模块表示网站中的物理产品和服务。 产品集合模块通常链接到详细信息页，客户可在此页中购买产品或服务，或了解其详情。 

产品集合的来源可以是三种类型的列表：

- Dynamics 365 Retail 中手动定义为产品的相关产品的产品的编辑列表，或产品列表
- 算法列表，如新品列表、最畅销产品或热门产品
- 基于机器学习的建议列表

下图显示电子商务站点中正在使用的不同产品集合类型。

![电子商务站点中不同产品集合类型的示例](./media/ProductCollectionsAcrossTheSiteUseProductPlacement.png)

> [!NOTE]
> 始终使用产品集合模块显示相似类型或主题的一组产品。

## <a name="product-collection-modules-and-types"></a>产品集合模块和类型

下表介绍 Dynamics 365 Commerce 中的各种产品集合模块。

| 产品集合模块  | 类型 | 说明 |
|----------------------------|------|-------------|
| 类别浏览            | 评论 | 这种产品集合模块使用零售商为零售渠道创建的导航类别层次结构显示在特定站点类别中提供的产品的浏览流程。 |
| 搜索结果             | 搜索查询 | 这种产品集合模块显示最符合客户输入的搜索查询的产品的列表。 |
| 相关产品           | 评论 | 这种产品集合模块显示推销经理在 Retail 中为作者选择的关系类型配置为相关产品的产品的列表。 |
| 精选产品列表      | 评论 | 这种产品集合模块显示推销者和编辑者在 Retail 中创建的自定义列表。 |
| 全新                        | 算法 | 这种产品集合模块显示已经归类为渠道和目录的最新产品的列表。 |
| 最畅销               | 算法 | 这种产品集合模块显示按最大销售数排列的产品的列表。 |
| 热门                   | 算法 | 这种产品集合模块显示给定期间表现最出色的产品的列表。 |
| 经常搭配购买的产品 | 人工智能/机器学习 | 这种产品集合模块使用机器学习分析客户购买模式和推荐经常与给定产品搭配购买的相关商品。 |
| 人们还喜欢           | 人工智能/机器学习 | 这种产品集合模块使用机器学习分析客户购买模式和推荐与给定产品有关的商品。 |

## <a name="add-a-product-collection-module-to-a-category-page"></a>向类别页添加产品集合模块

若要向类别页添加产品集合模块，请执行以下步骤。

1. 在 Dynamics 365 Commerce 中，转到您的站点，然后创建一个与默认类别页使用同一个模板的页面。
1. 在页面大纲中，选择**子页脚**插槽，选择省略号按钮 (**...**)，然后选择**添加模块**。
1. 在**添加模块**对话框中，选择**容器**，然后选择**确定**。
1. 在容器模块中，选择省略号按钮，然后选择**添加模块**。
1. 在**添加模块**对话框中，选择**产品集合**，然后选择**确定**。
1. 通过为产品集合选择相应数据源和输入来配置设置。
1. 在产品集合模块的属性窗格中，选择**添加产品列表**。
1. 在**选择产品列表配置**对话框中，选择列表类型，输入商品数，然后选择列表类型的其他任何可用选项。 有关列表类型的详细信息，请参见下表。 
1. 选择**确定**。
1. 保存页面，然后签入。

下表显示**选择产品列表配置**对话框中可供选择的列表类型。
   
| 类型                       | 说明 | 一般实践 | 可从页面上下文派生的上下文 | 作者可将页面上下文替换为的上下文 |
|----------------------------|-------------|------------------|-------------------------------------|-----------------------------------------------|
| 产品(按类别)       | 属于给定类别的产品的列表。 通过页面上下文或作者提供的上下文确定此类别。 | 扩充类别页、主页、结帐与购物车页，以及产品页 | 类别 | 作者决定的类别 |
| 相关产品           | 推销经理已在 Retail 中为关系类型配置为相关产品的产品的列表。 | 产品页、结帐与购物车页，愿望列表页和客户帐户页 | 产品、关系类型（必需）  | 产品、关系类型 |
| 策划                    | 推销者和编辑者已在 Retail 中创建的自定义列表。 | 扩充类别页、主页、结帐与购物车页，以及产品页 | 不适用 | 列表选择器 |
| 算法                | <ul><li>**新商品** – 已经分类为渠道和目录的最新产品的列表。</li><li>**最畅销** – 按销售数最高排名的产品的列表。</li><li>**热门** – 给定期间表现最出色的产品的列表。</li></ul> | 主页、扩展类别页和结帐与购物车页 | 类别 | 作者决定的类别 |
| 经常搭配购买的产品 | 使用机器学习分析客户购买模式和推荐经常与给定产品搭配购买的相关商品的列表。 | 产品页和结帐与购物车页 | 产品、购物车 | 包括购物车 |
| 人们还喜欢           | 使用机器学习分析客户购买模式和推荐与给定产品有关的商品的列表。 | 产品页和结帐与购物车页 | 产品、购物车 | 不适用 |

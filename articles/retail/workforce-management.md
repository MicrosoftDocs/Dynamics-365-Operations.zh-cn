# <a name="workforce-management-overview"></a>劳动力管理概览

[!include[banner](includes/banner.md)]
    
劳动力管理 (WFM) 服务利用熟悉的销售点 (POS) 客户端，现代 POS 和云 POS，使店长能够轻松管理劳动力。 WFM 服务使用扩展框架下载 POS 中的相关包。 当关闭并重新打开 POS 时，这些包即下载完成。 因此，当 Microsoft 发布 WFM 的新功能时，必须关闭并重新打开 POS 应用。

店长可以使用该服务轻松创建和发布商店的每周工作计划。 商店工作人员可以迅速查看自己的计划以及同事的计划。 这样一来，他们可以了解谁将在分配的班次中与他们一起工作。 商店工作人员还可以创建缺勤、班次交换和班次提议请求。 所有这些请求触发一个定义的工作流。

在班次中，商店工作人员可以利用 POS 客户端中内置的上下班打卡功能。 之后，数据将被发送到总部 (HQ) 进行工资处理。 上下班打卡功能是 POS 现有的一项功能。 有关设置和支持的方案的详细信息，请参阅[上班打卡和下班打卡](retail-time-attendance.md)。

## <a name="supported-scenarios"></a>支持的方案
本节提供有关支持的方案的详细信息。

- **创建和发布计划** - WFM 服务使用在 HQ 中配置的 POS 权限来确定某个工作人员是否具有店长权限。 只有店长可以使用计划管理操作来创建和发布计划。 他们可以将现有的工作班次从一位工作人员复制粘贴到另一位工作人员，从而快速创建计划。 还可以通过将前面任何一周的计划导入到当前周的方式创建计划。

    如果未发布计划，则计划处于草稿状态，与已发布的计划在外观上不一样。 店长可以通过单击**发布**按钮发布任何一周的计划。 发布一个星期的计划后，任何更改都会自动发布。 更改示例包括添加或删除工作班次或缺勤或编辑工作班次或缺勤。 商店工作人员只能查看各周已经发布的计划。
    
- **创建和审核请求** - 商店工作人员可以创建三种类型的请求：

    - 请求缺勤
    - 请求班次交换
    - 请求班次提议

    可以使用计划请求操作创建这些请求。 每个请求都遵循一个已定义的工作流，并且工作人员可以轻松查看其请求的当前状态。
    
    缺勤请求自动发送给店长以供审核。 如果存在多个店长，所有店长都可查看给定的请求，但只有一个店长可以批准或拒绝该请求。 缺勤类型在零售总部中使用**零售**模块中的**休假和缺勤类型**页进行配置。 在店长批准或拒绝请求之后，该请求将移至**历史记录**选项卡用于计划请求操作。
    
    班次交换或班次提议请求先发送到被请求的工作人员处。 仅当该工作人员批准该请求后，店长才可以查看请求。 因此，如果工作人员拒绝班次交换或班次提议请求，则经理不能查看该请求。 创建请求的工作人员也可以在经理批准或拒绝该请求前取消该请求。

- **在计划视图中显示活动的商店工作人员** - 通过将某位工作人员与商店的员工通讯簿相关联等方式将该工作人员添加到商店后，该工作人员在 WFM 服务中可供调配。 在 HQ 中运行命名为**处理工作人员数据以进行劳动力管理**的重复批处理作业。 该作业准备好要被发送到 Common Data Service (CDS) 的商店-工作人员关联。

    从商店中删除工作人员时使用同一个流程。 但是，被删除的工作人员仍然显示过去、当前和未来的周中是否有活动的工作班次或缺勤被分配到该工作人员。 因此，除非这些工作班次和缺勤被删除，否则这些周将继续显示被删除的工作人员。

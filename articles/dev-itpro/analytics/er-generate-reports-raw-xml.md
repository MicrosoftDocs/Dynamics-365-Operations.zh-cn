[!include[banner](../includes/banner.md)]

<span data-ttu-id="c4410-104">可使用新的**原始 XML** 格式元素设计电子申报 (ER) 格式，以生成 XML 格式的传出单据。</span><span class="sxs-lookup"><span data-stu-id="c4410-104">You can use the new **RAW XML** format element to design Electronic reporting (ER) formats that generate outgoing documents in XML format.</span></span> <span data-ttu-id="c4410-105">在某些情况下，您可能出于下面的一项或多项原因，向这些报表添加原始 XML 数据。</span><span class="sxs-lookup"><span data-stu-id="c4410-105">In some cases, you might prefer to add raw XML data to these reports for one or more of the following reasons:</span></span>

- <span data-ttu-id="c4410-106">将原始 XML 用于报表的最初设计和不间断的维护更为方便，因为可以通过执行运行时表达式自动生成 XML 结构。</span><span class="sxs-lookup"><span data-stu-id="c4410-106">It's more convenient to use raw XML for the original design and ongoing maintenance of a report, because the XML structure can be automatically generated by executing a runtime expression.</span></span> <span data-ttu-id="c4410-107">因此，不必在设计时为多个格式元素确定多个绑定。</span><span class="sxs-lookup"><span data-stu-id="c4410-107">Therefore, multiple bindings don't have to be determined for multiple format elements at design time.</span></span> <span data-ttu-id="c4410-108">如果在生成报告时，使用的数据源中包含可用于创建 XML 元素的信息，这就有可能。</span><span class="sxs-lookup"><span data-stu-id="c4410-108">It is possible when the data sources that you're using contain information that can be used to make XML elements while the report is generated.</span></span> 
- <span data-ttu-id="c4410-109">其他方法不能使用以前收到并存储在系统中的 XML 内容填充报表。</span><span class="sxs-lookup"><span data-stu-id="c4410-109">No other method can be used to fill the report with XML content that was previously received and stored in the system.</span></span> <span data-ttu-id="c4410-110">例如，生成的 XML 响应可能必须包含以前发送的 XML 请求的内容。</span><span class="sxs-lookup"><span data-stu-id="c4410-110">For example, the XML response that is generated might have to contain the content of an XML request that was sent earlier.</span></span>
- <span data-ttu-id="c4410-111">没有其他方法可以基于字符的数字代码将字符插入生成的单据中。</span><span class="sxs-lookup"><span data-stu-id="c4410-111">No other method can be used to insert characters into the generated document based on their numeric codes.</span></span> <span data-ttu-id="c4410-112">对于某些语言和字符来说。不存在这种代码。</span><span class="sxs-lookup"><span data-stu-id="c4410-112">For some languages and characters, codes of this type don't exist.</span></span> <span data-ttu-id="c4410-113">例如，希腊字母 rho (ρ) 和 HTML 实体代码（如 \&eacute）；对于 *e* 来说，就有重音符 (é)。</span><span class="sxs-lookup"><span data-stu-id="c4410-113">Examples include the Greek letter rho (ρ) and HTML entity codes such as \&eacute; for an *e* that has an acute accent (é).</span></span>

> [!NOTE]
> <span data-ttu-id="c4410-114">请注意，此框架不控制通过使用**原始 XML** 格式元素放入生成的单据中的 XML 内容是否正确。</span><span class="sxs-lookup"><span data-stu-id="c4410-114">Be aware that the framework doesn't control whether the XML content that is placed to the generated document by using the **RAW XML** format element is correct.</span></span>

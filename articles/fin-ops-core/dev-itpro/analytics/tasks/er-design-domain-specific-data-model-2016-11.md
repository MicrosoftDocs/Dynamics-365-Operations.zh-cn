# <a name="er-design-domain-specific-data-model"></a>ER 设计域特定数据模型

[!include [banner](../../includes/banner.md)]

以下步骤说明属于系统管理员或电子报表开发人员的用户如何创建新电子报表 (ER) 配置，使其包含电子付款文件中的数据模型。 此数据模型稍后将用于您创建付款文件的格式时的数据源。

在此示例中，您将创建示例公司 Litware，Inc. 的配置。这些步骤可在任何公司执行，因为公司之间共享 ER 配置。 为了完成这些步骤，您必须首先完成“创建配置提供商并标记为有效”这一过程中的步骤。

1. 转到“组织管理”>“工作区”>“电子申报”。

    为示例公司“Litware 公司”选择配置提供程序。 如果没有看到此配置提供程序，您必须首先完成“创建配置提供程序并标记为有效”这一过程中的步骤。  
    
2. 单击“申报配置”。

    您要创建含有电子付款文件的数据模型的模板。 此数据模型稍后将用于您创建付款文件的格式时的数据源。  

## <a name="create-a-new-data-model-configuration"></a>创建新的数据模型配置
1. 单击“创建配置”，以打开下拉对话框。
2. 在“名称”字段中，键入“付款（简化模型）”。
3. 在“描述”字段，键入“付款模式配置”。

    有效配置提供商将自动在此输入。 此提供商可以维护此配置。 其他提供商可以使用此配置，但不能对其进行维护。  

4. 单击“创建配置”按钮以完成配置创建任务

## <a name="create-a-data-model"></a>创建数据模型
您正在为所选配置创建新的数据模型。 此配置版本将为“草稿”状态。  
1. 单击“设计器”。

## <a name="define-the-structure-of-a-party-participating-in-a-payment-process"></a>定义参与付款流程的当事方的结构
1. 单击“新建”，以打开对话框。
2. 在“名称”字段中，键入“当事方”。
3. 单击“添加”。
4. 单击“新建”，以打开对话框。
5. 在“名称”字段中，键入“名称”。
6. 在“物料类型”字段中，选择“字符串”。
7. 单击“添加”。
8. 在“查找”字段中，键入“当事方”。
9. 单击“查找上一个”。

## <a name="define-the-bank-structure-for-this-model"></a>定义此模型的银行结构
1. 单击“新建”，以打开对话框。
2. 在“名称”字段中，键入“代理”。
3. 在“物料类型”字段中，选择“记录”。
4. 单击“添加”。
5. 在“描述”字段，输入“负责维护当事方（借方/贷方）帐户的金融机构（例如银行）”。

    负责维护当事方（贷方/借方）帐户的金融机构（例如银行）。  

6. 单击“新建”，以打开对话框。
7. 在“名称”字段中，键入“名称”。 
8. 在“物料类型”字段中，选择“字符串”。
9. 单击“添加”。
10. 单击“新建”，以打开对话框。
11. 在“名称”字段中，键入“SWIFT”。
12. 单击“添加”。
13. 在“描述”字段中，输入“银行标识代码”。 
14. 单击“新建”，以打开对话框。
15. 在“名称”字段中，键入“银行代号”。
16. 单击“添加”。
17. 在“描述”字段中，输入“银行代号”。
18. 单击“查找上一个”。

## <a name="define-the-bank-account-structure-for-this-model"></a>定义此模型的银行帐户结构
1. 单击“新建”，以打开对话框。
2. 在“名称”字段中，键入“帐户”。 
3. 在“物料类型”字段中，选择“记录”。
4. 单击“添加”。
5. 在“描述”字段中，输入“当事方在金融机构（如银行）中的帐户的标识。”。

    当事方在金融机构（如银行）中的帐户的标识。  

6. 单击“新建”，以打开对话框。
7. 在“名称”字段中，键入“货币”。 
8. 在“物料类型”字段中，选择“字符串”。
9. 单击“添加”。
10. 在“描述”字段中，输入“货币代码”。
11. 单击“新建”，以打开对话框。
12. 在“名称”字段中，键入“数字”。 
13. 单击“添加”。
14. 单击“新建”，以打开对话框。
15. 在“名称”字段中，键入“IBAN”。 
16. 单击“添加”。
17. 在“描述”字段中，输入“国际银行帐号”。

## <a name="define-the-payment-message-structure-for-credit-transfer-payment-type"></a>定义贷方转帐付款类型的付款消息结构
1. 单击“新建”，以打开对话框。
2. 在“作为以下项的新节点”字段中，输入“模型根”。
3. 在“名称”字段中，键入“客户贷方转帐发起”。
4. 单击“添加”。
5. 在“查找”字段中，键入“客户贷方转帐发起”。 
6. 单击“查找上一个”。
7. 单击“新建”，以打开对话框。
8. 在“名称”字段中，键入“消息标识”。 
9. 单击“添加”。
10. 在“描述”字段，输入“由指示方分配（并发送给下一个当事方）以标识消息的点对点引用。”。

    点对点引用由指示方分配（并发送给下一个当事方）以明确消息。  

11. 单击“新建”，以打开对话框。
12. 在“名称”字段中，键入“处理日期时间”“。
13. 在“物料类型”字段中，选择“日期时间”。
14. 单击“添加”。
15. 在“描述”字段中，输入“创建付款消息的日期和时间。”。 
16. 单击“新建”，以打开对话框。

    定义此模型的付款交易结构。  

17. 在“名称”字段中，键入“付款”。
18. 在“物料类型”字段中，选择“记录列表”。
19. 单击“添加”。
20. 在“描述”字段中，输入“当前消息付款行”。
21. 单击“新建”，以打开对话框。
22. 在“名称”字段中，键入“贷方”。 
23. 在“物料类型”字段中，选择“记录”。
24. 单击“添加”。
25. 在“描述”字段中，输入“应将金额支付给的当事方。”。 
26. 单击“切换项引用”。
27. 在“查找”字段中，键入“当事方”。
28. 单击“查找下一个”。
29. 单击“确定”。
30. 在“查找”字段中，键入“付款”。
31. 单击“查找下一个”。
32. 单击“新建”，以打开对话框。
33. 在“名称”字段中，键入“借方”。 
34. 单击“添加”。
35. 在“描述”字段，输入“欠(最终)贷方款项的当事方。”。
36. 单击“切换项引用”。
37. 在“查找”字段中，键入“当事方”。
38. 单击“查找下一个”。
39. 单击“确定”。
40. 单击“查找下一个”。
41. 单击“新建”，以打开对话框。
42. 在“名称”字段中，键入“描述”。
43. 在“物料类型”字段中，选择“字符串”。
44. 单击“添加”。
45. 单击“新建”，以打开对话框。
46. 在“名称”字段中，键入“货币”。
47. 单击“添加”。
48. 在“描述”字段中，输入“货币代码”。
49. 单击“新建”，以打开对话框。
50. 在“名称”字段中，键入“交易记录日期”。 
51. 在“物料类型”字段中，选择“日期”。
52. 单击“添加”。
53. 在“描述”字段中，输入“交易日期”。 
54. 单击“新建”，以打开对话框。
55. 在“名称”字段中，键入“指示金额”。  
56. 在“物料类型”字段中，选择“实时”。
57. 单击“添加”。
58. 在“描述”字段，输入“在借方和贷方之间移动的扣除费用前的金额。 该金额应以发起方指定的币种表示”。

    在借方和贷方之间移动的扣除费用前的金额。 该金额应以发起方指定的币种表示。  

59. 单击“新建”，以打开对话框。
60. 在“名称”字段中，键入“End2EndID”。 
61. 在“物料类型”字段中，选择“字符串”。
62. 单击“添加”。
63. 在“描述”字段中，输入“输入发起方分配的唯一标识。 该标识在整个端对端链中传递，无更改”。 
64. 在“名称”字段中，键入“付款模型”。

    付款模型名称与付款形式的预定义接口相符。  

65. 单击“保存”。
66. 关闭该页面。
